To learn how to create a group policy, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in 'Group Policy Management', you'll need to edit a Group Policy. In this case, we're going to edit the 'Domain Wide Group Policy'. To do this, right-click the Group Policy.
  • And then select 'Edit...'.
  • Once in the 'Group Policy Management Editor', the first thing we need to do is configure the service that is required for AppLocker. If this service is not running on the client, the AppLocker settings will be ignored. By default, this service is not running. To turn on this service, we will expand 'Policies' under 'Computer Configuration'.
  • Then 'Windows Settings'.
  • And finally, 'Security Settings'.
  • Once you expand 'Security Settings', select 'System Services'.
  • The service that we need to enable is 'Application Identity'. To enable it, first double-click on the service.

 
  • Once the settings have been opened, select the option 'Define this policy setting'.
  • And then set the startup mode to 'Automatic'. This ensures that when the client computer starts up, the 'Application Identity' service starts running, which will enable AppLocker to work.

  • Press 'OK' to continue.
  • Now that the service is configured, we will now need to configure AppLocker. To do this, first expand 'Application Control Policies'.
  • And then select 'AppLocker'.
  • Before we start configuring AppLocker, we will first start with the AppLocker properties. To open the properties, select the option, 'Configure rule enforcement'.
  • Once the properties have been opened, select the 'Configured' option under 'Executable rules'.
  • Then select the dropdown.
  • Notice you have the option, 'Audit only'. It is recomended, that when you deploy AppLocker in your organization, configure it in audit mode first. This will allow you to test your configuration out first before deploying it.
  • If you choose the option 'Enforce rules' however, this will enforce any rules that have been defined rather than simpy logging the results.
  • The 'Audit only' and 'Enforce rules' options are also available for 'Windows Installer' and 'Scripts' if you wish to use AppLocker with msi packages or scripts like VB script.

  • Press 'OK'.
  • If we now scroll down to the bottom...
  • You'll notice that this gives you an overview of what rules have been created. In this case, no rules have been created as of yet.
  • There are three different areas where rules can be created, 'Executabe Rules', 'Windows Installer Rules', and 'Script Rules'. In this case, we will create some rules for executables. To do this, click on 'Executable Rules'.
  • Once in the 'Executable Rules' section, right-click anywhere in the white space...
  • And select the option, 'Automatically Generate Rules...'. This will launch a wizard that will create a set of rules based on the software on this computer. In this case, we are using a server to create these rules, however, in a production enviroment, you most likely want to use a client computer to create the rules set. This will allow the wizard to see the kinds of software that your organization is using, therefore creating rules based on those applications.
  • The first screen of the wizard will ask which folder do you want to analyze. In this case, we will leave it on the default, 'Program Files'.

  • Press 'Next >'.
  • On this screen, you can decide which type of rules you want the wizard to create. The top option will create Publisher rules for files that are digitally signed.
  • If you choose to create Publisher rules, then you also have the option to use hash values if the file does not contain a digital signature.
  • Or this option of a path rule for non-digitally signed files.
  • This option will create hash rules for all files.
  • The last option will allow the wizard to analyze the rules that it will create, checking to see if there are any overlaps. It is recommended to leave this option on, as it will greatly reduce the number of files created. The less rules you have, the eaiser it will be to manage.

  • Press 'Next >'.

 
  • After pressing 'Next >' on the previous screen, the wizard will start generating the rules.
  • After the rules have been generated, the wizard will bring you to the last screen. This screen will allow you to review what results the wizard has created.
  • If you want to see which files were reviewed, you can select the option, 'Review files that were analyzed'.
  • Here you can see a list of all the files that a rule will be created for. You can clear the check box next to a file to prevent a rule from being created for that file.

  • Press 'OK'.
 
  • You can also select the option, 'View rules that will be automatically created' to review the rules.
  • This will open a screen that shows you all the rules that will be created.

  • Press 'OK'.
  • No rules have been created yet. To create the rules, press the 'Create' button.
  • After pressing the 'Create' button, you will get a message asking if you want to create default rules. Default rules determine what will happen if no other rules match. Without any default rules, this may prevent system files from being allowed to run, so it's a good idea to press 'Yes'.
  • The wizard will now create the rules. However, you also have the option to manually create a rule. To do this we first have scroll down to the bottom...
  • And right-click on the available white space.

    •  
  • And select the option, 'Create New Rule...'. Selecting this option will launch the 'Create Executable Rules' wizard.
  • The first screen of the wizard is the 'Welcome' screen. Press the 'Next >' button to continue.
  • This next screen will ask what the action of this rule will be. In this case, we will select the option 'Deny'. An application that matches this rule will not be allowd to run.
  • On this screen we can also select which users or groups this rule will apply to.

  • Press 'Next >'.

 
  • On this screen, we can select which type of rule to create. In this case, we will leave the default option as 'Publisher'.

  • Press 'Next >'.
  • This screen is where we select the executable that we want to create the rule for. To browse to an executable, press the 'Browse...' button.
  • For this example, we have browsed to executable for Google Chrome. The next step is to select the executable and then press 'Open'.
  • Once the executable has been selected, information about the executable can be read.
  • If we slide the sliding scale up...
  • We can change the matching of the rule to less specific settings. In this case, the rule will look for the publisher of Google and the product Chrome. File name and version number will not be considered by the rule.

  • Press 'Next >'.
  • On this screen, you can configure exceptions for the rule. This is a useful screen that can replace having to create additional rules. For example, lets say you wanted to give a user access to only a particular version of Google Chrome. On this screen, you could add the version of Google Chrome you wanted to run. So efffectively, this rule would say, "deny all versions of Google Chrome, except this one". In this case, we will not add any execptions to this rule.

  • Press 'Next >'.
  • On the last screen, you can configure a name for the rule and add a description if you wish. When you're ready to create the rule, press the 'Create' button.
  • You can see that a new rule has been created, which will prevent Google Chrome from running.
