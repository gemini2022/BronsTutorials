You can use fine grained password policies to specify multiple password policies within a single domain. You can use fine-grained password policies to apply different restrictions for password and account lockout policies to different sets of users in a domain. For example, you can apply stricter settings to privileged accounts and less strict settings to the accounts of other users. In other cases, you might want to apply a special password policy for accounts whose passwords are synchronized with other data sources. To learn how to configure a fine grained password policy, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'ADSI Edit'...
  • ADSI Edit is a low level Active Directory editing tool that is used to modify the Active Directory database. The first thing we will do once we've opended the ADSI Edit window, is to right-click on 'ADSI Edit'.
  • Then select the option 'Connect to...'.
  • In this case, the default option will connect to our domain. This is what we want.

  • Press 'OK' to continue.
  • Next, select the 'Default naming context'.
  • Then expand it.
  • Then select this...

 
  • And expand it.
  • Next, select 'System'...
  • Then expand it.
  • And then finally, select 'Password Settings Container'.
  • To create a password settings object, right-click it.
  • Then select 'New'.
  • 'Object...'.
  • In this case there is only one object to choose from, which is 'msDS-PasswordSettings'.

  • To continue, press 'Next >'.
  • Throughout this wizard, you will be asked several questions that help define the type of 'PSO' you want to create. The first question asks for a friendly name to be entered to identify the 'PSO'. In this case, the name we will enter is 'Password PSO'.

  • Press 'Next >'.

 
  • This next screen will ask for the precedence number. This needs to be a value of '1' or above, with '1' have the highest priority, if there is a conflict. In this case, we will use the value of '2', incase we want to create another 'PSO' later on, we can then use the value of '1' to override the settings in this 'PSO' if we choose to.

  • Press 'Next >'.
  • The next setting in the wizard will ask if the password should be stored using reverible encryption. In this case we will enter in 'False' and move on

  • Press 'Next >'.
  • The next question will ask how many passwords you would like to save using the password history option in Active Directory. For this value, we will enter in '24' and move on.

  • Press 'Next >'.
  • The next question will ask if you want to make sure that when passwords are changed, they must meet complex password requirements. In this case we will enter in 'True' and move on.

  • Press 'Next >'.
  • The next screen will ask what the minimum length of the password should be. In this case, we want to increase the security for password requirements for this 'PSO', therefore will will enter in the value of '10', forcing the users to have a password that is at least ten characters long.

  • Press 'Next >'.
  • The next screen will ask what the minimum age for the password needs to be. Settings like these use the format of 'Day' : 'Hours' : 'Minutes' : 'Seconds'. In this case we will set it to one day, '1:00:00:00', which means the user will only be able to change their password once a day.

  • Press 'Next >'.
  • On this next screen we need to enter in the time period that the user will be able to use the password before it expires and will need to be changed. For this value we will enter in 30 days '30:00:00:00'.

  • Press 'Next >'.
  • On this screen we will be asked to enter in the number in incorrect passwords before the user account will be locked out. In this case we will enter in '5' and move on.

  • Press 'Next >'.
  • The next option indicates the time period before the incorrect password counter is reset. In this case we will enter in a half-hour '0:00:30:00'. This means when an account is locked out because of the number of unsuccessful login attempts exceeds the password policy lockout threshold, it will take a half-hour before the number of unsuccessful login attempts is reset to 0.

  • Press 'Next >'.
  • On this next screen we can enter in a value for how long the account will stay locked out before it will automatically be unlocked. In this case we will enter in one hour '0:01:00:00'. If you want the account to stay locked out until an administrator unlocks it, you need to enter in the value of '(never)'. If you enter in the value of '(never)', the account will stay locked out until an administrator unlocks it.

  • Press 'Next >'.
  • This will complete the wizard. To create the 'PSO', press 'Finish'.
  • The next step is to associate the 'PSO' with a user or group. To do this, right click the object...

 
  • And select 'Properties'.
  • This will show the attributes of the object. If we scroll down...
  • The attribute that we need to edit is 'msDS-PSOAppliesTo'. To edit this attribute, select it...
  • And then press 'Edit'.
  • Then press the 'Add Windows Account...' button.
  • Here is where we will enter in the user or group that the 'PSO' will associate with. In this case we will enter in the group 'laptop users'.

  • Press 'OK'.
  • This is all that needs to be configured. The Password settings object is now being applied to all the users in the 'Laptop Users' group.
