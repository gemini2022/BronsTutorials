To take advantage of the benefits of .admx files, you must create a Central Store in the SYSVOL folder on a domain controller. The Central Store is a file location that is checked by the Group Policy tools. The Group Policy tools use any .admx files that are in the Central Store. The files that are in the Central Store are later replicated to all domain controllers in the domain. To learn how to create a Central Store, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in 'Group Policy Management', right-click 'Group Policy Modeling'.
  • Then select 'Group Policy Modeling Wizard...'.
  • The first screen of the wizard is the welcome screen, to continue, press 'Next >'.
  • The next screen of the wizard will ask which domain controller you wish to connect to. You can choose any domain controller that is above Windows Sever 2003.

  • Press 'Next >'.
  • On this screen, you can select which container or user you want to perform the modeling with. For the user information, we will select 'User'.
  • Then press the 'Browse...' button.
  • And then enter in the user we want to use for the modeling. In this case we will enter in 'administrator'.

  • Press 'OK'.

 
  • For the computer information, we will select the 'Computer' option.
  • Then press 'Browse'.
  • And enter in the computer we want to use for the modeling. In this case we'll enter in 'dc1'.

  • Press 'OK'.
  • Now press 'Next >' to continue.
  • This screen allows you to choose some additional simulation options. Like for example, you can select this option to simulate a slow network connection. This means that the Group Policy setting will be processed as if the computer detected a slow network connection.
  • The next option allows you to configure Loopback processing. If you are thinking about using loopback processing, it is a good idea to test the configuration out in Group Policy Modeling before you configure it for real, in case there are any potential problems.
  • If you select the dropdown...
  • You can select what site will be used. In this case, there is only on site to select from.
  • Press 'Next >' to continue.
  • On this next screen, you can choose where you want the simulation to consider where the user and computer account is to be located. We will keep the default and move on.

  • Press 'Next >'.
  • On this screen, you can choose what security group the user is a member of. This is useful to test out group changes if you are using security filtering.

  • Press 'Next >'.
  • This screen is like the previous screen, but allows you to choose what security group the computer is a member of.

  • Press 'Next >'.
  • This screen you can configure which WMI filters will be used for the user. If we select 'Only these filters'.
  • We then will have the option to select which WMI filters to use. However, as you can see, in this case, there are no WMI filters to choose from.

  • Press 'Next >'.
  • On this next screen we have the same options for WMI filters, except these apply to computer configuration. Again, we are not using any WMI filters, so we will move on.

  • Press 'Next >'.
  • This next screen will provide a summary of all the options that have been chosen.

  • Press 'Next >'.
  • Any finally, the finish screen.

  • Press 'Finish'.
  • Once the wizard is complete, the results will be displayed on the far right of the 'Group Policy Management' window. The first tab will display the summary of the results. If we select the 'Settings' tab...
  • You will be able to view all the Group Policy settings. If we select the 'Query' tab.
  • This tab shows the settings that were used to generate the report.
