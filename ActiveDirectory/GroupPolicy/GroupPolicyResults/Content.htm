To take advantage of the benefits of .admx files, you must create a Central Store in the SYSVOL folder on a domain controller. The Central Store is a file location that is checked by the Group Policy tools. The Group Policy tools use any .admx files that are in the Central Store. The files that are in the Central Store are later replicated to all domain controllers in the domain. To learn how to create a Central Store, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in 'Group Policy Management', right-click on 'Group Policy Results'.
  • Then select 'Group Policy Results Wizard...'.
  • The first screen of the wizard is the welcome screen, to continue, press 'Next >'.
  • This next screen will ask which computer you want to display the Group Policy results on. In this case, we will select a remote computer rather than the local computer. To do this, select the option 'Another computer:'.
  • Once you have selected the option 'Another computer:', you can then enter the computer you want to display the Group Policy results on. In this case, we will type the computer 'ws1'.
  • This option allows for generating results only for user configuration. This speeds up the process if you only require user settings.

  • Press 'Next >'.
  • After pressing the 'Next >' button on the previous screen, you will notice that you will get an error message telling you that the RPC server is unavailable. This is because the other computer has not yet been configured to allow the connection.

  • To fix this problem, there are a few steps that need to be taken on the remote computer you specified earlier. The first step is:
    • Open services on the remote computer and turn on the 'WMI Performance Adapter' service.
  • Then, open the firewall of the remote computer and enable the following inbound rules:
    • Remote Event Log Management (NP-In) - Domain Profile
    • Remote Event Log Management (RPC) - Domain Profile
    • Remote Event Log Management (RPC-EPMAP) - Domain Profile
    • Windows Management Instrumentation (WMI-In) - Domain Profil
  • After all those steps have been completed, press the 'Close' button.

 
  • Now, press 'Next >' to continue.
  • On this next screen, you can choose which user will be used to generate the user results. Any user that has logged into the computer that you want to generate results for will be seen here. You have the option to choose the most current user that has logged in or a specific user. In this case, there has been only one user that has logged in.

  • Press 'Next >'.
  • This next screen of the wizard will show you the settings that were selected.

  • Press 'Next >'.

 
  • Press 'Finish' to complete the wizard.
  • Once the wizard is complete, the results will be displayed on the far right of the 'Group Policy Management' window. The first tab will display the summary of the results.
  • If we select the 'Settings' tab...
  • You will be able to view all the Group Policy settings.
  • And then finally, if you select the 'Policy Events' tab...
  • All events that were generated will be displayed here.
