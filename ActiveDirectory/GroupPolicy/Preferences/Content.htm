Group Policy Preferences is a collection of Group Policy client-side extensions that deliver preference settings to domain-joined computers running Microsoft Windows desktop and server operating systems. Preference settings are administrative configuration choices deployed to desktops and servers. Preference settings differ from policy settings because users have a choice to alter the administrative configuration. Policy settings administratively enforce setting, which restricts user choice. To learn how to configure a Group Policy Preferences, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in 'Group Policy Management', you'll need to open 'Group Policy Management Editor' for any of the group policies defined, or create a new group policy. In this case, we're going to edit the 'Domain Wide Group Policy' that we created in an eariler tutorial. To do this, right-click it.
  • Then select 'Edit...'.
  • Once in 'Group Policy Management Editor', expand the 'Preferences' folder.
  • You can see that 'Preferences' are divided into two sections, 'Windows Settings'...
  • And 'Control Panel Settings'.
  • Let's take a look at 'Window Settings'.
  • For this example, we will start by using 'Preferences' to create a folder on all of the computers in the domain that are under the 'Domain Wide Group Policy'. To do this, right-click on the 'Folders' icon...

 
  • Select 'New'...
  • Then 'Folder'.
  • On the 'New Folder Properties' form, click on the ellipsis button.
  • We want to create a folder on the (C:) drive of each computer, so we'll expand 'Computer'.
  • And then select 'Windows (C:)'.
  • Now click on the 'Make New Folder' button.
  • Once the 'Make New Folder' Button is clicked, a new folder will appear with the option to allow us to name that new folder.
  • We'll name this folder, 'Wallpaper'.
  • When finished naming the folder, press the 'Select' button.
  • Now that the folder is created, we can click on the 'OK' button to continue.
  • In the next step, we'll create some files. To do this, right-click on the 'Files' icon...
  • Select 'New'...
  • Then 'File'.
  • On the 'New File Properties' form, begin by entering in a path to a source file. In this case, the source file we're going to use is a jpg file found under the 'DesktopWappers' directory on 'DC1'.
  • The next part is to enter in the destination. This is where the magic happens. For the destination, we will enter in the 'Wallpaper' folder we created on the (C:) drive.
  • However, we will change the name of the file to 'default'.
  • Next, select the 'Common' tab.
  • On the 'Common' tab, is the option, 'Item-level targeting', probably the most powerful feature of 'Preferences'. Select this option...
  • Then click on the 'Targeting...' button.
  • At the top, select the option 'New Item'.
  • Here you are provided with a list of items that you can specifically target settings for. You can test for a particular operating system, the CPU, RAM, the computer name, just to name a few. In this case, we will select the option 'Operating System'.
  • Here we can choose the operating system we want to target..
  • As you can see, there are several operating systems to choose from, but we will go with the default, 'Windows 7'.
  • Along with choosing an operation system, you also have the option to chose its edition, release, or computer role.

  • Click 'OK' when finished.
  • We now have a setting that will copy the Windows7.jpg image to every computer in the domain, but only if the operating system on that computer is Windows 7.

  • Click the 'OK' button.
  • The next part we will configure is the the 'desktop Wallpaper' setting found under 'User Configuration'. This is not a 'Preference' setting, but this will show how the settings can be used together to achieve a result that would normally not possible without the use of 'Preferences'.

  • First, close 'Preferences'.
  • Then expand 'Policies' under 'User Configuration'.
  • Then 'Administrative Templates'...
  • And then 'Desktop'...
  • And then finally, click on 'Desktop'.
  • Next, double-click on the 'Desktop Wallpaper' setting.
  • This setting is not available under the 'Computer Configuration' side of group policy, but what we are essentially going to do is configure the desktop wallpaper based on the operating system a computer is using.

  • First, we'll enable the setting.
  • Then enter the wallpaper image we're going to use. In this case we're going to use the 'default.jpg' image we created earlier using 'Preferences'. What 'Preferences' will do is make sure that the correct wallpaper image will be used based on the operating system of the computer. However, the setting here that we're configuring now will only ensure that this file is used for the wallpaper.

    •  
  • After specifying the wallpaper image, select the dropdown...
  • Choose the wallpaper style, in this case we're going to select 'Stretch'.
  • And then finally click the 'OK' button to configure the setting.
