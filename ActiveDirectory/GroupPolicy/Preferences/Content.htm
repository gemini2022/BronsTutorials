To learn how to create a group policy, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in 'Group Policy Management', you'll notice a group policy named 'Default Domain Policy'. This group policy is created when the domain is first created. This group policy is required to define the security settings for the domain. It also defines password settings in the domain, such as how long and complex a password must be when setup for a user.
  • The next group policy that is created by default with the domain, can be found under 'Domain Controllers'.
  • If you select this group policy...
  • You will get a warning message. This message informs the administrator that this is a link to another group policy. What this means is the group policy that you selected, esentially works the same as a hyper-link or a shortcut would link to an electronic document. Just like a document can have many shortcuts that link to the same document, group policy can have many links that refer to the same group policy object.
  • Select the 'Do not show this message again' option.

  • Then press 'OK'.
  • If we expand 'Group Policy Objects'...
  • You can see the two default group policy objects that were created when the domain was created. These are the actuall group policy objects. The group policy that we selected earlier was just the link. The benefit of having group policy links is that you can create a group policy and link it to multiple OUs.

 
  • We will now create a brand new group policy. To do this, we can either right-click an OU or right-click on the domain. In this case, we are going to right-click on the 'bronstutorials.local' domain and create the new group policy under this domain.
  • Because we right-clicked on the domain we will get the option, 'Create a GPO in this domain, and link it here...'.
  • Once you select that option, you will be presented with a form that will allow you to name your new group policy. In this case, we are going to name our group policy, 'Domain Wide Group Policy'.

  • Press 'OK' to continue.
  • Notice now that our new group policy that we created is now within the 'Group Policy Object'
  • And the link to that group policy is under the domain where we specified it to be created.
  • Now that the group policy has been created, it can be edited by right-clicking the link, or the object itself. Here we will right-click on the group policy link...
  • And select 'Edit...'.
  • The 'Group Policy Management Editor' will now open and display the settings for this group policy. You can see that this group policy is divided into two parts, which are sometimes referred to as nodes. The first part is, 'Computer Configuration'. Any settings that are configured in this section are applied to the computer as a whole, effecting all users that use that computer.
  • The second part is, 'User Configuration'. Any settings that are configured in this section are applied to users. Since this group policy was created under the domain, the settings that are configured will effect all users in the domain.
  • Both 'Computer Configuration' and 'User Configuration' sections have a folder called 'Policies'. Within the policy folder are where the traditional group policy settings reside. I say traditional because before Sever 2008, the 'Policies' folder did not exist. All of the settings found under policies were essentially all the settings you would find in group policy.
  • But now starting with server 2008, both 'Computer Configuration' and 'User Configuration' sections have a folder called 'Preferences'. Preferences was originally a third party product called 'Policy Maker' that was purchased by Microsoft. There is a big fundamental difference between policies and preferences. Any settings applied under 'Policies', the user is forced to have, they do not have a choice. Any settings applied under 'Preferences', the user has a choice if they want the setting or not.
  • If we open the 'Policies' folder under 'User Configuration'...
  • We will see three additional folders. The first folder is 'Software Settings'.
  • If you decide to use group policy to deploy and manage software, this is where the group policy software settings would be configured.
  • The next folder down is 'Windows Settings'.
  • Things like user logon and logoff scripts can be configured here.
  • Also, things relating to security settings can be configured here.
  • The third folder is 'Administrative Templates'. This folder contains the bulk of the group policy settings.
  • If we select the option at the bottom, 'All Settings', notice that all the group policies settings found under 'Administrative Templates' are shown, 1454 to be exact.
  • To make it easier to find a particular setting, we can right-click any of the folders, in this case the 'All Settings' folder...
  • And select 'Filter On'.
  • Once the filter is on, I can then right-click on that folder again...
  • And select 'Filter Options...' to configure what group policy settings you want to show.
  • Once you select 'Filter Options...' you will be presented with a form that gives you a lot of filtering options. The first option is, 'Managed'. Selecting this option will allow you to filter the list to contain either group policy settings that are managed or unmanaged. If a group policy setting is managed and it is later removed, any changes made by that group policy setting will be reversed. However, if the group policy setting is unmanaged, this will mean that once the group policy setting is removed, the changes will not be reversed and will have to be reversed manually.
  • In this case, we will select 'Yes' so that the list contains only managed group policy settings.
  • The next option allows you to filter the list to contain either group policy settings that are configured or not configured.
  • Since no group policy settings have not yet been configured, we will leave this filter option as 'Any'.
  • The next option allows you to filter the list to contain either group policy settings that have comments applied to them or no comments applied to them.
  • Again, since no group policy settings have not yet been configured, we will leave this filter option as 'Any'.
  • The next option allows a keyword to be entered in. First you must enable this option...
  • Then, enter in the keyword. In this case we're going to enter the keyword 'Desktop Wallpaper'. Doing this will filter out the list to only the group policy settings that contain word 'Desktop Wallpaper' in it.
  • If we then select the option to right...
  • We then have three additional options:
    • Any: The list includes any of the words in the keywords you defined.
    • All: The list includes all of the words in the keywords you defined.
    • Exact: The list includes exact matches of the words in the keywords you defined.

  • In this case, we will go with the option 'Any'.

    •  
  • Here you can choose whether or not the keywords will be within the policy setting title. In this case, we will leave this option selected.
  • Next you can choose whether or not the keywords will be within the help text. As you can see, this option is selected by default, but we will deselect it so that the list doesn't contain any group policy settings that have the words 'Desktop Wallpaper' in its help text.
  • Next you can choose whether or not the keywords will be within the policy setting comments. Again, as you can see, this option is also selected by default, but we will deselect it so that the list doesn't contain any group policy settings that have the words 'Desktop Wallpaper' in its comments.
