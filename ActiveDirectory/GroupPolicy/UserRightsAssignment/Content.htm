To learn how to configure user rights assignments, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in 'Group Policy Management', you'll need to edit a Group Policy. In this case, we're going to edit the 'Domain Wide Group Policy'. To do this, right-click the Group Policy.
  • And then select 'Edit...'.
  • Once in the 'Group Policy Management Editor', we next have to expand 'Policies'...
  • Then 'Windows Settings'.
  • Then, 'Security Settings'.
  • Once you expand 'Security Settings', select 'User Rights Assignment'.
  • On the right side of the window, you will see many different user right policy settings that can be configured. However, the policy settings we are interested in is the 'deny' policy settings. These options can be used to effectively remove access to users or groups without requiring any knowledge of what permissions have already been applied.

 
  • If we right-click on one of the 'Deny' settings, for example, the 'Deny log on through Remote Desktop Services'.
  • And select 'Properties'.
  • We can then configure the settings. To do this is a simple matter of selecting the option 'Define these policy settings'.
  • And then adding the required user or group by selecting the 'Add User or Group...' button
  • In this case, we will add the group 'domain users'.

  • And then press 'OK'.
  • This setting will now prevent any user in the 'domain users' group from using Remote Desktop to access the computer.
