To learn how to configure password policies for the domain, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in 'Group Policy Management', we'll go ahead and take a look at the 'Default Domain Policy'. This Group Policy is created when the domain is created, and is the only Group Policy you should consider configuring the default password settings for your domain. To begin, we'll edit this policy by first right-clicking it.
  • And then select 'Edit...'.
  • Once in the 'Group Policy Management Editor', the password policies can be found in 'Policies'...
  • Then 'Windows Settings'.
  • And then, 'Security Settings'.
  • Then finally, 'Account Policies'.
  • The settings are divided into three parts. The first part is 'Password Policy'.

 
  • Within 'Password Policy' you have the option to configure the following six settings:
    • Enforce password history
    • Maximum password age
    • Minimum password age
    • Minimum password length
    • Password must meet complexity requirements
    • Store passwords using reversible encryption

 
  • Next is 'Account Lockout Policy'.
  • This policy allows the option to configure these three settings:
    • Account lockout duration
    • Account lockout threshold
    • Reset account lockout counter after
  • And then finally 'Kerberos Policy'.
  • With this policy, you have the option to configure these five settings:
    • Enforce user logon restrictions
    • Maximum lifetime for service ticket
    • Maximum lifetime for user ticket
    • Maximum lifetime for user ticket renewal
    • Maximum tolerance for computer clock synchronization
