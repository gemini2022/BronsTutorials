To learn how to create a user, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Users and Computers'...
  • Once in 'Active Directory Users and Computers', expand your domain, in this case it's 'bronstutorials.local'.
  • Once you expand your domain, right-click on the 'Users' OU.
  • Then select 'New'...
  • Then 'User'.
  • Once you get to the new user form, start by entering in the first name of the user.
  • We could also include the user's initials, but it's not necessary, but for this example we will include it.
  • And then finally, the last name. Notice that the 'Full name' field gets populated from the previous fields.

 
  • For the 'User logon name' field, we will use the first initial and last name.
  • Now, if we select the drop down button that's to the right of the logon name...
  • You will notice an option for a .com suffix. This option is available because we created this upn suffix in an earlier tutorial. It is important to note that if you log in using a upn suffix, you need to have access to the global catalog server. This is because the user / upn suffix combination must be unique for the forest.
  • Next, you have the option to configure a pre-Windows 2000 name. In most cases, it is simplest to leave this the same as the user logon name above. This will generally only be a problem if the user name is over twenty characters.

  • Click on the 'Next' button to continue.
  • On this screen, you first have to enter in a password for the new user. Type the new user's password in the 'Password' field and the 'Confirm password' field.
  • By default, the option 'User must change password at next logon' will be selected. When the user attempts to logon to the network for the frist time, they will be asked to change their password. This provides additional security, as it means the administrator will not know the password. If this option was not selected and the user did not change their password, the user could accuse the administrator of accesing their account, since the administrator set the password originally and thus knows what the password is.
  • The next option prevents the user from changing the password. Usually this option is selected when creating a shared account. For example, an account that may be used to access an internet kiosk used by many users. You don't want one user changing the password and preventing other users from using the kiosk later on. Also this option is sometimes used for a service account, or for an account used to run a particular application. If the password for this account was to change, especially if the account was used on multiple computers, this would prevent the application or service from running. Therefore, keeping this option un-checked for those reasons is a good idea.
  • The next option, 'Password never expires', means the account will be exempted from the domain password policy requiring the password to be changed after a certain amount of time. This option is normally selected when creating service accounts. Service accounts are often created to run applicaions like Exchange. If the password for the service account were to expire, and it was being used to run software, such as Exchange, all the Exchange servers using the account would stop running, and no one would be able to access their email. You can see why this option if often selected for service accounts.
  • The last option disables the account. Once the account is created, you can disable the account at any time, but you may want to create a new user account that is in a disabled state for many different reasons. Until the account is enabled, no one will be able to use the account, for this reason, it is perfect for a template user account.

  • Press 'Next' to continue.
  • After pressing 'Next' on the previous screen, you will be taken to the last screen of the wizard.

  • Press the 'Finish' button to create the new user.

Once the account is created, you can disable the account at any time, but you may want to create a new user account that is in a disabled state for many different reasons. Until the account is enabled, no one will be able to use the account, for this reason, it is perfect for a template user account.
