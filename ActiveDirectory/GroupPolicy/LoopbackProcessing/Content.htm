Loopback processing changes the algorithm used to apply Group Policy to a computer. This allows the administrator to effectively ignore or merge the user configuration that would be normally applied to the computer. To learn how to configure Loopback Processing, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • In this example we have an OU that we created in an earlier tutorial called 'New York'. We will use this OU to demonstrate how to configure loopback processing. First, expand the 'New York' OU.
  • Next, we will expand the 'Computers' OU.
  • Then we'll expand the 'Training Lab' OU.
  • Inside the 'Training Lab' OU we have created a group policy called 'Training Lab Group Policy'. This group policy will be applied to all of the computers that reside in the 'Training Lab' OU. Right-click this group policy...
  • And select 'Edit...'.
  • Configuring Group Policy 'Loopback Processing' is done in the 'Computer Configuration' section of Group Policy. But before we configure Loopback processing, we'll first open 'Policies' found under the 'User Configuration' section.
  • Then we'll open 'Administrative Templates'.

 
  • Inside 'Administrative Templates', we'll open 'Desktop'.
  • And then finally we'll select the 'Desktop' settings.
  • In the 'Desktop' settings, we'll open the setting 'Desktop Wallpaper'.
  • You'll notice that this setting has been configured, and keep in mind that this setting has been configured under the user side of group policy. This user setting will configure the desktop wallpapper when it is applied. However, since no user account exists in the 'Training Lab' OU where this group policy resides, this setting will never get applied. This is where 'Loopback Processing' will come into play. When 'Loopback Processing' is configured to this 'Group Policy', it will apply this 'User Configuration' setting to the 'Computer Configuration' side of group policy.
  • So lets configure 'Loopback Processing' for this 'Group Policy'! Configuring 'Loopback Processing' is done in the 'Computer Configuration' section of Group Policy, so we'll close the 'User Configuration' section.
  • And then open 'Policies' under the 'Computer Configuration' section.
  • And then expand 'Administrative Templates'.
  • Inside 'Administrative Templates', open 'System'.
  • And then finally select the 'Group Policy' settings.
  • The setting that we are looking for is called 'User Group Policy Loopback Processing Mode'. To get to it, we have to scroll down the list.
  • Then when you find it, open the 'User Group Policy Loopback Processing Mode' setting by double-clicking it.
  • Once this setting is open, all you need to do is enable it...
  • And choose which mode type this setting will be configured with.
 
  • Selecting the 'Replace' option will change how user configuration is applied. Instead of user configuration being applied based on where the user account is in Active Directory, user settings are applied based on where the computer account is located in Active Directory.
  • If the 'Merge' option is selected, Group Policy is first applied like it would be normally. That is, the computer configuration based on where the computer account is located in Active Directory and user configuration based on where the user account is located in Active Directory is applied. The difference is that an extra step is added. The extra step applies user configuration based on where the computer account is located in Active Directory. This is often used for Remote Desktop Services where you want the user to have their user settings applied, but want the option to override or add additional settings as required. 
  • Press 'OK' when finished.
