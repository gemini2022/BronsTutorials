To learn how to create a group policy, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in 'Group Policy Management', right-click a Group Policy object. In this case we are going to right-click on the 'Domain Wide Group Policy'. We are selecting this GPO because we want to deploy some software to users and computers throughout the entire domain.
  • After right-clicking a Group Policy, select 'Edit'.
  • In this example we will be deploying software to all computers in this domain. Since we will be deploying software to computers, we will begin by expanding the 'Policies' folder under 'Computer Configuration'.
  • Then expand 'Software Settings'.
  • After expanding 'Software Settings', right-click on 'Software installation'.
  • Then select 'New'...
  • And then 'Package...'.

 
  • The first step is to browse to the location of the install media.
  • Here we will enter in the path to the shared folder that holds the install media.

  • Then press 'Open'.
  • Then select the msi package you want to use for installation. In this case, we are going to use 'googlechromestandaloneenterprise64.msi' to install Google Chrome to all computers in the domain.

  • Then press 'Open'.
  • After pressing the 'Open' button, you will be given a screen asking the method of deployment. You will notice that the 'Published' option is greyed out. This is because the software is being deployed at the computer level. When deploying software at the computer level, only the 'Assigned' method of deployment can be used.
  • However, for this example, we will select the 'Advanced' option. Selecting the 'Advanced' option gives us the ability to reconfigure any options later. Selecting the 'Assigned' option would limit us to what options could be changed at a later time. For this reason, it's a good idea to always select the 'Advanced' option to configure the options you want, knowing you can change them later.

  • Press 'OK'.
  • Pressing 'OK' on the previous screen brings us to the properties of the msi package. From here, we will select the 'Deployment' tab.
  • Here on the 'Deployment' tab, are additonal options that can be configured. In this case, because the application is being deployed at the computer level, most of the options are greyed out. We do however, have the option to 'Uninstall this application when it falls out of the scope of management'. What this means is that if the Group Policy is no longer being applied to a computer, for example, if it is deleted or permissions are changed, the application will then be removed from the computer.
  • This is all that is required for software to be installed on all computers for this Group Policy. The installation will be performed when each computer starts up next.

  • Press 'OK'.
  • Now for this example, we will deploy software to all users in the domain. Therefore, we will begin by expanding the 'Policies' folder under 'User Configuration'.
  • Then expand 'Software Settings'.
  • And then like last time, after expanding 'Software Settings', right-click on 'Software installation'.
  • Select 'New'...
  • And then 'Package...'.
  • 'Windows' will return us back to the location we were last time when we selected an msi package. So we will again select 'googlechromestandaloneenterprise64.msi' to install Google Chrome, but now to all users in the domain.

  • Then press 'Open'.
  • After pressing the 'Open' button, we will again be given a screen asking for the deployment method. You will now notice that the 'Published' option is now available. This is because the software is being deployed at the user level. The 'Published' option is not available when deploying software to computers.
  • In this case we will be using the 'Published' option, but we will select the 'Advanced' option so we can configure all the options later on if we need to.
  • If we select the 'Deployment' tab...
  • Notice that the 'Published' option is selected by default.
  • If we select the 'Modifications' tab...
  • This tab will allow us to apply a transform to the msi package when it is installed. A Transform (*.MST) file allows you to collect installation options for programs that use the Microsoft Windows Installer in a file. For example, automatically accepting license agreement so a user does not need to. They can be used on the Installer (MSIEXEC.EXE) command line, or used in a software installation Group Policy in a Microsoft Active Directory domain.
  • If we now select the 'Upgrades' tab...
  • Here we have the option of adding upgrade packages. For example, if an update for a particualr software was released, you could add it here. When you add an upgrade package, you will get the option to either uninstall the old package, or leave the old package if the upgrading pacakge upgrades it.
  • If you select the option 'Required upgrade for existing packages', this will make sure the upgrade is installed if the previous package was installed. This is a useful feature to select if you deploy an important security update or your company requires that old versions of the software be upgraded.

  • Press 'OK'.
  • If we now go back to 'Software installation', and select it.
  • We can then right-click on one of the software applications, in this case we will select the only one we have, which is 'Google Chrome'.
  • And select 'All Tasks'.
  • And then take a look at the option, 'Redeploy application'. This will, as it suggests, reinstall the application. In some cases, you may want to reinstall an application after it was installed. However, in most cases, once the application has been installed, you will not have a reason to redeploy it.
  • This option here, if selected, removes the application. If we select it...

    •  
  • You will notice you are then given two options. The first option uninstalls the application immediately. If you no longer require the application, this is the best option to select.
  • The second option does NOT remove the application and allows the user to continue to use it. This effectively stops any new installs, but allows any current installs to be used.
