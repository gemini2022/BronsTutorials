The 'Active Directory Domain Services' role has been added, but this server is not yet a domain controller. In order to make this server a domain controller, proceed with the following steps:
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then type, 'dcpromo' in the search field.
  • After, run the 'dcpromo' command by clicking on 'dcpromo' at the top of the menu. This will launch the 'Active Directory Domain Services Installation Wizard'.
  • The first screen of the wizard is the 'Welcome Screen'.

  • Click on the 'Next' button to continue.
  • The next screen explains about old NT style cryptography and that any client using the old NT style cryptography will not be able to use a Windows 2008 and 2008 R2 Server.

  • Click on the 'Next' button to continue.
  • Select the 'Create a new domain in a new forest' option because we are creating a brand new domain from scratch.

  • Then click on the 'Next' button.
  • Type in your domain name or better known as your fully qualified domain name (FQDN) in the text field.

  • Click on the 'Next' button to continue.
  • This screen gives you the option to raise your forest functional level. If you do choose to raise the forest functional level, it is important to know that you will not be able to change it back to a lower functional level. However, you may raise it again to a higher functional level, if you have that option, but you can only raise it, not lower it.

  • Click on the 'Next' button to continue.
  • This screen gives you the option to raise your domain functional level. The same rules apply for raising the domain functional level as it is for raising the forest functional level. Once you raise the domain functional level, you will not be able to change it back to a lower functional level.

  • Click on the 'Next' button to continue.
  • This screen gives you additional domain controller options. Leave the 'DNS server' option as selected because Active Directory needs a DNS server to operate. Having this option selected will install DNS for us on this server. Also, since this is the first domain controller in the forest, this server must also be a global catalog server. I will cover more about global catalog servers in a later tutorial. Read-only domain controllers we will not be concerned with at this time.

  • Click on the 'Next' button to continue.
  • Once you press 'Next' on the previous screen, a message will appear that simply states that no DNS zone could be found for bronstutorials.local. Since DNS has not been installed yet, this is to be expected.

  • Click on the 'Yes' button to continue.
  • The next screen will ask you where you want to store the Active Directory database. In this case, we will leave them on the same hard drive. On a busy domain controller it is a good idea putting the database on a seperate hard disk.

  • The log files record changes as they happen, allowing the database to be recovered if the hardware was to suddenly crash. You could also put the log files on a seperate hard disk if you wanted to.

  • The SYSVOL folder contains all the login scripts and group policy settings for the domain. This data is replicated throughout the domain to every domain controller in the domain. In most cases, you will leave this on the default.

  • Click on the 'Next' button to continue.
  • The next screen will ask you for a 'Directory Services Restore Mode Administrator' password. This password is different from any other user name and password on the system. It does not allow you access to the domain or to any services. It is included to provide access to the database for recorvery tools when things go wrong.

  • Enter in a password fo both the password and confirm password input fields.

  • Then click on the 'Next' button to continue.
  • The next screen of the wizard gives you a summary of all the settings you have configured.

  • Click on the 'Next' button to continue.
  • Once you press 'Next' on the previous screen, Active Directory Domain Services will then start installing. The process of installing Active Directory for the first time will take some time to complete.
  • The last screen of the wizard is the 'Finish' screen. Click on the 'Finish' button to compelte the wizard.
  • Once the wizard is complete, click on the 'Restart Now' button to restart the server so that the changes made by the Active Directory Domain Services wizard can take effect.
