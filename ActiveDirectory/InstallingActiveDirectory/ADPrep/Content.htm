Adprep.exe is a command-line tool that is included on the installation disk of each version of Windows Server. If you are adding a domain controller that runs a version of Windows Server that is later than the latest version that is running in your existing domain or if you asre upgrading a domain controller to a version of Windows Server that is later than the latest version that is running in your existing domain, then you will have to run Adprep. Adprep.exe has parameters that perform a variety of operations that help prepare an existing Active Directory environment for a domain controller that runs a later version of Windows Server. To learn what these parameters are and how to run Adprep, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Once the command prompt is open, you'll have to navigate to the adprep folder found in the support directory of your Windows Server installation DVD.
  • The first parameter of Adprep will prepare the forest. To do this, run the command adprep /forestprep.
  • After you run the command, you will be given a message informing you that all domain controllers need to be at least Windows 2000 with service pack 4 or above.
  • When you are ready to proceed, press the 'c' key and then press 'Enter'.
  • Changes will now be made to the Active Directory forest to prepare it for a newer version of Windows Server.
  • Now that the changes to the forest have been made, you are now ready to implement the second parameter of Adprep, which will prepare the domain. To do this, run the command adprep /domainprep.
  • After you run the command, you will be given another message telling you to run the 'gpprep' command as well to update some of the security settings. The 'gpprep' command is required to run on the domain controller that is holding the 'Operations Master' role. Once you verify that you are on the server that is holding the 'Operations Master' role, run the command adprep /domainprep /gpprep.
  • The last command you will only have to run if you are planning on deploying a read-only domain controller. The command is simply adprep /rodcprep.
  • That's it! Your Active Directory enviroment is now ready for the new version of Windows Server.
