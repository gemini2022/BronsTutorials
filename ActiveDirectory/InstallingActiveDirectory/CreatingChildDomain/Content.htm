The purpose for creating child domains can be for a few different reasons. One reason my be that a group of users/departments needs to be separated for legal reasons, for example a branch of the company that is being spun-off as an independent business. Another reason may be that there is a separation of the IT staff. Some companies may have departments or groups that must run independent of one another based on the specific technology needs. Or, another reason may be simply for security reasons. HR is a typical example. The addition of another domain should not be taken lightly. The extra expense and complexity are typically not warranted. Handling isolation of users, hardware or resource for administrative or security reasons can easily be handled via Organizational Structures, Group Policies and Access Control Lists (ACLs). To create a child domain, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then type, 'dcpromo' in the search field.
  • After, run the 'dcpromo' command by clicking on 'dcpromo' at the top of the menu.
  • Some Active Directory binaries will have to be installed on the system before the Active Directory wizard can launch.
  • The first screen of the wizard is the 'Welcome Screen'.

  • Click on the 'Next' button to continue.
  • The next screen explains about old NT style cryptography and that any client using the old NT style cryptography will not be able to use a Windows 2008 and 2008 R2 Server.

  • Click on the 'Next' button to continue.
  • In this next screen, you have the option to create a domain controller in an existing forest or for a new forest. In this tutorial, we will be creating a domain controller in an existing forest, so select the 'Existing forest' option.

  • Below the 'Existing forest' option, are two additional options. The first option allows you to add a domain controller to an existing domain. The second option will create a new domain in an existing forest, often referred to as a child domain. Since we want to create a child domain, select the 'Create a new domain in an existing forest' option.

  • Then click on the 'Next' button to continue.
  • In this screen, you are asked to provide the name of the domain where you want to create the child domain. In this case, we will be creating the child domain in the 'bronstutorials.local' domain.
  • Next, you'll need to specify an account that has the proper credentials to perform this task of creating a child domain. To do this, first click on the 'Set...' button.
  • And then, once the 'Set..' button has been clicked, a little dialog box will appear. It is here where you will provide a user name and password of an account that has enough access to add child domains. This will need to be someone who is a member of the 'Enterprise Administrators' group.

  • Once the account credentials has been specified, click 'OK'.
  • Click on the 'Next' button to continue.
  • This screen will now ask for the parent domain of the new child domain. Type the name of your parent domain in the 'FQDN of the parent domain' text field. In this case, the root domain will be 'bronstutorials.local'.

  • In the next text field labeled 'Single-label DNS name of the child domain', enter the name of the child domain. In this case, the name of the child domain will be called, 'east'.

  • In the bottom text field, you will see the fully qualified domain name being displayed. In this case it is 'east.bronstutorials.local'.

  • Click on the 'Next' button to continue.

  • This screen gives you the option to raise your domain functional level. If you do choose to raise the domain functional level, it is important to know that you will not be able to change it back to a lower functional level. However, you may raise it again to a higher functional level, if you have that option, but you can only raise it, not lower it.

  • Click on the 'Next' button to continue.
  • If you have more sites in your environment, select the one that corresponds to this domain, otherwise, leave as the 'Default-First-Site-Name'.

  • Click on the 'Next' button to continue.
  • This screen gives you additional domain controller options. Since we currently have a DNS server on this network already, we will deselect the 'DNS server' option. However, having the 'Global catalog' option selected is a good idea. In most cases, you'll want all your domain controllers to be a global catalog server. Read-only domain controllers we will not be concerned with at this time, so leave that option unchecked.

  • Click on the 'Next' button to continue.
  • Once you press 'Next' on the previous screen, a message will appear that simply states that we have chosen not to install DNS and that our existing DNS infrastructure that is installed on the parent domain will be used.

  • Click on the 'Yes' button to continue.
  • The next screen will ask you where you want to store the Active Directory database. In this case, we will leave them on the same hard drive. On a busy domain controller it is a good idea putting the database on a seperate hard disk.

  • The log files record changes as they happen, allowing the database to be recovered if the hardware was to suddenly crash. You could also put the log files on a seperate hard disk if you wanted to.

  • The SYSVOL folder contains all the login scripts and group policy settings for the domain. This data is replicated throughout the domain to every domain controller in the domain. In most cases, you will leave this on the default.

  • Click on the 'Next' button to continue.
  • The next screen will ask you for a 'Directory Services Restore Mode Administrator' password. This password is different from any other user name and password on the system. It does not allow you access to the domain or to any services. It is included to provide access to the database for recorvery tools when things go wrong.

  • Enter in a password fo both the password and confirm password input fields.

  • Then click on the 'Next' button to continue.
  • The next screen of the wizard gives you a summary of all the settings you have configured.

  • Click on the 'Next' button to continue.
  • Once you press 'Next' on the previous screen, Active Directory Domain Services will then start installing.
  • The last screen of the wizard is the 'Finish' screen. Click on the 'Finish' button to compelte the wizard.
  • Once the wizard is complete, click on the 'Restart Now' button to restart the server so that the changes made by the Active Directory Domain Services wizard can take effect.
