A domain tree is made up of several domains that share a common schema and configuration, forming a contiguous namespace. What this means is, if you are creating a new domain in a forest and the domain DNS namespace is not related to the other domains in the forest, then you would create a new domain tree. Some of the reasons why you might want to create a domain tree would be either an acquisition or a merger. For example, company A has been acquired by company B, or company A has merged with company B, and part of that acquisition, or part of that merger, might be that company A gets to maintain their own identity. Therefore, if compnay B brings them in and they migrate them in as a child domain of their active directory forest, then company A really wouldn't be maintaining their identity. So, what they would want to have is a separate tree in the environment, so that that company A can maintain their own identity. To learn how to setup a new domain tree, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then type, 'dcpromo' in the search field.
  • After, run the 'dcpromo' command by clicking on 'dcpromo' at the top of the menu.
  • Some Active Directory binaries will have to be installed on the system before the Active Directory wizard can launch.
  • The first screen of the wizard is the 'Welcome Screen'. Select the option, 'Use advanced mode installation'.

  • Then click on the 'Next' button to continue.
  • The next screen explains about old NT style cryptography and that any client using the old NT style cryptography will not be able to use a Windows 2008 and 2008 R2 Server.

  • Click on the 'Next' button to continue.
  • In this next screen, you have the option to create a domain controller in an existing forest or for a new forest. In this tutorial, we will be creating a domain controller in an existing forest, so select the 'Existing forest' option.

  • Below the 'Existing forest' option, are two additional options. The first option allows you to add a domain controller to an existing domain. The second option will create a new domain in an existing forest, often referred to as a child domain. However, because we are in 'Advanced mode installation', we are presented with an aditional option, 'Create a new domain tree root instead of a new child domain'. Select that option.

  • Then click on the 'Next' button to continue.
  • In this screen, you are asked to provide a domain of the forest where you plan to create the new domain tree.
  • Next, you'll need to specify an account that has the proper credentials to perform this task of creating a domain tree. To do this, first click on the 'Set...' button.
  • And then, once the 'Set..' button has been clicked, a little dialog box will appear. It is here where you will provide a user name and password of an account that has enough access to add a new domain tree. This will need to be someone who is a member of the 'Enterprise Administrators' group.

  • Once the account credentials has been specified, click 'OK'.
  • Click on the 'Next' button to continue.
  • Type in the name of the new domain tree, or better known as your fully qualified domain name (FQDN) in the text field. In this example, we are going to use, 'otherdomaintree.local'.

  • Click on the 'Next' button to continue.
  • Since we are in 'Advanced mode installation', we are presented with the option to accept the default NetBIOS name that the wizard generated or type a new name.

  • After you have made the choice to either accept the default NetBIOS name or change it, proceed by clicking on the 'Next' button.
  • This screen gives you the option to raise your domain functional level. If you do choose to raise the domain functional level, it is important to know that you will not be able to change it back to a lower functional level. However, you may raise it again to a higher functional level, if you have that option, but you can only raise it, not lower it.

  • Click on the 'Next' button to continue.
  • If you have more sites in your environment, select the one that corresponds to this domain, otherwise, leave as the 'Default-First-Site-Name'.

  • Click on the 'Next' button to continue.
  • This screen gives you additional domain controller options. Leave the 'DNS server' option as selected. This is because we are creating a new tree in the forest and its a good idea to have this domain and any child domain of this tree to use this DNS. Also, having the 'Global catalog' option selected is a good idea too. In most cases, you'll want all your domain controllers to be a global catalog server.

  • Click on the 'Next' button to continue.
  • Once you press 'Next' on the previous screen, a message will appear that simply states that no DNS zone could be found for 'otherdomaintree.local'. Since DNS has not been installed yet, this is to be expected.

  • Click on the 'Yes' button to continue.
  • The next screen will ask you to choose a replication partner for the installation or have the wizard choose an appropriate domain controller for you. Since we only have one domain controller to choose from, we can just simply leave the default selection as, 'Let the wizard choose an appropriate domain controller'.

  • Click on the 'Next' button to continue.
  • The next screen will ask you where you want to store the Active Directory database. In this case, we will leave them on the same hard drive. On a busy domain controller it is a good idea putting the database on a seperate hard disk.

  • The log files record changes as they happen, allowing the database to be recovered if the hardware was to suddenly crash. You could also put the log files on a seperate hard disk if you wanted to.

  • The SYSVOL folder contains all the login scripts and group policy settings for the domain. This data is replicated throughout the domain to every domain controller in the domain. In most cases, you will leave this on the default.

  • Click on the 'Next' button to continue.
  • The next screen will ask you for a 'Directory Services Restore Mode Administrator' password. This password is different from any other user name and password on the system. It does not allow you access to the domain or to any services. It is included to provide access to the database for recorvery tools when things go wrong.

  • Enter in a password fo both the password and confirm password input fields.

  • Then click on the 'Next' button to continue.
  • The next screen of the wizard gives you a summary of all the settings you have configured.

  • Click on the 'Next' button to continue.
  • Once you press 'Next' on the previous screen, Active Directory Domain Services will then start installing.
  • The last screen of the wizard is the 'Finish' screen. Click on the 'Finish' button to compelte the wizard.
  • Once the wizard is complete, click on the 'Restart Now' button to restart the server so that the changes made by the Active Directory Domain Services wizard can take effect.
