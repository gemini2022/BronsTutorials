If a domain controller that holds one or more of the five FSMO roles becomes permanently unavailable, you’ll ultimately need to seize the roles to another domain controller. Seizing FSMO roles is not a graceful process and is intended only to be performed when the unexpected occurs. In normal day-to-day operations, if you need to change what domain controller a FSMO role is held by, you should instead transfer the role. But, if you can not transfer the role, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Then run the command 'ntdsutil'.
  • When you launch 'ntdsutil', you will get a 'ntdsutil' prompt. The 'ntdsutil' has a lot of sub-components, which will allow you to run different functions and configure different settings. You can always tell which part you are in by looking at the prompt. In this case, we want to seize an operation master role.
  • To begin with seizing a role, type in 'Roles', then press enter.
  • After you have pressed enter from the previous screen, you will notice that the prompt has changed to 'fsmo maintenance'. This is to let you know that you are now in a different part of the utility.
  • Before you can make changes to any of the operation master roles, you need to enter in the command, 'connections'. In order for the 'fsmo maintenance' to run any function, it needs to be connected to a domain controller.
  • The command prompt has now changed to 'server connections'.
  • The next command that you will run is, 'connect to server dc1'.
  • This will establish a connection back to one of your domain controllers.
  • Now that you have a connection to a domain controller, you can enter in 'quit' and press enter.
  • This will return you back to the 'fsmo maintenance' section.
  • If you enter in '?'...
  • You can get a list of all the commands that can be run from 'fsmo maintenance'. In this case, the command that we are interested in is the 'Seize RID master' command.
  • Type the command, 'Seize RID master'.
  • Before continuing, Windows will fist confirm if you want to seize the role. Since this is what you want to do, press 'Yes'.
  • After you press 'Yes' on the prompt, you will notice that Windows will attempt to transfer the operation master role first before seizing it.
  • Here you will notice that the 'RID Master' is now on 'DC1'. This means that the 'RID Master' role has been seized by 'DC1'. However, the configuration information about 'DC2' (the server where the 'RID Master' role used to be), still remains in the the 'Active Directory' database. It is best to remove 'DC2' information data from 'Active Directory'.
  • To do this, first type in, 'quit' then press enter.
  • This will bring you back to the 'ntdsutil' prompt.
  • To remove 'DC2' information data from 'Active Directory' you nedd to enter in 'metadata cleanup'.
  • This will bring you to the 'metadata cleanup' prompt.
  • Now we need to select the server where we want to perform the metadata cleanup on, in this case it is 'DC2'. To do this, type in, 'select operation target'.
  • There are a number of settings that need to be set so the correct records can be selected and removed.
  • First you need to select the domain. To find out which domains are available, run the command, 'list domains'.
  • In our case, there is only one domain in our network, therefore the domain number we are after is '0'. If we had more domains in our network, the 'list domains' commnad would list each domain with its associated number.
  • Type, 'select domain 0'.
  • Next, you need to select the site that 'DC2' was assigned to.
  • First, type in 'list sites'.
  • Again, in our case there is only one site in our network, therefore the site number we are after is '0'.
  • Type, 'select site 0'.
  • Lastly, you need to select the server that you want to remove.
  • To do this, run the command, 'list servers in site'.
  • As you can see, 'DC2' is listed as server number '1'.
  • Therefore, you can select it with the command, 'select server 1'.
  • Now you have everything selected that needs to be selected in order to remove 'DC2' information data from 'Active Directory'.
  • Type in, 'quit' then press enter.
  • This will bring you back to the 'metadata cleanup' prompt.
  • To remove 'DC2' from 'Active Directory', run the command, 'remove selected server'.
  • Windows will prompt you, asking if you want to proceed. Press, 'Yes' to continue.
  • Once you press 'Yes' on the prompt, the domain controller 'DC2' will be removed from 'Active Directory'.
  • There is one last record that needs to be removed to complete the process. To remove this record, begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select, 'Active Directory Sites and Services'.
  • Once in 'Active Directory Sites and Services', open 'Sites'.
  • Then open 'Default-First-Site-Name'.
  • Then open 'Servers'.
  • Then select on the 'Servers' folder.
  • If you look to the right side of your screen you'll find a list of all the domain controllers in your domain. As you can see, 'DC2' is no longer listed as being in the domain as a domain controller, so this means the 'metadata cleanup' has worked.
  • Select 'DC2' in the list, and press 'Delete' to complete the removal process.
  • Once you press 'Delete', you will get a prompt asking if this is what you want to do. Press the, 'Yes' button to proceed.
  • Once you press 'Yes' on the prompt, 'DC2' will be removed from 'Active Directory'.
