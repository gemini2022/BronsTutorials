The Schema Master is found only in the forest root domain (The first domain you create when you create a new forest). This means there is only one Schema Master in a forest, and the purpose of this role is to replicate schema changes to all other domain controllers in the forest. Since the schema of Active Directory is rarely changed however, the Schema Master role will rarely do any work. Typical scenarios where this role is used would be when you deploy Exchange Server onto your network, or when you upgrade domain controllers from Windows 2000 to Windows Server 2003, as these situations both involve making changes to the Active Directory schema. To view which domain controller is currently holding this master role or to move this master role to another domain controller, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Then run the command 'regsvr32 schmmgmt.dll'.
  • Windows will then give you a message to let you know the .dll was registered.
  • Next, go back down to the bottom left corner of your screen and select 'Start'.
  • There is no shortcut in the start menu to run the necessary tools needed to view the Schema Master, so you'll need to run mmc. To do this, type 'mmc' in the search field
  • Then click on 'mmc' at the top of the menu. This will open the mmc console.
  • Once the 'mmc console' is open, select 'File' located in the upper left corner of your screen.
  • Then select 'Add/Remove Snap-in...'. This will open the 'Add or Remove Snap-ins' form.
  • When the 'Add or Remove Snap-ins' form opens, select the 'Active Directory Schema' snap-in. This snap-in is available in the list because we registered the .dll.
  • Then click on the 'Add' button to add it to the 'Selected snap-ins' list.
  • Once the 'Active Directory Schema' snap-in is added to the 'Selected snap-ins' list, click 'OK'.
  • Back at the 'mmc console' in the upper left corner of your screen, select 'Active Directory Schema'.
  • Then right click on 'Active Directory Schema'.
  • From here, select the option 'Operations Master...'.
  • At the top, you can see what domain controller is currently holding the 'Schema Master'. At the bottom, we can see what domain controller we want the 'Schema Master' to be transfered to. In this case, both the current domain controller and the target domain controller are the same, 'DC1'.
  • Because both the current domain controller and the target domain controller are the same, when the 'Change' button is clicked...
  • You will get a message telling you that in order to transfer the Operations Master, you must first target a different domain controller.
  • To do this, first close the 'Change Schema Master' form by clicking on the 'Close' button at the bottom of the form.
  • Then go back to the 'mmc console' and at the upper left corner of your screen, right click on 'Active Directory Schema'.
  • This time, select the option 'Change Active Directory Domain Controller...'. This will open the 'Change Directory Server' form.
  • At the top of the 'Change Directory Form' you'll see the current directory server. In this case, it's 'DC1'. At the bottom of the form there will be a list of all the domain contollers in your domain. It is there where you will select the target domain controller. Here we are selecting 'DC2'.

  • Once you select the domain controller of your choice, click 'OK'.
  • After clicking on the 'OK' button on the previous form, you will get a message informing you that only changes of the 'Schema Master' will be able to be made on the domain controller that holds the 'Schema Master'.

  • Click 'OK'.
  • Now that we've established the new domain controller that we will transfer the 'Schema Master' role to, we can once again go back to the 'mmc console'. Notice that the 'Active Directory Schema' is now displaying 'DC2'.

  • Right click on 'Active Directory Schema' again.
  • Then select 'Operations Master...'.
  • Notice the current 'Schema Master' holder still remains as 'DC1', this is because we haven't changed that yet, but as you can see at the bottom, the target domain controller is now 'DC2'.
  • Click on the 'Change' button.
  • Once we click on the 'Change' button on the previous form, we will get a message asking if transfering the 'Schema Master' is what we want to do. Since this is what we want to do, click on the 'Yes' button.
  • We will then get a message telling us that the transfer was successful. Proceed by clicking 'OK'.
  • As you can see, the domain controller that now holds the 'Schema Master' role is 'DC2'.

  • Click 'Close'.
