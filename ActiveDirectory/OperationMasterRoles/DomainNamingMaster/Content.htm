Like the Schema Master role, the Domain Naming Master also resides in the forest root domain. The Domain Naming Master role processes all changes to the namespace, for example adding the child domain vancouver.mycompany.com to the forest root domain mycompany.com requires that this role be available, so if you can't add a new child domain or new domain tree, check to make sure this role is running properly. To view which domain controller is currently holding this master role or to move this master role to another domain controller, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Domains and Trusts'...
  • Right click on 'Active Directory Domains and Trusts' found at the top left side of your screen. 
  • From here, select the option 'Operations Master...'.
  • At the top, you can see what domain controller is currently holding the 'Domain Naming Master'. At the bottom, we can see what domain controller we want the 'Domain Naming Master' to be transfered to. In this case, both the current domain controller and the target domain controller are the same, 'DC1'.
  • Because both the current domain controller and the target domain controller are the same, when the 'Change'button is clicked...
  • You will get a message telling you that in order to transfer the Operations Master, you must first target a different domain controller
  • To do this, first close the 'Operations Master' form by clicking on the 'Close' button at the bottom of the form.
  • Then go back and right click on 'Active Directory Schema'.
  • This time, select the option 'Change Active Directory Domain Controller...'. This will open the 'Change Directory Server' form.
  • At the top of the 'Change Directory Form' you'll see the current directory server. In this case, it's 'DC1'. At the bottom of the form there will be a list of all the domain contollers in your domain. It is there where you will select the target domain controller. Here we are selecting 'DC2'.

  • Once you select the domain controller of your choice, click 'OK'
  • Now that we've established the new domain controller that we will transfer the 'Domain Naming Master' role to, we can once again go back to 'Active Directory Domains and Trusts'. Notice that the 'Active Directory Domains and Trusts' is now displaying 'DC2'.

  • Right click on 'Active Directory Domains and Trusts' again.
  • Then select 'Operations Master...'.
  • Notice the current 'Domain Naming Master' holder still remains as 'DC1', this is because we haven't changed that yet, but as you can see at the bottom, the target domain controller is now 'DC2'.
  • Click on the 'Change' button.
  • Once we click on the 'Change' button on the previous form, we will get a message asking if transfering the 'Operations Master' is what we want to do. Since this is what we want to do, click on the 'Yes' button.
  • We will then get a message telling us that the transfer was successful. Proceed by clicking 'OK'.
  • As you can see, the domain controller that now holds the 'Domain Naming Master' role is 'DC2'.

  • Click 'Close'.
