If you have seized an operational master role from a domain controller and later recover the domain controller, Active Directory will need to be removed from the domain controller before it can be added and reused on the domain. This can be done with the following steps...
  • The first thing you need to do before you start up the server, is unplug the network cable. This is very important because if you start up the server with the network cable plugged in, you will essentially have two domain controllers on the network with the same operation master role. This could potentially cause all kinds of problems.

  • After you unplug the network cable from the server, start the server up.
  • Once the server has been started, go to 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Once the command prompt is open, type in the command 'dcpromo', with the switch, ' /forceremoval'. The '/forceremoval' swith will allow 'Active Direcroy' to be removed without the need to contact other domain controllers first. Without this switch, 'Active Directory' would try to contact other domain controllers but would fail becuase of the network cable not being plugged in.
  • Windows will then check if 'Active Directory Services' binaries are installed.
  • You will then get a message telling you that an operation master role is held by this server and that the role should be transferred before removing 'Active Directory'. Since in this case the role has been seized, we can ignore this message and move on.

  • Press the 'Yes' button to continue.
  • Next, since this server was a 'global catalog server', we will get a message reminding us that this domain controller is a 'global catalog server' and should ensure that there is enough 'global catalog servers' in the network before removing this server from 'Active Directory'.

  • Press the 'Yes' button to start the 'Active Directory Domain Services Installation Wizard'.
  • The first screen of the wizard is the 'Welcome Screen'. This screen explains that this wizard will uninstall 'Active Directory Domain Services'.

  • Click on the 'Next' button to continue.
  • This screen of the wizard tells us to consider performing a normal removal of 'Active Directory' before moving on and that any metadata of this server will still remain if decided to continue. Since we've already removed all the metadata and performing of a normal removal of 'Active Directory' is not an option, we can proceed.

  • Press 'Next'.
  • Here we will be asked to choose a local administrator password.

  • Press 'Next', after you typed in the local administrator password and confirmed it.
  • This screen of the wizard expalins that 'Active Directory Domain Services' will be removed from this computer without updating the metadata. But again, we've already cleaned up the metadata, so we can safely move on.

  • Click on the 'Next' button to continue.
  • Once you press 'Next' on the previous screen, Active Directory Domain Services will start uninstalling.
  • The last screen of the wizard explains that 'Active Directory Domain Services' was removed from this computer but that the 'Active Directory Domain Services' binaries will still remain installed.

  • Click 'Finish' to complete the wizard.
  • Once the wizard is complete, click on the 'Restart Now' button to restart the server so that the changes made by the Active Directory Domain Services wizard can take effect.
  • After the server has been rebooted, the 'Server Summary' page will appear showing you that the server was put into a workgroup. Notice that the 'Local Area Connection' is showing as still 'Not connected'.

  • Now that this server is clean from holding any operation master roles, the next step is to plug the network cable back in and reinstall 'Active Directory'.
