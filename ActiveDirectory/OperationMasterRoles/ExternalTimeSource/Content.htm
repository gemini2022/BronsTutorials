Windows Server includes W32Time, the Time Service tool that is required by the Kerberos authentication protocol. The Windows Time service makes sure that all computers in an organization use a common time. To guarantee appropriate common time usage, the Windows Time service uses a hierarchical relationship that controls authority, and the Windows Time service does not allow for loops. In this hierarchy, the PDC operations master at the root of the forest becomes authoritative for the organization. To set up an external time service for you orgainization, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then launch 'Internet Explorer'.
  • Once 'Internet Explorer' is open, type in the address, 'http://support.microsoft.com/kb/262680'.
  • This will bring you to Microsoft's web page. Here, there is a link to another website that has a list of external time servers. Scroll down about half-way down the page...
  • Here you can see a link to 'Stratum Two' time servers. This is the link you want to select. Due to the ever increasing load on 'Stratum One' time servers, they ask that you use 'Stratum Two' servers where possible, as there is more of them and there is generally less load on them.
  • On this website, it is a matter of selecting a time server that is close to you. Scroll down till you find a time server that best matches your location.
  • When you find a time server that is the closest to your location, click on the link to the left. Make sure that the link you have chosen has 'OpenAccess', and not 'RestrictedAccess'. If it is listed as 'RestrictedAccess', you will not be able to use it.
  • Once you click on the time server link of your choice, it will bring you to a page that provides additional information about that time server. The information that you want is the 'Hostname'. Take note of the host name of the time server and then close 'Internet Explorer'.
  • Next, go back down to 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Once you have the command prompt open, run the command, 'w32tm'. This is used to configure the Windows time service.
  • The first parameter you want to add is, '/config'. This indicates that you want to make configuration changes.
  • Next, you will add the time server. This is done with the parameter, '/manualpeerlist:', and then the time server. In this case, the time server we are using is 'ntp4.jimsoffice.org'.
  • The next parameter is '/syncfromflags:manual'. This tells the time server to use the manual entry that has just been put in.
  • The next parameter is '/reliable:yes'. This tells Windows to consider this external time server as a reliable time server.
  • Finally, the last parameter is '/update'. This tells the Windows time server to attempt to update its time from this server right away.

  • Press 'Enter'.
  • Once you launch the command, Windows will start syncing off its external time server and update its time from the server immediately. Or just run w32tm /config /manualpeerlist:"time.windows.com,0x1" /syncfromflags:manual /reliable:yes /update w32tm /config /update
