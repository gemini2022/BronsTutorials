The RID Master (Relative ID) is responsible for processing RID Pool requests from all DCs within a given domain. It is also responsible for moving an object from one domain to another during an interdomain object move. When a DC creates a security principal object such as a user or group, it attaches a unique SID to the object. This SID consists of a domain SID (the same for all SIDs created in a domain) and a relative ID (RID) that is unique for each security principal SID created in a domain. Each DC in a domain is allocated a pool of RIDs that it is allowed to assign to the security principals it creates. When a DC's allocated RID pool falls below a threshold, that DC issues a request for additional RIDs to the domain's RID Master FSMO role owner, the RID Master FSMO role owner responds to the request by retrieving RIDs from the domain's unallocated RID pool and assigns them to the pool of the requesting DC. To view which domain controller is currently holding this master role or to move this master role to another domain controller, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Users and Computers'...
  • Right click on your domain found at the top left side of your screen. In this case, it's 'bronstutorials.local'.
  • Then select the option 'Operations Masters...'.
  • At the top, you can see what domain controller is currently holding the 'RID Master'. At the bottom, we can see what domain controller we want the 'RID Master' to be transfered to. In this case, both the current domain controller and the target domain controller are the same, 'DC1'.
  • Because both the current domain controller and the target domain controller are the same, when the 'Change'button is clicked...
  • You will get a message telling you that in order to transfer the Operations Master, you must first target a different domain controller.
  • To do this, first close the 'Operations Master' form by clicking on the 'Close' button at the bottom of the form.
  • Then go back to 'Active Directory Users and Computers' and right click on your domain found at the top left side of your screen. In this case, it's 'bronstutorials.local'.
  • This time, select the option 'Change Domain Controller...'. This will open the 'Change Directory Server' form.
  • At the top of the 'Change Directory Form' you'll see the current directory server. In this case, it's 'DC1'. At the bottom of the form there will be a list of all the domain contollers in your domain. It is there where you will select the target domain controller. Here we are selecting 'DC2'.

  • Once you select the domain controller of your choice, click 'OK'
  • Now that we've established the new domain controller that we will transfer the 'RID Master' role to, we can once again go back to 'Active Directory Users and Computers'. Notice that the 'Active Directory Users and Computers' is now displaying 'DC2'.

  • Right click on your domain again.
  • Then select 'Operations Masters...'.
  • Notice the current 'RID Master' holder still remains as 'DC1', this is because we haven't changed that yet, but as you can see at the bottom, the target domain controller is now 'DC2'.
  • Click on the 'Change' button.
  • Once we click on the 'Change' button on the previous form, we will get a message asking if transfering the 'Operations Master' is what we want to do. Since this is what we want to do, click on the 'Yes' button.
  • We will then get a message telling us that the transfer was successful. Proceed by clicking 'OK'.
  • As you can see, the domain controller that now holds the 'RID Master' role is 'DC2'.

  • Click 'Close'.
