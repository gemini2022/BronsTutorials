Selective authentication is a security setting that can be enabled on both external trusts and forest trusts. Selective authentication provides Active Directory administrators who manage a trusting forest more control over which groups of users in a trusted forest can access shared resources in the trusting forest. Because creating an external trust or forest trust provides a pathway for all authentication requests between the forests, this increased control is especially important when administrators need to grant access to shared resources in their organization’s forest to a limited set of users in another organization’s forest. To learn how to configure selective authentication, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Users and Computers'...
  • Once in 'Active Directory Users and Computers', go to 'View'.
  • Then select 'Advanced Features'. We are doing this because by default, the option we are after is not be present. Selecting 'Advanced Features' will show a lot of options that are normally hidden.
  • Now, expand your domain...
  • Then select the 'Domain Controllers' OU.
  • Once you've selected the 'Domain Controllers' OU, at the right side of your screen, right click the domain conroller you want to give access to. In this case, we are selecting 'DC1'.
  • Then select 'Properties'.
  • Once in the properties of the domain controller, go to the 'Security' tab. The 'Security' tab would normally not be present, but because of us selecting 'Advanced Features' earlier, it made this option available.
  • Now select the 'Add...' button.
  • Because the forest trust has been added, we can now enter in the administrator from the 'otherguystutorials' domain and 'Windows' will find this user without a problem. By typing 'administrator@otherguystutorials.local' in the space provided, this administrator will be added to permissions listed in this domain controller.

  • Click 'OK'.
  • As you can see, the administrator from the other domain has been added to the domain contoller's permissions list.
  • Make sure the administrator you just added to the list is selected, and then select the option 'Allowed to authenticate'.

  • Click 'OK'.
