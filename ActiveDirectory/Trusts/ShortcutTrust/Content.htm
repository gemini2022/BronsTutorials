Shortcut trusts are one-way or two-way, transitive trusts that can be used when administrators need to optimize the authentication process. Authentication requests must first travel a trust path between domain trees, and in a complex forest this can take time, which can be reduced with shortcut trusts. A trust path is the series of domain trust relationships that must be traversed in order to pass authentication requests between any two domains. To learn how to make a shorcut trust, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Domains and Trusts'...
  • Once in 'Active Directory Domains and Trusts', expand down to the domain where you want to create the shortcut trust from. For example, in this case we are going to first expand the root domain 'bronstutorials.local'.
  • In this example, after 'bronstutorials.local' has been expanded, you'll notice two child domains, 'east' and 'west'. We'll continue by expanding the 'west' child domain.
  • Now you'll notice that the 'west' domain has its own child domain called 'sales'. In this example, if a resource was to be accessed in the 'east' domain from the 'sales' domain, the request would have to first go to the 'west' domain and then to the 'east' domain. But, instead we are going to make a shortcut from the 'sales' domain to the 'east' domain.
  • To do this, first right click on the 'sales.west.bronstutorials.local' domain.
  • Then select 'Properties'.
  • Once in the properties of the 'sales' domain, select the 'Trusts' tab.
  • Here in the Trusts tab, the top portion of this screen displays all the domains that the outgoing trust is pointing to. This means any domain that the 'sales' domain is pointing to, it trusts. In this case, the 'sales' domain trusts the 'west' domain.

  • The bottom portion of this screen displays all the domains that are pointing to the 'sales' domain. This means any domain that points to the 'sales' domain trusts the 'sales' domain. In this case, the 'west' domain trusts the 'sales' domain.

  • Press the 'New Trust...' button to launch the 'New Trust Wizard'.

 
  • The first screen of the wizard is the 'Welcome Screen'.

  • Click on the 'Next' button to continue
  • In this screen of the wizard you are asked to type in the domain you want to make a shortcut trust to. In this case, we are going to type 'east.bronstutorials.local'.

  • After you type in the domain you want to make a shortcut trust to, press 'Next'.
  • Here on this screen of the wizard you are to choose the direction of the shortcut trust. In this example, we are going to choose the option of 'Two-way', which will allow both the specified domain which we specified in the last screen and the 'sales' domain to trust each other.

  • Press 'Next' to continue.
  • On this screen, you are asked to decide which side of the trust you want to create. Since we belong to 'sales' domain, creating an 'outgoing' trust can easily be done. However, in order to create an 'incoming' trust from the specified domain, you need to have the appropriate permissions from that domain to do so. If not, someone in the specified domain will have to create the other side of the trust. In this case, we have the appropriate permissions from the specified domain, therefore we can choose the option 'Both this domain and the specified domain'.

  • Click 'Next' to continue.
  • Since we chose the option to create both sides of the trust in the last screen, we are now asked to provide a user name and password of the specified domain that has the appropriate administrative privileges.

  • Once you enter in the user name and password of the specified domain that has the appropriate administrative privileges, click 'Next' to continue.
  • This screen of the wizard gives you an overview of the settings that you have selected. Notice that 'Windows' has detected the type of trust you are going to create.

  • Click 'Next' to create the trusts.
  • Provided things went right, you should arrive at this screen telling you that the trust was created successfully.

  • Click 'Next' to continue.
  • In this screen of the wizard, you are asked to confirm that the 'outgoing' trust has been created. Since we have created the 'outgoing' trust, we can confirm it by selecting the option, 'Yes, confirm the outgoing trust'.

  • Click 'Next' to continue.
  • In this screen of the wizard, you are asked to confirm that the 'incoming' trust has been created. Since we have created the 'incoming' trust, we can confirm it by selecting the option, 'Yes, confirm the incoming trust'.

  • Click 'Next' to continue.
  • The last screen of the wizard is the 'Finish' screen. Click on the 'Finish' button to compelte the wizard.
  • Once the wizard is complete, notice that the new shortcut trust has been created in both the 'outgoing' and 'incoming' sections.

  • Press, 'OK'.
