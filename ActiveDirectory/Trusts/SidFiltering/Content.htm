SID filtering causes the domain controllers in a trusting domain to remove all SIDs that aren't members of the trusted domain. In other words, if a user in a trusted domain is a member of groups in other domains in the forest, the trusting domain will remove those groups' SIDs from the user's access token. This is to prevent elevation-of-privilege attacks. SID filtering is enabled by default, at least since sever 2003. You would want to disable SID filtering (temporarily) if you have a situation where you need to break and reestablish trusts. To learn how to turn SID filtering on or off, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Once the command prompt is open, type in the command 'netdom trust'.
  • Now type in the 'trusting' domain. In this case its 'bronstutorials.local'.
  • Then type the parameter '/domain:', followed by the 'trusted' domain.
  • In this case the 'trusted' domain is 'nt4domain.local'.
  • For the next parameter, type '/quarantine:'. This parameter will determine whether SID filtering will be turned on or off.
  • In this case, we'll be turning SID filtering off, so we'll type 'no' for the value. If we were turning SID filtering on, then the value we would type would be 'yes'.
  • The next parameter will specify the administrator account of either the 'trusting' domain or the 'trusted' domain.

  • If the administrator account is to be from the 'trusting' domain, then type '/usero:', which is what we are using in this case.

  • If the administrator account is to be from the 'trusted' domain, then type '/userd:'.
  • Only a member of the Administrators group or the Domain Admins or Enterprise Admins group has the privileges to turn on or off SID filtering. In this case we will be using the 'administrator' account.
  • The next parameter is for the password of the account you specified earlier.

  • Type either '/passwordo:' if the administrator account you specified is from the 'trusting' domain, which is what we are using in this case.

  • Or, type '/passwordd:' if the administrator account you specified is from the 'trusted' domain.
  • Then type the password. In this case, we are using the password 'Pa$$w0rd1'. If you didn't want to reveal the password, you could always type '*' instead. This would prompt you for the password which allows you to type the password secretly.

  • Then press 'Enter'.
  • After you press 'Enter', the command should either turn on filtering or turn filtering off depending on the parameters you have set.
