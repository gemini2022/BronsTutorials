A forest trust allows administrators to federate two Active Directory forests with a single trust relationship to provide a seamless authentication and authorization experience across the forests. A forest trust can be created only between a forest root domain in one forest and a forest root domain in another forest. Forest trusts can be created between two forests only and cannot be implicitly extended to a third forest. This means that if a forest trust is created between Forest 1 and Forest 2, and another forest trust is created between Forest 2 and Forest 3, Forest 1 does not have an implicit trust with Forest 3. To learn how to make a forest trust, proceed with the following steps...
  • Before you create a forest trust, there are name resolution requirements that must be met so that you can establish the trust successfully. You can use DNS to resolve names for a forest for which you are attempting to establish a trust. There are two common DNS configurations that you can choose from when you establish DNS name resolution between forests, they are conditional forwarders and secondary zones with zone transfers enabled. Stub zones can be used to resolve DNS names, but are less commonly used than secondary zones and conditional forwarders.

  • Create either a conditional forwarder or a secondary zone with zone transfers enabled and then verify that name resolution can occur.
  • After name resolution has been configured and tested for connectivity between the two domains, begin creating your forest trust by first selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Domains and Trusts'...
  • Once in 'Active Directory Domains and Trusts', select and right click your root domain. In this case our root domain is, 'bronstutorials.local'.
  • Then select 'Properties'.
  • Once in the properties of your root domain, select the 'Trusts' tab.
  • Here in the Trusts tab, the top portion of this screen displays all the domains that the outgoing trust is pointing to. This means any domain that the 'bronstutorials' domain is pointing to, it trusts. The 'bronstutorials' domain has two child domains, so in this case the 'bronstutorials' domain trusts its child domains, 'east' and 'west'.

  • The bottom portion of this screen displays all the domains that are pointing to the 'bronstutorials' domain. This means any domain that points to the 'bronstutorials' domain trusts the 'bronstutorials' domain. In this case, the 'east' and 'west' domains trusts the 'bronstutorials' domain.

  • Press the 'New Trust...' button to launch the 'New Trust Wizard'.
  • The first screen of the wizard is the 'Welcome Screen'.

  • Click on the 'Next' button to continue.
  • In this screen of the wizard you are asked to type in the domain you want to make a forest trust to. In this case, we are going to type 'otherguystutorials.local'.

  • After you type in the domain you want to make a forest trust to, press 'Next'.
  • The next screen allows you to make a choice between two types of trust relationships. The first option is an 'External' trust, which is a nontransitive trust between the two domains, so only the two domains selected will be able to use this trust. No child of either domain would be part of this trust relationship. This trust is also the trust you would need to use for NT4 domains.

  • The second option is a 'Forest' trust, which is the trust we will select. This trust is a transitive trust and will allow any domain in either forest to access any other domain in the other forest.

  • Click on the 'Next' button to continue.

 
  • On this next screen you can select which direction the trust will go, 'One-way: incoming', 'One-way: outgoing' or choose the default option, 'Two-way'. In this case, both forests will be sharing resources with each other so we will accept the default option of, 'Two way'.

  • Click on the 'Next' button to continue.
  • The next screen gives you the option to create the trust in this domain or both domains. In a lot of cases, you won't have an account in the other forest. This is especially the case if the other foest is another company. Unfortunately, we do not have creation privileges for the other domain, so we will select the option, 'This domain only'. Because we are only creating our side of the trust, the other side of the trust relationship will have to be created by the other forest.

  • Click on the 'Next' button to continue.
  • The next screen allows you to determine how the default authentication will work. The first option, 'Forest-wide authentication' automatically authenticates users from the other forest for all resources in this forest. In other words, this means that users will still have to have been granted access to resources, but users will be able to access any resource where the permissions are set to 'Everyone' or be able to authenticate off of any domain controller in the forest.

  • The second option, 'Selective authentication' will only allow users to access servers that you decide they can access. This is a good option to choose if the other forest is another company. In our case, since the other forest is another company, we will select this option, 'Selective authentication'.

  • Press 'Next' to continue.
  • On this screen, you get to enter in a password for the trust. This password will need to be entered in on the other side using the same wizard as here. When the two forests connect up for the first time, this password is used to ensure that both servers are talking to the correct party.

  • After you typed and confirmed a password that both sides agreed on, click 'Next' to continue.
  • This screen of the wizard gives you an overview of the settings that you have selected. Notice the type of trust you are going to create.

  • Click 'Next' to create the trust.
  • On this screen of the wizard, you will be informed that the forest trust was created.

  • Press 'Next'.
  • STOP! Before you can perform the next step, you need to ensure that an administrator in the other forest has also completed this wizard up to this point. Once you have confirmed that an administrator from the other forest has created their side of the trust, you may proceed...
  • This screen will ask you if you want to confirm the 'outgoing' trust. Since your side of the trust has been created and as long as you have confirmed from an administrator from the other forest that their side of the trust has been created, then you may select the option, 'Yes, confirm the outgoing trust'.

  • Click 'Next' to continue.
  • In the next screen, the 'incoming' trust can also be confirmed. In this case you will be required to enter in a user name and password from the other forest. In a lot of cases you may not have this, especially if you are connecting to another company. Since we are connecting to another comapny, and we do not have user name and password information of an account with administrative privileges from the other forest, we will select the option, 'No, do not confirm the incoming trust'.

  • Click 'Next' to continue.
  • The last screen of the wizard is the 'Finish' screen. Click on the 'Finish' button to compelte the wizard.
  • Once the wizard is complete, notice that the new forest trust has been created in both the 'outgoing' and 'incoming' sections.

  • Press, 'OK'.
