A realm trust can be established between any non-Windows Kerberos V5 realm and a Windows domain. This trust relationship allows cross-platform interoperability with security services based on other Kerberos V5 versions such as UNIX and MIT implementations. Realm trusts can switch from nontransitive to transitive and back. Realm trusts can also be either one-way or two-way. To learn how to make a realm trust, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Domains and Trusts'...
  • Once in 'Active Directory Domains and Trusts', select and right click the domain you want to make the realm trust with. In this case we're going to use, 'bronstutorials.local'.
  • Then select 'Properties'.
  • Once in the properties of the domain, select the 'Trusts' tab.
  • To begin creating a realm trust, press the 'New Trust...' button to launch the 'New Trust Wizard'.
  • The first screen of the wizard is the 'Welcome Screen'.

  • Click on the 'Next' button to continue.
  • In this screen of the wizard you are asked to type in the domain you want to make a realm trust to. In this case, we are going to type 'mykerberosrealm.local'.

  • After you type in the domain you want to make a realm trust to, press 'Next'.
  • Select the option, 'Realm trust'.

  • Then press 'Next'.
  • The next screen allows you to make a choice between two types of trusts. The first option is a 'Nontransitive' trust, which is a trust between only two domains, so only the two domains selected will be able to use this trust. No other domain of either realm would be part of this trust relationship.

  • The second option is a 'Transitive' trust, which is the option we will select. This type of trust and will allow any domain in either realm to access any other domain in the other realm.

  • Click on the 'Next' button to continue.
  • On this next screen you can select which direction the trust will go, 'One-way: incoming', 'One-way: outgoing' or choose the default option, 'Two-way'. In this case, both realms will be sharing resources with each other so we will accept the default option of, 'Two way'.

  • Click on the 'Next' button to continue.
  • On this screen, you get to enter in a password for the trust. This same password will need to be entered in when the other side of the trust gets created.

  • After you typed and confirmed a password that both sides agreed on, click 'Next' to continue.

 
  • This screen of the wizard gives you an overview of the settings that you have selected. Notice the type of trust you are going to create.

  • Click 'Next' to create the trust.
  • The last screen of the wizard is the 'Finish' screen. Click on the 'Finish' button to compelte the wizard.
  • Once the wizard is complete, notice that the new realm trust has been created in both the 'outgoing' and 'incoming' sections.

  • Press, 'OK'.
