You can create an external trust to form a one-way or two-way, nontransitive trust with domains outside of your forest. External trusts are sometimes necessary when users need access to resources located in a Windows NT 4.0 domain or in a domain located within a separate forest that is not joined by a forest trust. To learn how to configure an external trust, proceed with the following steps...
  • Before you create an external trust, there are name resolution requirements that must be met so that you can establish the trust successfully. You can use DNS to resolve names for an external domain for which you are attempting to establish a trust. There are two common DNS configurations that you can choose from when you establish DNS name resolution between external domains, they are conditional forwarders and secondary zones with zone transfers enabled. Stub zones can be used to resolve DNS names, but are less commonly used than secondary zones and conditional forwarders.

  • Create either a conditional forwarder or a secondary zone with zone transfers enabled and then verify that name resolution can occur.
  • After name resolution has been configured and tested for connectivity between the two domains, begin creating your external trust by first selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Domains and Trusts'...
  • Once in 'Active Directory Domains and Trusts', select and right click your root domain. In this case our root domain is, 'bronstutorials.local'.
  • Then select 'Properties'.
  • Once in the properties of your root domain, select the 'Trusts' tab.
  • To begin creating a external trust, press the 'New Trust...' button to launch the 'New Trust Wizard'.
  • The first screen of the wizard is the 'Welcome Screen'.

  • Click on the 'Next' button to continue.
  • In this screen of the wizard you are asked to type in the domain you want to make a external trust to. In this case, we are going to type 'nt4domain.local'.

  • After you type in the domain you want to make a external trust to, press 'Next'.
  • Select the option, 'External trust'.

  • Then press 'Next'.

 
  • On this next screen you can select which direction the trust will go, 'One-way: incoming', 'One-way: outgoing' or choose the default option, 'Two-way'. In this case, both 'bronstutorials' and the 'nt4domain' will be sharing resources with each other so we will accept the default option of, 'Two way'.

  • Click on the 'Next' button to continue.
  • The next screen gives you the option to create the trust in this domain or both domains. In a lot of cases, you won't have an account in the external domain. This is especially the case if the other external domain is another company. Unfortunately, we do not have creation privileges for the external domain, so we will select the option, 'This domain only'. Because we are only creating our side of the trust, the other side of the trust relationship will have to be created by the other domain.

  • Click on the 'Next' button to continue.
  • The next screen allows you to determine how the default authentication will work. The first option, 'Domain-wide authentication' automatically authenticates users from the other domain for all resources in this domain. In other words, this means that users will still have to have been granted access to resources, but users will be able to access any resource where the permissions are set to 'Everyone' or be able to authenticate off of any domain controller in either domain.

  • The second option, 'Selective authentication' will only allow users to access servers that you decide they can access. This is a good option to choose if the other domain is another company. In our case, since the other domain is another company, we will select this option, 'Selective authentication'.

  • Press 'Next' to continue.
  • On this screen, you get to enter in a password for the trust. This password will need to be entered in on the other side using the same wizard as here.

  • After you typed and confirmed a password that both sides agreed on, click 'Next' to continue.
  • This screen of the wizard gives you an overview of the settings that you have selected. Notice the type of trust you are going to create.

  • Click 'Next' to create the trust.
  • On this screen of the wizard, you will be informed that the external trust was created.

  • Press 'Next'.
  • STOP! Before you can perform the next step, you need to ensure that an administrator in the other domain has also completed this wizard up to this point. Once you have confirmed that an administrator from the other domain has created their side of the trust, you may proceed...
  • This screen will ask you if you want to confirm the 'outgoing' trust. Since your side of the trust has been created and as long as you have confirmed from an administrator from the other domain that their side of the trust has been created, then you may select the option, 'Yes, confirm the outgoing trust'.

  • Click 'Next' to continue.
  • In the next screen, the 'incoming' trust can also be confirmed. In this case you will be required to enter in a user name and password from the other domain. In a lot of cases you may not have this, especially if you are connecting to another company. Since we are connecting to another comapny, and we do not have user name and password information of an account with administrative privileges from the other domain, we will select the option, 'No, do not confirm the incoming trust'.

  • Click 'Next' to continue.
  • The last screen of the wizard is the 'Finish' screen. Click on the 'Finish' button to compelte the wizard.
  • Once the wizard is complete, you'll get a message telling you that (SID) filtering is enabled and that you have the option to disable it if you wish.

  • Press, 'OK'.
  • Now that the wizard is complete, notice that the new external trust has been created in both the 'outgoing' and 'incoming' sections and that their transitive type is set to 'No'.

  • Press, 'OK'.
