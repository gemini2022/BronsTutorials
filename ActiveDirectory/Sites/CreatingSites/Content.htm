Active Directory gives system administrators the option to define sites, consisting of one or more subnets. While it may not be necessary to define sites in all AD deployments, they can come in useful if you have geographically dispersed offices that are separated by a wide area network, or if you have slow network links between offices and need to control AD replication. Client PCs will also be able to find a domain controller that is physically close to them for logon. And other applications that rely on AD, such as Exchange Server, can use site topology information to locate services that are close by. To learn how to create and configure sites, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Sites and Services'...
  • Once in 'Active Directory Sites and Services', expand 'Sites'.
  • Then expand, 'Default-First-Site-Name'.
  • Then finally expand 'Servers'.
  • When you first install 'Active Directory', the 'Default-First-Site-Name' is created so that there is somewhere to store site related items. In this default site, you will see three domain controllers that have already been installed, one in New York, one in Washington, and one in London. You are free to create your own sites and leave the default site as is, however, most administrators will rename the default site like this...
  • Right click 'Default-First-Site-Name'.
  • Then select 'Rename'.
  • In this case, we will change the defalut site to 'New York'.

 
  • The next step is to create a new site for Washington and London.
  • To do this, right click 'Sites'.
  • And select 'New Site...'.
  • When the 'New Object - Site' form appears, start by entering in the name of the site in the input field labeled, 'Name:'. In this case, we'll name this site 'Washington'.
  • Then select a site link. At this point there is only one option, which is the default site link. Like the default site, a default site link is created when 'Active Directory' is first installed. Later we will learn how to create our own site links, but for now, select 'DEFAULTIPSITELINK'.

  • Then press 'OK'.
  • Once the site has been created 'Windows' will give you an information dialog reminding you that further configuration is required, like assigning a subnet to the site.

  • Press 'OK'.
  • Once we press 'OK' on the previous form, we will then follow the same procedure to create a new site for London.
  • Again, right click 'Sites'.
  • Select 'New Site...'.
  • Type 'London' in the input field.
  • Then select the default site link for this site as well.

  • Press, 'OK'.
  • Now we have sites for 'New York', 'Washington', and 'London'.
