The Knowledge Consistency Checker (KCC) generates the Active Directory replication topology on every domain controller every 15 minutes by default. You can force the KCC to run on any domain controller, but the topology that is generated depends on the domain controller on which you run the command. To generate the intersite replication topology, force the kcc to run on the domain controller in the site that holds the ISTG role. To generate the intrasite replication topology, force the kcc to run on any domain controller in the site that does not hold the ISTG role. To learn how to force the kcc to run, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Sites and Services'...
  • Once in 'Active Directory Sites and Services', expand 'Sites'.
  • Then expand the 'NewYork' site.
  • Then expand the 'Servers' container.
  • Once in the 'Servers' container, expand one of the domain controllers. In this case, we're going to expand the 'NYDC1' domain controller.
  • Then right-click on 'NTDS Settings'.
  • And select 'All Tasks'.
  • Then select 'Check Replication Topology'.

 
  • This option will force the 'Knowledge Consistency Checker' to run. Windows will give you a confirmation dialog telling you that the 'Knowledge Consistency Checker' was run and that you may need to refresh the screen.

  • Press the 'OK' button.
