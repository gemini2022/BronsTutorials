Networks usually consist of a set of LANs that are connected by WANs. In 'Active Directory', site link objects represent the WAN connections between sites. 'Active Directory' creates a default site link called 'DEFAULTIPSITELINK'. You are free to rename the defalut site link, or create a new site link and remove the default site link, or simply leave the default site link in 'Active Directory' unused. Because replication within a site is triggered automatically when a directory update occurs, replication between sites is scheduled to occur every 3 hours. You can change the default schedule to occur during the periods that you specify, and at the intervals that you specify, so that you can control WAN link traffic. To learn how to create a site link, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Sites and Services'...
  • Once in 'Active Directory Sites and Services', expand 'Sites'.
  • Then expand, 'Inter-Site Transports'.
  • Once you exapand 'Inter-Site Transports' you will see that there are two transports available. 'IP' and 'SMTP'. The 'SMTP' transport is used for old technology, so we will go ahead and click on the 'IP' transport.
  • Once you click on the 'IP' transport, you will notice that on the right hand side of your screen there is already a site link called 'DEFAULTIPSITELINK'. This is created by default by 'Active Directory' when it is first installed. If you never create any additional site links in 'Active Directory', all of your sites that you create will be linked to each other using the default site link. You are free to rename the defalut site link, or create a new site link and remove the default site link, or simply leave the default site link in 'Active Directory' unused.
  • In this case, we are going to rename the default site link. To do this, right click on 'DEFAULTIPSITELINK'.
  • Then select 'Rename'.
  • In this case, we're going to rename the default site link to 'NewYorkToWashington'.

 
  • Once renaming is complete, we need to configure our newly named site link. To do this, right click on the site that is now named 'NewYorkToWashington'.
  • And select 'Properties'.
  • Once in the properties of the site link, remain on the 'General' tab.
  • Since this site link was originally the default site link, all three sites are listed here. We only want the 'NewYork' and 'Washington' site to be in this site link, so we will remove the London site from the list. To do this, select the 'London' site.

  • Then press the '<< Remove' button.
  • Now you can see that 'London' is under the 'Sites not in this site link:' section, and 'NewYork' and 'Washington' are under the 'Sites in this site link:' section.
  • Now we will also change the replication cost to '40' so that this site link is favored over any other site link that has a higher cost value assigned to it.
  • And we will change the replication interval to '15' minutes, the lowest it can go. This means that replication will occur every 15 minutes on this site link.
  • At the bottom of the form, you'll notice a button labeled 'Change Schedule...'. When this button is pressed...
  • A form will appear that allows you to select the times when replication will occur. However, you can also configure replication to be disabled at certain times as well.
  • For example, you could disable replication if you wanted to for Monday through Friday during business hours. To do this, select an area where you want to disable replication.
  • Then select the option 'Replication Not Available'.

  • Then press 'OK'.

 
  • We are now finished configuring this site link, so we can press 'OK' to close this form.
  • We now have to create two more site links, one for New York to London and another one for Washington to London. To do this we going to go back to the left side of our screen and right click on the 'IP' transport.
  • Then select 'New Site Link...'.
  • Here in the top section fo the form is where you give the new site link a name. In this case, we are going to name it 'NewYorkToLondon'.
  • Then select the two sites that will be used for this site link. In this case we are going to select 'London' and 'NewYork'.

  • Once the sites are selected, press the 'Add >>' button.
  • As you can see, the two sites that will be used for this site link is 'London' and 'NewYork'.

  • Now press 'OK' to complete the creation of this site link.
  • Now that the 'NewYorkToLondon' site link is created, go back to the right side of the screen. Here we want to now configure the cost for the 'NewYorkToLondon' site to '40', just like the 'NewYorkToWashington' site link. This is so that this site will also get preference over any site link that has a higher cost value.
  • To do this, right click on the 'NewYorkToLondon' site link.
  • And select 'Properties'.
  • And set its cost to '40'.

  • Then press 'OK'.
  • Finally, we need to create the site link for Washington to London. To do this, we need to go back to the left side of our screen and right click again on the 'IP' transport.
  • Then once again select 'New Site Link...'.
  • Name the site link 'WashingtonToLondon'.
  • Then select the 'London' and 'Washington' site and press 'Add >>'.
  • As you can see, the two sites that will be used for this site link is 'London' and 'Washington'.

  • Now press 'OK' to complete the creation of this site link.
  • Now we have site links for all the sites. We will leave the cost for the 'WashingtonToLondon' site link as '100'. This will ensure that the 'WashingtonToLondon' site link will only be used if one of the other site links were to go down.
