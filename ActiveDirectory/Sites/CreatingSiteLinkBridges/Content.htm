A site link bridge connects two or more site links. A site link bridge enables transitivity between site links. Each site link in a bridge must have a site in common with another site link in the bridge. By default, all site links are transitive and it is recommended to keep transitivity enabled by not changing the default value of "Bridge all site links" (enabled by default). However, you may need to disable "Bridge all site links" and create a site link bridge if the IP network is not fully routed because of the networks separated by firewalls or when you need to control the replication flow in Active Directory yourself. To learn how to create a site link bridge, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Sites and Services'...
  • Once in 'Active Directory Sites and Services', expand 'Sites'.
  • Then expand, 'Inter-Site Transports'.
  • Once you exapand 'Inter-Site Transports' you will see that there are two transports available. 'IP' and 'SMTP'. The 'SMTP' transport is used for old technology, so we will go ahead and right click on the 'IP' transport.
  • Then select 'Properties'.
  • When you first open the properties for the 'IP' transport, you will notice an option, 'Ignore schedules'. Selecting this option will disable any schedules you configured, but we will leave this option unchecked.

  • For the second option, 'Bridge all site links' or (BASL), we want this option unchecked. In order for any site link bridge that is manually created to work, we first have to have this option disabled. By disabling this option 'Bridge all site links' or (BASL), we are essentially preventing a connection to be made between two sites that are are not part of the same site link. To put it in another way, we're making all the site link connections non-transitive. This option is selected by default. To disable this option, deselect it and leave it unchecked.

  • Press 'OK'.
  • Now we want to go back and right click again on the 'IP' transport.
  • This time select 'New Site Link Bridge...'.

 
  • The first thing you'll want to do is name your new site link bridge. In this case, we're naming it 'Bridge Link'.
  • Then select the site links you want to be in this site link bridge. In this case we're selecting 'NewYorkToLondon' and 'NewYorkToWashington'.

  • Then press the 'Add >>' button.
  • Once you press the 'Add >>' button, then press the 'OK' button to create the site link bridge.
  • If you look to the right side of your screen, you'll notice the site link bridge that you created is within the list of all the site links we created earlier.
