If your sites and subnets are configured, new servers automatically get added to the site that owns that subnet. However, you can manually move a server to a different site. To learn how to move servers to a site, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Sites and Services'...
  • Once in 'Active Directory Sites and Services', expand 'Sites'.
  • Then, if we expand the 'NewYork' site...
  • Then expand 'Servers'.
  • We can see that all the domain controllers are within the 'NewYork' site. However, what we really want is the London domain controller to be in the 'London' site and the Washington domain controller to be in the 'Washington site'.
  • To do this, we can simply first right click on a server. In this case we'll right click on 'LONDC1'.
  • Then select 'Move...'.
  • Then select the site where you want to move the server to. In this case, we want to move the server to the 'London' site.

  • Then press 'OK'.

 
  • As you can see the London server is no longer in the 'NewYork' site. So if you expand the 'London' site...
  • Then expand the 'Servers' container under the 'London' site.
  • You can see that the London server has been moved to the 'London' site.
  • Now we want to move the Washington server to the 'Washington' site, but this time we are going to drag and drop the server to its intended site instead.

  • To do this, first expand the 'Washington' site.
  • Then select the server you want to drag. In this case we are going to select 'WASHDC1'.
  • Then we'll drag and drop the Washington server into the 'Servers' container under the 'Washington' site.
  • When moving the server, you will get a dialog box reminding you that moving a domain controller from site to site may change the settings that are applied to that server in Group Policy.

  • To avoid seeing this message again, select the 'Don't show this warning while this snap-in is open' option.

  • Then press the 'Yes' button to continue.
  • Now expand the 'Servers' container under the 'Washington' site.
  • Now you can see that all the domain controllers are located under their proper site.
