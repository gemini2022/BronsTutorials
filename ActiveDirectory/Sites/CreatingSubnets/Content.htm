The subnet objects in Active Directory are the logical representation of the subnets in your physical network environment. Subnet information is used for Active Directory replication to determine the best routes to domain controllers between sites. Subnets must be defined in Active Directory to ensure accurate and efficient directory replication and resource usage. To learn how to create subnets, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Sites and Services'...
  • Once in 'Active Directory Sites and Services', expand 'Sites'.
  • Then right click on 'Subnets'.
  • And select 'New Subnet'.
  • On this form, you then need to enter in the network address and the number of fixed bits in the subnet mask of one of the sites you created earlier. In this case, we are going to define the network address of the 'NewYork' site. The 'NewYork' site has an address of '192.168.0.0', and has a 24 bit subnet mask, so we will enter '192.168.0.0/24'.
  • Once you have entered in the network address information, we can then select the site this subnet will be associated with. In this case it's 'NewYork'.

  • Then press 'OK'.
  • Now that we've created a subnet for New York, we then need to make a subnet for Wahsington and London.
  • So we're going to again select 'Subnets'.

 
  • And again select 'New Subnet'.
  • Now enter in the network address and the number of fixed bits in the subnet mask of the Washington site. In this case it's '192.168.1.0/24'.

  • Then select the 'Washington' site.

  • Press 'OK' when finished.
  • Finally we need to create a subnet for London.
  • So go back and right click on 'Subnets' one last time.
  • And once again select 'New Subnet'.
  • Now enter in the network address and the number of fixed bits in the subnet mask of the London site. In this case it's '192.168.2.0/24'.

  • Then select the 'London' site.

  • Press 'OK' when finished.
  • Now we have all the subnets created for our sites.
