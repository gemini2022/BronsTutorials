In the 'Active Directory', you can right-click the computer object in the Computers OU, or any appropriate OU and then click Reset Account. This resets the machine account. Resetting the password for domain controllers using this method is not allowed. Resetting a computer account breaks that computer's connection to the domain and requires it to rejoin the domain. You will most likely have to perform this procecure if the password of the computer account in the domain and the password on the local computer become out of sync. To learn how to reset a computer account, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Users and Computers'...
  • Once in 'Active Directory Users and Computers', expand your domain, in this case it's 'bronstutorials.local'.
  • And then click on the 'Computers' OU.
  • You will see that on the right side of the 'Active Directory' window will be a list of the computer accounts that were created in this OU. Right-click on the computer account you want to reset, in this case, we're going to reset the 'WS3' computer account.
  • Once you right-click on the computer account you want to reset, select the 'Reset Account' option.
  • After you select the 'Reset Account' option, a prompt will appear asking if you are sure you want to reset this computer account.

  • Press the 'Yes' button to reset the computer account.
  • After you press the 'Yes' button on the previous prompt, a second prompt will appear telling you that the reset was sucessful.

  • Press 'OK'.
  • If we now change to our Windows client computer and attempt to log in using a domain account...

 
  • Notice that we get a message stating, 'The trust relationship between this workstation and the primary domain failed'. We get this message because we reset this computer's domain account. This is also the same message you will receive if the password of the computer account in the domain and the password on the local computer become out of sync.

  • Press 'OK'.
  • Because we received the previous message by resetting this computer's domain account, we can fix this by removing this computer from the domain and then adding it back again. By performing this procedure, we are essentially reassigning a new password to this computer for the computer account of this domain. Once done, we then should be able to succesfully log into the domain using a domain account. To do this, we first need to log into the computer using the local administrator account.

 
  • Once logged in, select the 'Start' or the 'Start Orb' button at the bottom left corner of your screen.
  • And then right-click 'Computer'.
  • And select 'Properties'.
  • Once in the properties of the computer, go to the far right of the window and click on 'Change settings'.
  • From here, you'll want to click on the 'Change...' button.
  • Then in the 'Member of' section, select the option 'Workgroup'.
  • Then type in a suitable workgroup name. This is the part where we are removing this computer from the domain.

  • Press 'OK'.
  • Windows will then remind you that once the computer is taken out of the domain, you will need to know a local administrator account on the computer to access the computer.

  • Press 'OK' to continue.
  • Once you press 'OK' on the previous prompt, you will then be asked to enter in the name and password of an account with permissions to remove this computer from the domain. In this case, we're going to use the 'administrator' account.
  • Then type in the password. Normally when you perform this step of removing a computer from the domain, the computer account for this computer would be removed from 'Active Directory'. Since we reset the computer account, the computer account for this computer will not be removed from 'Active Directory'.

  • Click 'OK' when finished.
  • You will then get a dialog welcoming you to the workgroup you just defined.

  • Press 'OK'.
  • After the 'welcome' dialog, you will receive another dialog informing you that in order for the changes to take effect, you'll need to restart your computer. Since we are going to put the computer straight back into the domain, we do not need to restart the computer at this time.

  • Press 'OK'.
  • Once again, press 'Change'.
  • Then in the 'Member of' section, select the option 'Domain'.
  • And then add the computer back into the domain. The computer will be added back to the domain using the computer account that is currently in 'Active Directory'.

  • Press 'OK'
  • Again you will be asked to enter in the name and password of an account with permissions to join the computer to the domain. Again, we will use the 'administrator' account.
  • And then we'll type in the password for this account.

  • Click 'OK' when finished.
  • Once again, you will receive a dialog welcoming you back to the domain.

  • Press 'OK'
  • Like before, after the 'welcome' dialog, you will receive another dialog informing you that in order for the changes to take effect, you'll need to restart your computer.

  • Press 'OK'.
  • Then press the 'Close' button on the 'System Properties' form.
  • And then finally, press the 'Restart Now' button to restart your computer.
