Offline domain join is a new process that computers that run Windows 7 or Windows Server 2008 R2 can use to join a domain without contacting a domain controller. This makes it possible to join computers to a domain in locations where there is no connectivity to a corporate network. To learn how to join a computer to the domain using offline domain join, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Once the command prompt is open, run the following command, 'djoin'.
  • Then add the parameter, '/Provision'. This parameter indicates that a new computer account is to be created in 'Active Directory'. This computer account needs to be present in 'Active Directory' before offline domain join can be used to join the computer to the domain.
  • The next parameter is '/Domain', followed by the domain name the computer account is going to be created in. In this case, the computer account is going to be created in the 'bronstutorials.local' domain.
  • Next, is the '/Machine' parameter, followed by the computer name for the computer that is to be added to the domain. In this case, the computer name is 'WS3'.
  • And then finally the last parameter is, '/SaveFile', which allows a file name to be given where the offline domain join data will be saved.

  • Once you finish typing this last parameter, press 'Enter' to run the command.
  • Once the command has been run, the computer account for 'WS3' will be created in 'Active Directory' along with a file with the name and location you specified in the last parameter.
  • If you go back to 'Start' at the bottom left corner of your screen.
  • And then click on 'Administrative Tools'.

 
  • And open 'Active Directory Users and Computers'.
  • And then expand your domain.
  • And click on the 'Computers' OU.
  • You will see that on the right side of the 'Active Directory' window will be the computer account you just created.
  • Now, log into a Windows client computer. For this example, the client computer we are logging into has no network cards installed, therefore this computer is not connected to any network. This computer could not normally be joined to a domain until a network card has been installed and configured. To add this computer to the domain, select 'Start' or the 'Start Orb' at the bottom left corner of your screen.
  • And type 'cmd' in the search field.
  • Right-click on the command prompt icon that appears at the top.
  • And select 'Run as administrator'.
  • When the command prompt is open, once again type 'Djoin'.
  • The first parameter will be '/RequestODJ', rather than 'provision' like we used last time.
  • Next is the parameter '/LoadFile', followed by the file name of the file saved previously. This saved file that you created earlier will have to be moved from the domain controller you created it on to this computer. Since this computer is not on a network, you'll have to transfer the file by some removable media device. In this case, we transferred the file and placed in the root of the 'C' drive.
  • The next parameter is '/WindowsPath', followd by the directory where windows is located. You can enter in a path like, 'C:\Windows', but since we are logged into the computer we are changing, we can enter in, %windir%. This is an enviroment variable, which will substitute the current Windows directory.
  • The final parameter is, '/LocalOS'. This tells 'Djoin' that the computer to be joined to the domain is the local computer.

  • Once you finish typing this last parameter, press 'Enter' to run the command.
  • After the command has completed, you'll have to restart the computer, but this computer is now part of the domain.
