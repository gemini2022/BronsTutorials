With the 'Profile' tab, you can determine where 'Windows' will store the profile for this user. If no settings are configured, when the user logs on to a system for the first time, a user profile will be created on that computer for that user. If the user were to log on to multiple computers, each computer they log into will have a seperate profile created for them. The profile contains the settings for the user, such as the desktop icons and printers. To give the user a more consistant experience, you may decide to keep the user profile on the network. This is referred to as a roaming profile. In order to set a roaming profile, proceed with the following steps...
  • First, set the location on the network that the user has access to, to store the profile. For the user name, you may enter in '%username%'. Windows will substitute this user's login name for this. This is good for when making templates.
  • Here you have the option to enter in a logon script. This script will be run when the user logs in. In the old days a lot of settings including the mapping of network drives were performed using a logon script. Now days, a lot of settings that were originally configured using logon scripts are now performed using 'Group Policy'. So, in most networks, you may not even have a need to use a logon script. If you do, enter in the name of the script here and it will be run each time the user logs in.
  • Here you can configure a home folder for the user. This allows you to configure a location for the user to store their documents. Here you can either enter in a location...
  • Or select the connect option...
  • Then select the drive dropdown button.
  • Then select a drive of the location.
  • And the path.

  • Press 'OK' when finished.
