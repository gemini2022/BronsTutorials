On the 'Account' tab there are quite a few options and in most cases an IT administrator will be mostly using the settings on this tab for day to day administration.
  • The first field is the 'User logon name'. This field can be changed at any time. For example, if the user were to change their name, their login may also need to change.
  • If you press on the dropdown button to the right of the 'User logon name' field...
  • You'll notice that the UPN suffix for the account can also be changed at any time.
  • Here is the pre-Windows 2000 name. On most networks you will not need to worry about configuring this, unless you have some really old clients accessing the network.
  • Here you have the option to configure the logon hours the account can be used. If you click on the 'Logon Hours...' button...
  • You can configure which hours the user can logon to the network. For example, if you first select the 'Logon Denied' option on this form...
  • You are essentially configuring the user to never be able to log in.
  • But, we can then define the logon hours by simply left-clicking on the graph and dragging the cursor to form a rectangle that represents a time frame of when the user can login.
  • And then select the 'Logon permitted' option.

  • Press the 'OK' button when finished.
  • Next, you have the option to configure which computers the user can use. If you click on the 'Log On To...' button...
  • Here you can enter in all of the computers they will be able to use. In a secure enviroment, you may want to configure this option to ensure the user only uses certain computers. For example, if we select the 'The following computers' option...
  • Then type in the computer name...
  • And press the 'Add' button.
  • You can add as many computers as you wish.

  • When finished, press 'OK'.
  • This section determines whether or not the account is locked. If the user has too many wrong password attempts, the account will automatically be locked. Until the account is unlocked, the user will not be able to log on to the network. If they are already logged on, they will not be able to make any new connections. Depending on the configuration of the network, the account may unlock itself after a certain amount of time. If this does not happen, an administrator will need to come into this screen and unselect this option 'Unlock account' to allow the user to use this account again.
  • Here in the 'Account options' section, are some options already seen in the 'New Account' wizard. For example, the 'User must change password at next logon' option is one of them. Like before, you have the option to force the user to have to change their password the next time they log in.
  • The next option will prevent the user from changing their password. When this option is selected, an administrator will need to change the password for this account.
  • This option will configure the password for the user to never expire. This overrides what ever password options are configured in the domain.
  • Here, this option stores a copy of the user password that can be decrypted. This is considered to be as unsecure as storing the password in plain text. Anyone with enough access to 'Active Directory', can read the user account attribute and decrypt the password. For this reason, it is best to not use this option, unless you have a really good reason.
  • If we step down about four notches...
  • We will get to the next option, 'Account is disabled'. This option will do just that, disable the account. If the user leaves the company, it is good practice to disable the account first, rather than just deleting it.
  • This option will not allow a user to logon to the network unless they have a smart card to log in. This provides additional security if you have smart cards configured in your organization.
  • Here, this option prevents delegation on this account.
  • This option changes the encryption used to store the user password in DES. This is a weaker encryption standard, but the password is still encrypted.
  • If we scroll all the way down...
  • We will get to the next option, which if selected, 128 bit encryption will be used. This gives you more security, but also means that more processing power is required. Hower, just because this option is selected, doesn't mean that a higher encryption will be used. It just means that it is available and will be used when the client supports it and is configured to use it.
  • Here, if this option is selected, 256 bit encryption will be used. This gives you even more security, but again also means that more processing power is required. Hower, the same applies, just because this option is selected, doesn't mean that a higher encryption will be used. It just means that it is available and will be used when the client supports it and is configured to use it.
  • And finally, the last option when selected, means that a time stamp when the user was authenticated will not be added to the Kerberos ticket. Without the timestamp on the Kerberos ticked, a hacker could report communications between the client and the server and use it in what is called a replay attack. A replay attack is when the same communication is played back to the server from the hacker at a later date pretending to be that computer. Some authentication systems may need this option enabled in order to work, otherwise it is not a good idea to have this option enabled.
  • In this section, is the option to configure when this account expires. By default, the account will never expire. If you configure the account to expire, the account will stop working at midnight on this date. To configure the account to expire, first click on the 'End of' option.
  • Then select the dropdown...
  • And choose the date you want this account to expire.
  • When finished, press the 'Ok' button.
