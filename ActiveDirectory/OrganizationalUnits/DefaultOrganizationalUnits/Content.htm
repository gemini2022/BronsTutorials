An organizational unit is a subdivision within an Active Directory into which you can place users, groups, computers, and other organizational units. You can create organizational units to mirror your organization's functional or business structure. Each domain can implement its own organizational unit hierarchy. If your organization contains several domains, you can create organizational unit structures in each domain that are independent of the structures in the other domains. The term "organizational unit" is often shortened to "OU" in casual conversation. "Container" is also often applied in its place, even in Microsoft's own documentation. All terms are considered correct and interchangeable. However, default organizational units are automatically created and cannot be deleted. To view these default organizational units, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Active Directory Users and Computers'...
  • Once in 'Active Directory Users and Computers', expand your domain, in this case it's 'bronstutorials.local'.
  • Once you expand your domain, you will be able to see a list of the default organizational units. However, there are additional default organizational units that are created when 'Active Directory' is installed, but aren't present because they are hidden.
  • You can unhide these hidden default organizational units by going to 'View' at the top of your screen...
  • And then by selecting 'Advanced Features'.
  • Now if you expand your domain again...
  • You will see a complete list of all the default organizational units.
  • The first default OU is the 'Builtin' OU. This OU contains all the user accounts that normally exist only in the local accounts database. When a server is promoted to a domain controller, these local accounts are moved into the 'Active Directory' database. For most 'Active Directory' administration, you do not need to use this OU. In here are the user accounts that are shared between domain conrollers in the domain. If you were to create a user in this OU, this user account would only be available to domain controllers.

 
  • The second default OU is the 'Computers' OU. When you add a computer to the domain, the computer account for the computer is put in this OU. Like the other OUs, Group Policy can not be applied to this OU, so you are more than likely going to move the computer account to another OU after it has been created.
  • The next default OU is the 'Domain Controllers' OU. This OU contains all the computer accounts for all the domain controllers for that domain. Unlike the other OUs, Group Policy can be applied to this OU. When a server is promoted to a domain controller, the computer account for that server will move or be created in this OU.
  • The next default OU is the 'ForeignSecurityPrincipals' OU. When you have multiple domains connected by trusts, you may see accounts appear in here. These are created automatically by 'Active Directory' and are used to help bridge the two domains. In normal day to day administration, you would not need to go into this OU.
  • The next default OU is the 'LostAndFound' OU. This OU is the home of orphaned objects in 'Active Directory'. Under certain conditions in 'Active Directory', an object may become orphaned and moved in here. For example, this could happen if an object is created and deleted while still in the process of being replicated around the domain. If you see any object in here, it is up to you to review these and see if you still require any of these objects.
  • The next default OU is the 'Managed Service Accounts' OU. This OU will only be used with managed service accounts in Windows Server 2008 R2 and Windows 7.
  • The next default OU is the 'Program Data' OU. This OU holds application-specific data created by other programs. This OU is empty until a program designed to store information in Active Directory uses it.
  • The next default OU is the 'System' OU. This OU holds configuration information about the domain including security groups and permissions, the domain SYSVOL share, DFS configuration information, and IP security policies.
  • The next default OU is the 'Users' OU. This OU will contain all the default users and groups that are created when 'Active Directory' is first installed. For example, the default administrators account. This OU does not allow group policy to be applied, for this reason you are more than likely to move users that you create in this OU to another OU that you've created.
  • The last default OU is the 'NTDS Quotas' OU. This OU holds objects that contain limits on the number of objects users and groups can own.
