To learn about the Address Pool tab, proceed with the following steps...
  • This first section lists the configured ranges of public IP addresses assigned to you. The address ranges are typically obtained from your Internet service provider.
  • To add an address range to the list, begin by first pressing the 'Add...' button.
  • Once the 'Add' button is pressed, a dialog box will appear. It is here where the address range will be created. The first setting, 'Start address', is the address that will be added to the The IP address (in dotted decimal notation) of the starting number of your public address pool. In this case, we will add the starting address of '172.100.235.238'.
  • Next, we need to add the subnet mask. In this case, the subnet mask is '255.255.224.0'.
  • And then this last setting 'End address', is the IP address (in dotted decimal notation) of the ending number of your public address pool. In this case, the last address of the range is '172.100.255.255'.

  • Press 'OK'.
  • Now that the address range has been created, we now have the option to reserve an address from this range that you do not want to be used for address translation. Reserved public IP addresses can be used for Internet Web servers that are placed on the private network. You cannot configure a reservation until you configure at least one IP address pool. To reserve an address from the address pool, begin by pressing the 'Reservations...' button.
  • Once the 'Reservations' button is pressed, a dialog box will appear that lists all the configured reservations. Each reservation maps a specific public IP address from the public address pool to a specific private address. Reserved public IP addresses are not used for dynamically mapped traffic.
  • To add a reservation, begin by first pressing the 'Add...' button.
  • Once the 'Add' button is pressed, a dialog box will appear. It is here where the reservation will be created. The first setting, 'Reserve this public IP address', is the IP address in the public address pool that is being reserved. In this case, the public address that we will use for the reservation will be '172.100.235.250'.
  • This next setting, 'For this computer on the private network', is the IP address of the private network computer that you want to make available to public network users. In this case, the private network computer that we will be using for this reservation is our web server, it's IP address is '192.168.1.50'.
  • And then finally, this last option, 'Allow incoming sessions to this address', specifies whether sessions from the public network are allowed. If you are setting up a server, such as a Web server, and you want to make it accessible for Internet users, select this check box. Normally, the only traffic that is allowed from the public network is traffic sent in response to sessions initiated from the private network. Since we are making a reservation for a Web server, we will select this option.
  • Press 'OK'.
