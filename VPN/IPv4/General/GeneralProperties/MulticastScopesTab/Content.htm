To learn about the Multicast Scopes tab, proceed with the following steps...
  • This tab lists all the multicast scopes that have been configured. A multicast scope is a named range of IP multicast addresses that is expressed with an IP address and mask. Once multicast scopes are configured, you can use them to create scope-based multicast boundaries in the properties of an IP routing interface. To create a multicast scope, begin first by pressing the 'Add..' button.
  • Once the 'Add...' button is pressed, the first thing that needs to be configured is the name of the multicast scope. In this case, we will name the multicast scope 'MulticastScope1'.
  • Next, enter the IP address for the multicast scope. Valid administrative, nonlocal, multicast scopes must be in the range 239.0.0.0 through 239.254.255.255. The IP address that we will give this multicast scope will be '239.0.0.0'.
  • Then enter the mask. In this case, the mask will be '255.0.0.0'.

  • Press 'OK'.
  • A new multicast scope is now configured.
