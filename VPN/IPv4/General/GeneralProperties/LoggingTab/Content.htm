To learn about the Logging tab, proceed with the following steps...
  • This first option 'Log errors only', instructs the server to log IP routing related errors and nothing elese. This gives you adequate indication of problems after they happen, but it doesn't point out potential problems noted by warning messages.
  • This next option 'Log errors and warnings', instructs RRAS to log error and warning messages to the event log without adding any informational messages. If you get in the habit of carefully reviewing your event logs, these warning messages may give you welcome forewarning of incipient problems.
  • This option 'Log the maximum amount of information', causes the IP routing stack to log messages about almost everything it does. This gives you a lot of useful fodder when you're troubleshooting, but it can flood your logs wiht minutiae if you're not careful, don't turn it on unless you're trying to isolate and fix a problem.
  • And then finally, this last option 'Disable event logging', turns off all IP routing event logging.
