To learn about the General tab, proceed with the following steps...
  • This first option specifies whether TCP/IP is enabled on this interface. Select this option to set the Administrative Status of the interface to Up. Clear this option to set the Administrative Status of the interface to Down.
  • This next option specifies whether Internet Control Message Protocol (ICMP) router discovery advertisements are enabled over this interface. Router discovery clients use these messages to dynamically configure their default gateways. Router discovery is implemented by using multicast ICMP router advertisement messages.
  • This setting is the number of minutes that a router discovery advertisement from this router and this interface is valid. The default is 30 minutes.
  • This setting is the preference level of using this interface as the default gateway. A higher number means that this gateway is more preferred by router discovery clients as the default gateway.
  • This setting is the minimum number of minutes between router discovery advertisement packets. The default is 7 minutes. The router advertisement is sent at a random interval between the minimum time and the maximum time.
  • This setting is the maximum number of minutes between router discovery advertisement packets. The default is 10 minutes. The router advertisement is sent at a random interval between the minimum time and the maximum time.
  • Here you can set filters based on where the packet is coming from (the source), where the packet is going (the destination), or what protocol the packet is using. You can set all three of these values for a filter or set any one or combination of the three, depending on the level of control that you want. First, press the 'Inbout Filters...' button.
  • Then click 'New...' to add a filter.
  • This option specifies whether the filter checks the network from which the packet is coming. After selecting this check box, you must specify a network IP address and subnet mask.
  • Then enter the IP address of the network from which the packet was sent. In this case, we want to filter any packet that is coming from the network '192.168.100.0'.
  • Then enter the subnet mask that corresponds to the network IP address. In this case it is '255.255.255.0'.
  • This option specifies whether the filter checks the network to which the packet is going. After selecting this check box, you must specify a network address and subnet mask.
  • Then enter the IP address of the network from which the packet is going. In this case, we want to filter any packet that is going to the network '192.168.0.0'.
  • Then enter the subnet mask that corresponds to the network IP address. In this case it is '255.255.255.0'.
  • Here, this setting lists the available protocol types that you can base a filter on. The default is Any protocol. If you select Other, you must type a protocol identifier in Protocol number.
  • Once the dropdown is selected, you can choose one of the following options.
    • TCP
    • TCP [established]
    • UDP
    • ICMP
    • Any
    • Other
  • In this case, we will choose the option to use any protocol.

  • Press 'OK' to create the filter.
  • Now that the filter is created, we next have to decide whether this filter will be used to deny packets from the source network, or allow packets from the source network. This option specifies that all incoming packets that match the filters listed are discarded. All incoming packets are compared to the filters listed. If a packet matches a filter, the packet is dropped. If the packet does not match a filter, the packet is accepted.
  • This option specifies that all incoming packets that match the filters listed are accepted. All incoming packets are compared to the filters listed. If a packet matches a filter, the packet is accepted. If the packet does not match a filter, the packet is dropped.

  • Press 'OK'.
  • This time we will create a filter for all outbound packets.
  • Again, click 'New...' to add a filter.
  • Like before, this option specifies whether the filter checks the network from which the packet is coming. After selecting this check box, you must specify a network IP address and subnet mask.
  • Then enter the IP address of the network from which the packet will be sent. In this case, now we want to filter any packet that is coming from this network '192.168.0.0'.
  • Then enter the subnet mask that corresponds to the network IP address. In this case it is '255.255.255.0'.
  • And also like before, this option specifies whether the filter checks the network to which the packet is going. After selecting this check box, you must specify a network address and subnet mask.
  • Then enter the IP address of the network from which the packet is going. In this case, now we want to filter any packet that is going to the network '192.168.100.0'.
  • Then again, enter the subnet mask that corresponds to the network IP address. In this case it is '255.255.255.0'.
  • And like before, this setting lists the available protocol types that you can base a filter on. The default is Any protocol. If you select Other, you must type a protocol identifier in Protocol number.
  • And also like before, once the dropdown is selected, you can choose one of the following options.
    • TCP
    • TCP [established]
    • UDP
    • ICMP
    • Any
    • Other
  • In this case, we will again choose the option to use any protocol.

  • Press 'OK' to create the filter.
  • Now that the filter is created, we next have to decide whether this filter will be used to deny packets to the destination network, or allow packets to the destination network. This option specifies that all outgoing packets that match the filters listed are discarded. All outgoing packets are compared to the filters listed. If a packet matches a filter, the packet is dropped. If the packet does not match a filter, the packet is transmitted.
  • And this option specifies that all outgoing packets that match the filters listed are transmitted. All outgoing packets are compared to the filters listed. If a packet matches a filter, the packet is transmitted. If the packet does not match a filter, the packet is dropped.

  • Press 'OK'.
  • And then finally, this last option specifies whether the router discards all fragmented IP packets that do not correspond with traffic allowed by the filters configured for this interface.
