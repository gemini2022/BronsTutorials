To learn about the Multicast Boundries tab, proceed with the following steps...
  • In this first section of the Multicast Boundries tab is a dropdown that lists all the available multicast scopes that are configured on the Multicast Scopes tab in the properties of IP Routing-General. A multicast scope is a named range of IP multicast addresses that is defined by an IP address and mask. In this case there is only one multicast scope configured.
  • Next, we will press the 'Add' button to add our multicast scope to the list below. This list represents all the scopes for the multicast traffic that will not be forwarded out this interface.
  • Now that a multicast scope is added to the list, we can now specify whether or not the router performs Time-to-Live (TTL) scoping to prevent the forwarding of IP multicast traffic out this interface with a TTL less than a specified value. TTL scoping applies to all multicast packets regardless of the multicast group.
  • Once this option is selected, you will receive a warning message stating that using TTL scoping is not recommended and that you should use group address scoping instead.

  • Press 'OK'.
  • This setting specifies the maximum TTL for multicast traffic that is forwarded out this interface.
  • And then finally, this setting specifies the maximum rate (in kilobytes per second) for all multicast traffic forwarded by this interface.
