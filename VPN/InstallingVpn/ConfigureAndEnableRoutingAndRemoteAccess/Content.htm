To learn how to configure and enable Routing and Remote Access, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Routing and Remote Access'...
  • Once in Routing and Remote Acces, right-click the server node, which in this case is 'VPNSERVER (local)'.
  • Then select the option 'Configure and Enable Routing and Remote Access', this will launch the 'Routing and Remote Access Server Setup Wizard'.
  • The first screen of the wizard is the welcome screen.

  • Click 'Next >'.
  • On this screen, you're asked to choose what services you want to enable on this server. In this case, we will choose the option 'Virtual private network (VPN) access and NAT'.
  • Click 'Next >'.
  • On this screen, we need to select the network interface that connects this server to the internet, or in other words, our public IP. In this case, our public network interface is the interface with the IP address of '172.100.226.160'.
  • Click 'Next >'.
  • On this screen of the wizard, you have a couple of options on how you want your remote clients to be assigned IP addresses. The first option is automatically. If you have a DHCP server configured on your network, then you can have your remote clients get their IP address settings from the DHCP server. If you don't have a DHCP server, then the IP addresses can still be assigned automatically to your remote clients, but from a scope generated by this server.
  • The second option is to have your remote clients get their IP addresses from a range of addresses that you specify, which in this case is the option we will select.
  • Click 'Next >'.
  • Since we selected the option to specify a range of IP addresses for our remote clients on the previous screen, on this screen we need to create that range. To do this, begin by pressing the 'New...' button.
  • Then enter in the range of IP address you want for your remote clients. If the IP address range that you enter consists of IP addresses of a subnet that's different from the subnet of your network, then you need to either enable an IP routing protocol on the remote access server or add static IP routes of each range to the routers of your network. If this subnet does differ from your network subnet and routes are not added, then your remote clients will not be able to receive traffic from resources on the network.
  • In this case, we will configure the range to be on the same subnet as our network, but we will configure the range to start at an address that isn't in the address range of our DHCP server. This way we can be sure that there won't be any IP address conflicts. In this case, the first address of the range will be '192.168.0.200'.
  • Now, we can either specify the last IP address of the range here...
  • Or we can simply enter in how many address we want to be in the range. In this case we will enter in '20'.

  • Press 'OK'.
  • Now that we have the range configured, we can move on.

  • Click 'Next >'.
  • On this next screen, you'll need to specity if you will be authenticating your remote clients using Routing and Remote Access on this server or authenticating your remote clients using a RADIUS server. A RADIUS server enables remote access servers to communicate with a central server to authenticate dial-in users and authorize their access to the requested system or service. RADIUS allows a company to maintain user profiles in a central database that all remote servers can share. It provides better security, allowing a company to set up a policy that can be applied at a single administered network point.

  • In this case, we don't have a RADIUS server deployed on the network, therefore we will go with the default option 'No, use Routing and Remote Access to authenticate connection request'.

  • Click 'Next >'.
  • And then finally, press 'Finish' to complete the wizard.
  • Once you click 'Finish' on the wizard, a message will appear informing you that in order to support the relaying of messages from remote access clients, you must configure the properties of the DHCP Relay Agent with the IP addresses of your DHCP server.

  • Press 'OK'.
  • Routing and Remote Access will then begin to start.
  • Then all you'll have to do is wait for the Routing and Remote Access service to finish initializing.
