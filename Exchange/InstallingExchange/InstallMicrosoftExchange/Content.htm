This first step, updating the Active Direcory Schema and preparing the domain, is not a requirement for the installation of Exchange. This proccess will occur automatically if you install Exchange with an account that has permissions to update the schema and prepare the domain. However, you may have to perform this step if you are installing Exchange in a large organization or it may be a policy that has been put in place to do so. To learn how to update the Active Direcory Schema and prepare the domain, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the command prompt by clicking on the command prompt icon.
  • Once the command prompt is open, the first thing we need to do is change the directory to the folder where the Exchange setup file resides. To do this, begin by typing 'cd'.
  • Then type out the path to the setup file. In this case, our setup file is located in the 'Documents' folder, so the path will be, 'C:\Users\Administrator\Documents'.
  • Now that the directory has been changed to the location where the Exchange setup file resides, we can now begin entering the commands that will update the Active Direcory Schema and prepare the domain. First, start by entering 'setup'.
  • Followed by the parameter '/PrepareAD'.
  • Now, because Exhange has not been installed on our domain as of yet, no Exchange organization container exists at this time. Therefore, with this next step, we have to specify an organization name. To do this, begin by typing '/OrganizationName:'.
  • Then type the name of your organization, which in this case is 'bronstutorials'. After the organization name has been entered, press 'Enter'.
  • After 'Enter' is pressed, we then have the option of pressing any key to cancel setup if we don't accept the licensing terms. But in this case, we will let the setup continue.
  • Once the setup is done and if it completed successfully, a message will be displayed informing you that it completed successfully.
