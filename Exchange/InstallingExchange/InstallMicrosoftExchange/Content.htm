To learn how to install Exchange, proceed with the following steps...
  • Begin first by navigating to the directory where your Exchange setup files are located, then double-click on the file labeled 'setup' to run the Microsoft Exchange 2010 installer.
  • Once the Microsoft Exchange 2010 installer is open, we need to first install some pre-installation components. The first two will already be installed.
  • So we will begin by selecting the third step 'Choose Exchange language option'.
  • Once the step 'Choose Exchange language option' is selected, when then need to decide if we want to install all languages from the language bundle or install only languages from the DVD.
  • In this case, we will select the option 'Install only languages from the DVD'.
  • Now we are ready to install Exchange. To do this, click on the fourth step 'Install Microsoft Exchange' to open the 'Exchange Server 2010 Setup' wizard.
  • The first screen of the wizard is the introduction page, click 'Next >' to begin.
  • Before Microsoft Exchange Server can be installed, we must first accept the terms in the license agreement. The license agreement will be displayed here. You may read it if you wish.
  • If you agree to the terms, then select the option 'I Accept the terms in the license agreement'.
  • Click 'Next >'.
  • On this screen, you have the option of enabling the Exchange Error Reporting feature. If this option is enabled and an error occurs, then Microsoft Exchange will automatically send an error report to Microsoft. In this case, we will not enable this feature and move on.
  • Click 'Next >'.
  • On this next screen, you have the option of choosing between two installation types. The first installation type is 'Typical Exchange Server Installation'. This option will install the basic components needed to send and receive mail.
  • The second option is 'Custom Exchange Server Installation'. With this option, you can customize the installation by choosing the roles you want to install.
  • In this case, we will select the option 'Custom Exchange Server Installation'.
  • We can then specify where we want the Exchange Server program files to be installed. In this case, we will leave the default path and move on.
  • Click 'Next >'.
  • Since we selected the option 'Custom Exchange Server Installation', we can now select the roles we want to install. In this case, we will select these four roles.
  • The 'Edge Transport Role', can only be installed by itself. The edge transport server role is deployed in an Exchange organization's perimeter network. Message protection and security are handled by various agents that run on the edge transport server and act on email messages as they are processed.
  • Once any of the Exchange server roles get selected, the option to install Management Tools will automatically get selected as well.
  • Again, we can then specify where we want the Exchange Server program files to be installed. In this case, we will leave the default path and move on.
  • Click 'Next >'.
  • If you have client computers running Office Outlook 2003 or earlier and you select 'Yes', Exchange 2010 will create a public folder database on the Mailbox server. If all your client computers are running Outlook 2010, public folders are optional in Exchange 2010. If you select 'No', Exchange 2010 won't create a public folder database on the Mailbox server. You can add a public folder database later. For example, if you add client computers running Outlook 2003 and you need a public folder database, you can create one on the Exchange 2010 Mailbox server. You must then configure the OAB for public folder distribution, and then restart the Microsoft Exchange Information Store service before client computers running Outlook 2003 and earlier will be able to connect to the server. In this case, we do not have client computers running Office Outlook 2003 or earlier, so we leave the option of 'No' selected and move on.
  • Click 'Next >'.
  • If the Client Access server will be Internet-facing, you can configure a domain name that your Client Access server will use. To do this, begin by selecting the option 'The Client Access server role will be Internet-facing'.
  • Then enter the domain name here.
  • In this case, we will enter 'mail.bronstutorials.com'.
  • Click 'Next >'.
  • On this screen, you have the option of participating in the 'Customer Experience Improvement Program'. The Customer Experience Improvement Program (CEIP) collects information about how you use Microsoft Exchange in you organization. The information that is collected helps Microsoft identify which Microsoft Exchange features to improve. Here you have the option of either joining the Exchange Customer Experience Improvement Program or not joining at this time.
  • In this case, we will choose the option 'I don't wish to join the program at this time'.
  • Click 'Next >'.
  • On this screen, view the status to determine if the organization and server role prerequisite checks completed successfully. If they haven't completed successfully, you must resolve any reported errors before you can install Exchange 2010. You don't need to exit Setup when resolving some of the prerequisite errors. After resolving a reported error, click Retry to rerun the prerequisite check. Be sure to also review any warnings that are reported.
  • If all readiness checks have completed successfully, you will then be able to install Exchange 2010.
  • Click 'Install'.
  • Wait while the Exchange 2010 installation takes place.
  • Once the Exchange 2010 installation is complete, scroll down to the bottom of the list to verify that all installation components have installed successfully.
  • If you keep the option 'Finalize this installation using the Exchange Management Console' enabled, the EMC will automatically open and connect to the nearest server in your Exchange organization.
  • Click 'Finish'.
