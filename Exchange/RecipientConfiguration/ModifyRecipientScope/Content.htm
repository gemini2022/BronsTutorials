Recipient scope is the specified portion of Active Directory hierarchy that the Exchange Management Console (EMC) and the Exchange Management Shell uses for recipient management. When you set the recipient scope to a specific location within Active Directory, you can view and manage all recipients stored in that location, including all the containers under it. The EMC and Shell always start with the recipient scope at the domain-level of the computer that is running the management interface. Neither the user account that's being used nor the Exchange servers being managed has bearing on the default value of the recipient scope. When you change the recipient scope of the EMC, you change the set of recipients that are displayed in the result pane of the Recipient Configurationnode. The dialog boxes that you use to select recipients or OUs (located on various wizard pages) also work within the same scope. For example, if you're mail-enabling an existing contact, the Select Contact dialog box in the New Mail Contact wizard displays only the contacts within the recipient scope that are not already mail-enabled. To learn how to modify the Recipient Scope, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Exchange Management Console' by selecting the 'Exchange Management Console' icon.
  • Once the 'Exchange Management Console' is open, expand the node 'Microsoft Exchange On-Premise'.
  • Then select 'Recipient Configuration'.
  • After 'Recipient Configuration' is selected, select the option 'Modify Recipient Scope' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • First, you can select this option if you want to work with all recipients in the forest.
  • Then, if you want to use a specific global catalog server, begin by selecting the 'Global Catalog' check box.
  • Then we have to select a specific global catalog server. To do this, begin by clicking 'Browse' to open the 'Select Global Catalog' dialog box.
  • Once the 'Select Global Catalog' dialog box is open, select the global catalog server you want to use. In this case there is only one global catalog server, which is already selected by default.
  • Click 'OK'.
  • Or if you want to work with all recipients in a specific organizational unit (OU) and all OUs under it, select this option 'View all recipients in specified organizational unit'.
  • Then select a specific organizational unit. To do this, begin by clicking 'Browse' to open the 'Select Organizational Unit' dialog box.
  • Once the 'Select Organizational Unit' dialog box is open, select the organizational unit you want to use. In this case we will select the 'Users' organizational unit.
  • Then click 'OK'.
  • To use a specific domain controller, select the 'Recipient Domain Controller' check box.
  • Then select a specific domain controller. To do this, begin by clicking 'Browse' to open the 'Select Domain Controller' dialog box.
  • Once the 'Select Domain Controller' dialog box is open, select the domain controller you want to use. In this case there is only one domain controller, which is already selected by default.
  • Click 'OK'.
