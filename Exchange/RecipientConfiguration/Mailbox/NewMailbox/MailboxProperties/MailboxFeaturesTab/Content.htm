To learn about the Mailbox Features tab, proceed with the following steps...
  • This first feature, 'Outlook Web App' is enabled by default. To configure the properties for this feature, first select it.
  • Then click the 'Properties' button to open the 'Outlook Web App Properties' dialog box.
  • Once the 'Outlook Web App Properties' dialog box is open, select this option to add an Outlook Web App mailbox policy to this user's mailbox
  • Then click the 'Browse' button to open the 'Select Outlook Web App Mailbox Policy' dialog box.
  • Once the 'Select Outlook Web App Mailbox Policy' dialog box is open, select an Outlook Web App mailbox policy. In this case, we will select the Web App mailbox policy 'OutlookWebAppMailboxPolicy'.
  • After an Outlook Web App mailbox policy is selected, click 'OK'.
  • After the 'OK' button on the 'Select Outlook Web App Mailbox Policy' dialog box is clicked, the Outlook Web App mailbox policy that you selected will be displayed here.
  • Click 'OK'.
  • This next feature, 'Exchange ActiveSync' is also enabled by default. To configure the properties for this feature, first select it.
  • Then click the 'Properties' button to open the 'Exchange ActiveSync Properties' dialog box.
  • Once the 'Exchange ActiveSync Properties' dialog box is open, the current Exchange ActiveSync mailbox policy will be displayed here.
  • To change this policy to another Exchange ActiveSync mailbox policy, click the 'Browse' button to open the 'Select ActiveSync Mailbox Policy' dialog box.
  • And then select an Exchange ActiveSync mailbox policy in the list. In this case, we will select 'ExchangeActiveSyncMailboxPolicy'.
  • Once an Exchange ActiveSync mailbox policy is selected, click 'OK'.
  • After the 'OK' button on the 'Select ActiveSync Mailbox Policy' dialog box is clicked, the Exchange ActiveSync mailbox policy that you selected will be displayed here.
  • Click 'OK'.

  • This feature, 'Unified Messaging' is disabled by default. To enable Unified Messaging (UM) for this mailbox, in the EMC result pane, select the mailbox, and then, in the action pane, click 'Enable Unified Messaging'.

  • Once this feature is enabled, you can configure the properties for this feature. To do this, first select it.
  • Then click the 'Properties' button to open the 'Unified Messaging Properties' dialog box.
  • Once the 'Unified Messaging Properties' dialog box is open, this display-only field shows the status of the user's mailbox. By default, when a user is UM-enabled, the mailbox status is listed as Not locked out. However, if the user has input an incorrect Outlook Voice Access PIN multiple times, the status is listed as Locked Out.
  • This display-only field shows the name of the UM mailbox policy associated with the UM-enabled user.
  • You can specify the UM mailbox policy to be associated with this UM mailbox by first clicking the 'Browse' button.
  • Then select a UM mailbox policy.
  • Once a UM mailbox policy is selected, click 'OK'.
  • After clicking 'OK' on the dialog box, the UM mailbox policy that you selected will be displayed here.

  • This box displays the extension numbers and the Session Initiation Protocol (SIP) and E.164 addresses after the user has been enabled for Unified Messaging. If you've associated the user with a Telephone Extension dial plan, only the extension number configured for the user appears in this box. If you've associated the UM-enabled user with a SIP dial plan, an extension number and SIP address are listed. If you've associated the UM-enabled user with an E.164 dial plan, an extension number and E.164 address are listed.

  • Select this option to specify whether users can use ASR when they sign in to their mailbox. By default, UM-enabled users can use voice commands when they use Outlook Voice Access to access their mailbox. Even if Outlook Voice Access users are speech-enabled, they must still use the keypad to enter their extension number, PIN, and personal options.

  • Select this option to specify whether to allow incoming calls from unauthenticated callers through an auto attendant to be transferred to the UM-enabled user. By default, this setting is enabled. Enabling this setting allows callers from outside an organization to be transferred to a user inside an organization. The user's mailbox can still be accessed using directory searches. However, if an external caller tries to transfer to a user for whom this setting is disabled, the system will say, "I'm sorry, I am unable to transfer the call to this user." The caller is then transferred to the operator configured on the auto attendant. This setting doesn't apply to callers who've signed in to their mailbox using Outlook Voice Access and are sending a voice message to a user.

  • Select this option to specify whether a user is allowed to receive incoming faxes. By default, this setting is enabled. However, it can be disabled if you don't want the user to receive incoming faxes. This setting is also configured on dial plans. If you enable this setting for a UM-enabled user, but the dial plan is configured to disable fax receiving, the UM-enabled user is unable to receive faxes.
  • Select this option to specify whether, for diverted calls without a caller ID, the caller is allowed to leave a message. By default, this option is enabled. This enables the UM-enabled user to accept anonymous calls from callers.
  • Select or clear this check box to allow or prevent a user from creating personal auto attendants. If this option is disabled on the UM dial plan or the UM mailbox policy, this feature isn't available to UM-enabled users associated with the UM mailbox policy. This option isn't available to UM-enabled users that have a mailbox on an Exchange UM server. The default setting is enabled.

  • Use this field to specify the operator extension number for the user. By default, an extension number isn't configured. The range for the extension number is from 1 through 20 characters. This enables incoming calls for the UM-enabled user to be forwarded to the extension number that you specify in this field. You can configure other types of operator extension numbers on dial plans and auto attendants. However, those extensions are generally meant for company-wide receptionists or operators. The personal operator extension setting could be used when an administrative assistant or personal assistant answers incoming calls before they're answered for a particular user.
  • Click 'OK'.
  • This feature 'MAPI', is enabled by default. Click Disable to disable this feature for the mailbox. MAPI enables access to an Exchange mailbox from a MAPI client such as Outlook. There is no property page available for this feature.
  • This next feature, 'POP3' is enabled by default. To configure the properties for this feature, first select it.
  • Then click the 'Properties' button to open the 'POP3 Properties' dialog box.
  • Once the 'POP3 Properties' dialog box is open, you have the option of using the default protocol.
  • Or use customized settings that takes precedence over the protocol default settings.
  • Click 'OK'.
  • This next feature, 'IMAP4' is enabled by default. To configure the properties for this feature, first select it.
  • Then click the 'Properties' button to open the 'IMAP4 Properties' dialog box.
  • Once the 'IMAP Properties' dialog box is open, you have the option of using the default protocol.
  • Or use customized settings that takes precedence over the protocol default settings.
  • Click 'OK'.
  • If an archive doesn't exist for the mailbox, this feature is disabled . To enable an archive for this mailbox, in the EMC result pane, select the mailbox and then, in the action pane, click Enable Archive.
  • Once an archive does exist for the mailbox, you can configure the properties for this feature. To do this, begin first by selecting it.
  • Then click the 'Properties' button to open the 'Archive Properties' dialog box.
  • Once the 'Archive Properties' dialog box is open, you can specify a name for this archive.
  • Click 'OK'.
