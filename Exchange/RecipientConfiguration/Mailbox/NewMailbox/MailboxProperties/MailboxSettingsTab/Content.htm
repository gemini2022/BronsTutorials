To learn about the Mailbox Settings tab, proceed with the following steps...
  • With this first setting, 'Messaging Records Management', we have the option of configuring messageing records management policies. To do this, begin by first selecting this setting.
  • And then click the 'Properties' button to open the 'Messaging Records Management' dialog box.
  • Once the 'Messaging Records Management' dialog box is open, the first thing we can do is assign a managed folder mailbox policy to this mailbox. To do this, select the option 'Managed folder mailbox policy'.
  • Then click the 'Browse' button to open the 'Select Managed Folder Mailbox Policy' dialog box.
  • Once the 'Select Managed Folder Mailbox Policy' dialog box is open, select a managed folder mailbox policy from the list of available mailbox policies. In this case, there is only one managed folder mailbox policy to select, which is already selected by default.
  • Click 'OK'.
  • Next, select this check box to place this mailbox on retention hold. Placing a mailbox on retention hold suspends the processing of a retention policy or managed folder mailbox policy for that mailbox. Retention hold is designed for scenarios such as a user being on vacation or away temporarily.
  • Once this option is selected, we next have to specify when the retention hold should begin. To do this, first select the option 'Start date'.
  • Then select this dropdown.
  • Once the dropdown is selected, use the calendar to define a starting date for the retention hold.
  • You can also specify the time when the retention hold starts.
  • Next, we have to specify when the retention hold should end. To do this, select the option 'End date'.
  • Then select this dropdown.
  • Once the dropdown is selected, use the calendar to define an end date for the retention hold.
  • You can also specify the time when the retention hold ends.
  • Click 'OK'.
  • Next, we have the option of defining a sharing policy for this mailbox for Federated sharing. To do this, begin by first selecting the setting 'Federated Sharing'.
  • And then click the 'Properties' button to open the 'Federated Sharing' dialog box.
  • Once the 'Federated Sharing' dialog box is open, the current sharing policy that is defined will be displayed here.
  • To change the policy, first click the 'Browse' button to open the 'Select Sharing Policy' dialog box.
  • Once the 'Select Sharing Policy' dialog box is open, select a sharing policy from the list of available sharing policies. In this case, we will select the policy 'SharingPolicy2'.
  • Once a sharing policy is selected, click 'OK'.
  • After clicking 'OK' on the 'Select Sharing Policy' dialog box, the sharing policy you selected will be displayed here.
  • Click 'OK'.
  • Next, we have the option of specifying storage quotas for this mailbox. To do this, begin by first selecting the setting 'Storage Quota'.
  • And then click the 'Properties' button to open the 'Storage Quotas' dialog box.
  • Once the 'Storage Quotas' dialog box is open, we have the option of either using the mailbox database defaults for storage quotas or specifying a more custom storage quota configuration. In this case we will clear this checkbox to specify a custom storage configuration.
  • Use this check box and use the corresponding text box to specify the maximum storage limit in kilobytes (KB) before a warning is issued to the mailbox user. The value range is from 0 through 2,147,483,647 KB. If the mailbox size reaches or exceeds the value specified, Exchange sends a warning message to the mailbox user.
  • Use this check box and use the corresponding text box to specify a prohibit send limit in KB for the mailbox. The value range is from 0 through 2,147,483,647 KB. If the mailbox size reaches or exceeds the specified limit, Exchange prevents the mailbox user from sending new messages and displays a descriptive error message.
  • Use this check box and use the corresponding text box to specify a prohibit send and receive limit in KB for the mailbox. The value range is from 0 through 2,147,483,647 KB. If the mailbox size reaches or exceeds the specified limit, Exchange prevents the mailbox user from sending new messages and won't deliver any new messages to the mailbox. Any messages sent to the mailbox are returned to the sender with a descriptive error message.
  • Next, we have the option of either using the mailbox database defaults for deleted item retention or specifying a more custom deleted item retention configuration. In this case we will clear this checkbox to specify a custom deleted item retention configuration.
  • Use this text box to specify how many days to keep deleted items for.
  • If you don't want the deleted items to be permanently deleted until the database is backed up, select this checkbox.
  • Click 'OK'.
  • And then finally, we have the option of specifying archive quotas for this mailbox. To do this, begin by first selecting the setting 'Archive Quota'.
  • And then click the 'Properties' button to open the 'Archive Quota' dialog box.
  • Once the 'Archive Quota' dialog box is open, select the 'Issue warning at (MB)' check box.
  • And then use the corresponding box to type the personal archive size in megabytes (MB), at which a warning will be sent to the user.
  • In this case, we will enter '1000'.
  • Click 'OK'.
