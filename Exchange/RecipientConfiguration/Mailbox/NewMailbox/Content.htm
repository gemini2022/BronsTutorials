Mailboxes are the most common recipient type used by information workers in an Exchange organization. Each mailbox is associated with an Active Directory user account. The user can use the mailbox to send and receive messages, and to store messages, appointments, tasks, notes, and documents. You can also create mailboxes for resources such as meeting rooms and equipment. Mailboxes are the primary messaging and collaboration tools for the users in your organization. To learn how to create a new Mailbox, proceed with the following steps...
  • Begin by selecting the option 'New Mailbox' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • To begin, first we need to choose the mailbox type. The first option 'User Mailbox', is a mailbox that is owned by a user to send and receive e-mail messages. User mailboxes can't be used for resource scheduling. The Active Directory account that is associated with user mailboxes must reside in the same forest as the Exchange server. To use an account in a trusted forest, select Linked Mailbox.
  • The next mailbox type is 'Room Mailbox'. This mailbox type is used as a location resource for scheduling meetings. Room mailboxes can be included in meeting requests as resources and can be configured to automatically process incoming requests. If you create a new user account for the room mailbox in Active Directory, it will be disabled. If you plan to associate the room mailbox with an existing account, you must select a disabled account.
  • Next is 'Equipment Mailbox'. This mailbox type is used as a resource for scheduling meetings. Equipment mailboxes can be included in meeting requests as resources and can be configured to automatically process incoming requests. As a best practice, create mailboxes for shared meeting equipment, such as projectors or audio equipment, which can be moved to different meeting rooms. If you create a new user account for the equipment mailbox in Active Directory, it will be disabled. If you plan to associate the equipment mailbox with an existing account, you must select a disabled account.
  • And then finally, the last mailbox type is 'Linked Mailbox'. This mailbox type is a user mailbox that is accessed by a user in a separate, trusted forest. You must still create a user account in the forest in which Exchange Server resides. This is required to create the necessary Active Directory object for storing the mailbox information. Linked mailboxes might be necessary for organizations that choose to deploy Exchange in a resource forest. The resource forest scenario allows an organization to centralize Exchange in a single forest, while allowing access to the Exchange organization with user accounts in one or more trusted forests.
  • Click 'Next >'.
  • On this screen of the wizard we need to choose whom we want to create the new mailbox for. The first option 'New user', will simultaneously create a new user in Active Directory and mail-enable the user.
  • Or this option 'Existing users', which will mail-enable one or more existing users.
  • Once the 'Existing users' option is selected, we then need to select a user account in the forest that isn't mail-enabled or doesn't have an Exchange mailbox. To do this, begin first by clicking 'Add' to open the 'Select User' dialog box.
  • Once the 'Select User' dialog box is open, we then need to select a user account in the forest that isn't mail-enabled or doesn't have an Exchange mailbox. However, in this case, there is no user account available that isn't mail-enabled or doesn't have an Exchange mailbox.
  • Click 'Cancel'.
  • In this case, we'll have to choose the option 'New user'.
  • Click 'Next >'.

If the recipient scope is set to a specific domain, the Users container in that domain is selected by default. If the recipient scope is set to a specific OU, that OU is selected by default. To select a different OU, select the option 'Specify the organizational unit rather than using a default one'.
  • Then we need to select the desired OU. To do this, beging first by clicking 'Browse' to open the 'Select Organizational Unit' dialog box.
  • Once the 'Select Organizational Unit' dialog box is open, you will see all OUs in the forest that are within the specified scope.

  • Use this box to type the initials of the user. This field is also optional.
  • Use this box to type the last name of the user. This field is optional also. In this case, we will enter 'Mailbox'.
  • Use this box to type a name for the user. This is the name that's listed in Active Directory. By default, this box is populated with the names you enter in the First name, Initials, and Last name boxes. If you didn't use those boxes, you must still type a name in this field. The name can't exceed 64 characters.
  • Use this box to type the name that the user will use to log on to the mailbox. In this case, we will enter the name 'newmailbox'.
  • The user logon name consists of a user name and a suffix. Typically, the suffix is the domain name in which the user account resides. However, there may be more than one suffix option to choose from. To choose from a list of suffix options, select the dropdown.
  • Then choose a suffix from the list. In this case, there is only one suffix to choose from.
  • Use this box to type the name for the user that is compatible with the legacy versions of Microsoft Windows (prior to the release of Windows 2000 Server). This field is automatically populated based on the User logon name (User Principal Name) field. This field is required.
  • Use this box to type the password that the user must use to log on to his or her mailbox. This box won't be available if you're creating a room, equipment, or linked mailbox.
  • Use this box to confirm the password that you typed in the Password box. This box won't be available if you're creating a room, equipment, or linked mailbox.

  • Select this check box if you want the user to reset the password when they first logon to the mailbox. This box won't be available if you're creating a room, equipment or linked mailbox. If you select this check box, at first logon, the new user will be prompted with a dialog box in which to change the password. The user won't be allowed to perform any tasks until the password is successfully changed.

  • If you're creating a new user, or if you selected a single existing user to mail-enable, you can modify the alias for the mailbox. If you're mail-enabling more than one existing user, you can't specify a value for this field. The alias can't exceed 64 characters and must be unique in the forest.
