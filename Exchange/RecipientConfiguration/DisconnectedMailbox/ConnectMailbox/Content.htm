To learn how to connect a mailbox that has been disconnected, proceed with the following steps...
  • Begin first by selecting a disconnected mailbox.
  • Then select the option 'Connect' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • First, we need to specify the mailbox type for the mailbox we're connecting. The first mailbox type is 'User Mailbox'. Select this option to connect the mailbox as a mailbox owned by a user to send and receive e-mail messages. User mailboxes can't be used for resource scheduling. The Active Directory account that is associated with user mailboxes must reside in the same forest as the Exchange server. To use an account in a trusted forest, select Linked Mailbox.
  • The next mailbox type is 'Room Mailbox'. This mailbox type is used as a location resource for scheduling meetings (room mailbox) or a shared resource (equipment mailbox) that isn't location specific. Room and equipment mailboxes can be included in meeting requests as resources and can be configured to automatically process incoming requests.
  • Next is 'Equipment Mailbox'. This mailbox type is used as a resource for scheduling meetings. Equipment mailboxes can be included in meeting requests as resources and can be configured to automatically process incoming requests. As a best practice, create mailboxes for shared meeting equipment, such as projectors or audio equipment, which can be moved to different meeting rooms. If you create a new user account for the equipment mailbox in Active Directory, it will be disabled. If you plan to associate the equipment mailbox with an existing account, you must select a disabled account.
  • And then finally, the last mailbox type is 'Linked Mailbox'. This mailbox type is a user mailbox that's accessed by a user in a separate, trusted forest. To store the mailbox information, you must select a user account in the forest in which the Exchange server resides. Linked mailboxes might be required for organizations that choose to deploy Exchange in a resource forest. Using the resource forest scenario, you can centralize Exchange in a single forest, while allowing access to the Exchange organization with user accounts in one or more trusted forests.
  • Click 'Next >'.

  • Next, select this option to have Exchange locate a matching user object in Active Directory.

  • Once this option is selected, we next need to select a matching user. To do this, begin by first clicking 'Browse' to open the 'Select User' dialog box.

  • If Exchange locates a matching user, it will appear in this dialog box. In this case there is only one matching user, which is already selected by default.

  • Click 'OK'.
  • Select this option if you want to connect the mailbox to a user other than the matching user.
  • Once this option is selected, we next need to select an available user in Active Directory. To do this, begin first by clicking 'Browse' to open the 'Select User' dialog box.
  • Once the 'Select User' dialog box is open, we then need to select a user in the list. This list contains only users that don't have an associated mailbox.
  • In this case, we will not choose any of the users in the list. Click 'Cancel'.
  • Instead, we will select the option 'Matching user'.
  • And then we need to select a matching user. To do this, begin by first clicking 'Browse' to open the 'Select User' dialog box.
  • Now that the 'Select User' dialog box is open, we'll use the user that is already selected by default.
  • Click 'OK'.
  • Next, use this box to type an alias for the mailbox. In this case, we'll enter 'disconnectedmailbox' for the alias.
  • Select this check box to assign a manage folder mailbox policy to the mailbox.
  • Then click 'Browse' to select a policy from a list of available policies.
  • In this case, there are no managed folder mailbox policies available.
  • Therefore, we will cancel out of this dialog box.
  • And deselect the option to assign a manage folder mailbox policy to the mailbox.
  • Select this check box to assign a Microsoft Exchange ActiveSync policy to the mailbox.
  • Then click 'Browse' to select a policy from a list of available Exchange ActiveSync policies.
  • In this case there is one Exchange ActiveSync policy available.
  • However, we will not assign a Microsoft Exchange ActiveSync policy to the mailbox, so we will cancel out of this dialog box.
  • And then deselect the option to assign a Microsoft Exchange ActiveSync policy to the mailbox.
  • Click 'Next >'.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'Connect'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
