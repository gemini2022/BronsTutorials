Mail users are similar to mail contacts. Both have external e-mail addresses and both contain information about people outside your Exchange organization that can be displayed in the global address list (GAL) and other address lists. However, unlike a mail contact, a mail user has Active Directory logon credentials and can access resources. To learn how to create a new Mail User, proceed with the following steps...
  • Begin by selecting the option 'New Mail User' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • This option is selected by default. Select this option to create a new user account for the mail user. This allows you to provision the user account and mail-enable it simultaneously.
  • Or select this option to mail-enable an existing user account in Active Directory.
  • Then click 'Browse' to open the 'Select User' dialog box.
  • This dialog box displays a list of user accounts in the forest that aren't mail-enabled or don't have Exchange mailboxes.
  • In this case, we won't be selecting any existing users, so we will cancel out of this dialog box.
  • And instead, select the option 'New user'.
  • Click 'Next >'.
  • Select this check box to select an organizational unit (OU) other than the default (which is the recipient scope). If the recipient scope is set to the forest, the default value is set to the Users container in the Active Directory domain that contains the computer on which the Exchange Management Console is running. If the recipient scope is set to a specific domain, the Users container in that domain is selected by default. If the recipient scope is set to a specific OU, that OU is selected by default.
  • To select a different OU, click 'Browse' to open the 'Select Organizational Unit' dialog box.
  • Once the 'Select Organizational Unit' dialog box is open, you will see all OUs in the forest that are within the specified scope.
  • In this case, we will select the 'Users' OU.
  • And then click 'OK'.
  • Use this box to type the user's first name. This field is optional. In this case, we will enter 'Mail'.
  • Use this box to type the user's middle initials. This field is optional.
  • Use this box to type the user's last name. This field is optional. In this case, we will enter 'User'.
  • Use this box to type a name for the user. This is the name that's listed in Active Directory. By default, this box is populated with the names you enter in the First name, Initials, and Last name boxes. If you didn't use those boxes, you must still type a name in this field. The name can't exceed 64 characters.
  • Use this box to type the name that the user will use to log on to the mailbox. In this case, we will enter the name 'mailuser'.
  • The user logon name consists of a user name and a suffix. Typically, the suffix is the domain name in which the user account resides. This name can't exceed 1,024 characters and must be unique in the forest. However, there may be more than one suffix option to choose from. To choose from a list of suffix options, select the dropdown.
  • Then choose a suffix from the list. In this case, there is only one suffix to choose from.
  • Use this box to type the name for the user that is compatible with the legacy versions of Microsoft Windows (prior to the release of Windows 2000 Server). This field is automatically populated based on the User logon name (User Principal Name) field. This name can't exceed 20 characters and must not contain any of the following characters: \/ []:|<> + = ; ? , *.
  • Use this box to type the password for the user.
  • Use this box to confirm the password that you typed in the Password box.

  • Select this check box if you want the user to reset the password when they first logon to the mailbox. If you select this check box, at first logon, the new user will be prompted with a dialog box in which to change the password. The user won't be allowed to perform any tasks until the password is successfully changed.
  • Click 'Next >'.
  • Use this text box to type the alias of the user. The alias can't exceed 64 characters and must be unique in the forest.
  • In this case, we will enter 'mailuser' for the alias.
  • Next, we need to specify an external e-mail address for the user.
  • To specify a SMTP e-mail address, click 'Edit'. To specify a custom e-mail address, click the arrow next to 'Edit'. In this case, we will select 'Edit' which will open the 'SMTP address'dialog box.
  • Once the 'SMTP address'dialog box is open, we need to then enter the SMTP email address.
  • In this case, we will enter 'mailuser@externaladdress.com'.
  • Once the email adress is entered, click 'OK'.
  • Once 'OK' is clicked on the 'SMTP address'dialog box, the email address will be displayed here.
  • Click 'Next' to continue.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
