Mail contacts are mail-enabled Active Directory objects that contain information about people or organizations that exist outside your Exchange organization. Each mail contact has an external e-mail address. To learn how to create a new Mail Contact, proceed with the following steps...
  • Begin by selecting the option 'New Mail Contact' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • This option is selected by default. Select this option to create and mail-enable a new contact.
  • Or select this option to mail-enable an existing contact in Active Directory.
  • Then we need to select a contact in the forest that isn't mail-enabled. To do this, begin by first clicking 'Browse' to open the 'Select Contact' dialog box.
  • In this case, there is no contacts available to choose.
  • Therefore, we will cancel out of this dialog box.
  • And instead, select the option 'New contact'.
  • Click 'Next >'.
  • Select this check box to select an organizational unit (OU) other than the default (which is the recipient scope). If the recipient scope is set to the forest, the default value is set to the Users container in the Active Directory domain that contains the computer on which the Exchange Management Console is running. If the recipient scope is set to a specific domain, the Users container in that domain is selected by default. If the recipient scope is set to a specific OU, that OU is selected by default.
  • To select a different OU, click 'Browse' to open the 'Select Organizational Unit' dialog box.
  • Once the 'Select Organizational Unit' dialog box is open, you will see all OUs in the forest that are within the specified scope.
  • In this case, we will select the 'Users' OU.
  • And then click 'OK'.
  • Use this box to type the contact's first name. This field is optional. In this case we will enter 'Mail'.
  • Use this box to type the contact's initials. This field is optional.
  • Use this box to type the contact's last name. This field is optional. In this case, we will enter 'Contact'.
  • Use this box to type a name for the contact. This is the name that's listed in Active Directory. By default, this box is populated with the names you enter in the First name, Initials, and Last name boxes. If you didn't use those boxes, you must still type a name in this field. The name can't exceed 64 characters.
  • Use this box to type a unique alias (64 characters or less) for the contact. This field is required. In this case, we will enter 'mailcontact' for the alias.
  • Next, we need to specify an external e-mail address for the contact.
  • To specify a SMTP e-mail address, click 'Edit'. To specify a custom e-mail address, click the arrow next to 'Edit'. In this case, we will select 'Edit' which will open the 'SMTP address' dialog box.
  • Once the 'SMTP address' dialog box is open, we need to then enter the SMTP email address.
  • In this case, we will enter 'mailcontact@externaladdress.com'.
  • Once the email adress is entered, click 'OK'.
  • Once 'OK' is clicked on the 'SMTP address' dialog box, the email address will be displayed here.
  • Click 'Next' to continue.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
