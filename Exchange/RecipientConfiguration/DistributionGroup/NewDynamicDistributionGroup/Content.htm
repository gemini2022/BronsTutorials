Dynamic distribution groups are mail-enabled Active Directory group objects that are created to expedite the mass sending of e-mail messages and other information within an Exchange organization. Unlike regular distribution groups that contain a defined set of members, the membership list for dynamic distribution groups is calculated each time a message is sent to the group, based on the filters and conditions that you define. When an e-mail message is sent to a dynamic distribution group, it is delivered to all recipients in the organization that match the criteria defined for that group. To learn how to create a new Dynamic Distribution Group, proceed with the following steps...
  • Begin by selecting the option 'New Dynamic Distribution Group' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Select this check box to select an organizational unit (OU) other than the default (which is the recipient scope). If the recipient scope is set to the forest, the default value is set to the Users container in the Active Directory domain that contains the computer on which the Exchange Management Console is running. If the recipient scope is set to a specific domain, the Users container in that domain is selected by default. If the recipient scope is set to a specific OU, that OU is selected by default.
  • To select a different OU, click 'Browse' to open the 'Select Organizational Unit' dialog box.
  • Once the 'Select Organizational Unit' dialog box is open, you will see all OUs in the forest that are within the specified scope.
  • In this case, we will select the 'Users' OU.
  • And then click 'OK'.
  • Use this box to type the name for the dynamic distribution group. The name cannot exceed 64 characters. In this case we will enter the name 'DynamicDistributionGroup'.
  • Use this box to type the alias of the dynamic distribution group. The alias cannot exceed 64 characters and must be unique in the forest. In this case, we will enter 'DynamicDistributionGroup' for the alias.
  • Click 'Next >'.
  • Next, we need to define the recipient filter for the new dynamic distribution group. First, we have the option to specify the OU from which to select the recipients. A dynamic distribution group contains all recipients that are in the specified OU and any other OUs under it.
  • Next, we need to select the type of recipients that will be part of the dynamic distribution group. This first option specifies that all recipient types will be part of the group.
  • Or we have the option of selecting specific types of recipients.
  • Select this check box if you want to include users that have mailboxes. Users that have Exchange mailboxes are those that have a user domain account and a mailbox in the Exchange organization.
  • Select this check box if you want to include users that have external e-mail addresses. Users that have external e-mail accounts have user domain accounts in Active Directory, but use e-mail accounts that are external to the organization. This enables them to be included in the global address list (GAL) and added to distribution lists.
  • Select this check box if you want to include Exchange resource mailboxes. Resource mailboxes allow you to administer company resources through a mailbox, such as a conference room or company vehicle.
  • Select this check box if you want to include contacts that have external e-mail addresses. Contacts that have external e-mail accounts do not have user domain accounts in Active Directory, but the external e-mail address is available in the GAL.
  • Select this check box if you want to include security groups or distribution groups that have been mail-enabled. Mail-enabled groups are similar to distribution groups. E-mail messages that are sent to a mail-enabled group account will be delivered to several recipients.
  • In this case, we will select the option 'All recipient types'.
  • Click 'Next >'.
  • Next, we have the option of selecting conditions for the dynamic distribution group. However, if we don't want to set any conditions, we can simply not select any of the check boxes. The first condition that can be set is 'Recipient is in a State or Province'. Select this check box if you want the dynamic distribution group to include only recipients from specific states or provinces. This information is contained on the Address and Phone tab in the recipient's properties.
  • The next condition is 'Recipient is in a Department'. Select this check box if you want the dynamic distribution group to include only recipients in specific departments. This information is contained on the Organization tab in the recipient's properties.
  • The next condition is 'Recipient is in a Company'. Select this check box if you want the dynamic distribution group to include only recipients in specific companies. This information is contained on the Organization tab in the recipient's properties.
  • And then finally, there are 15 custom attributes for each recipient. There is a separate condition for each custom attribute. If you want the dynamic distribution group to include only recipients that have a specific value set for a specific custom attribute, select the check box that corresponds to that custom attribute.
  • In this case, we will select the condition 'Recipient is in a State or Province'.
  • After a condition is selected, we then need to click the underlined term to create your condition. By default, the underlined term for new conditions will read 'specified'. After you edit the condition, the underlined term will change to the value that you specified.
  • Once the underlined term is clicked, a dialog box appears in which you can specify the values for the condition. To begin, we first need to enter a value in the text box.
  • In this case, we will enter the value, 'NY'.
  • Once a value is entered, click 'Add'.
  • After clicking 'Add', the value will appear here. You can add more than one value, but you cannot enter duplicate values.
  • Click 'OK'.
  • You can see now that instead of the underlined term reading 'specified', the underline term now reads the new value that we just entered.
  • Click 'Next >' to continue.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
