Use the New Distribution Group wizard to create a new distribution group in your Exchange organization or to mail-enable an existing group in Active Directory. There are two types of distribution groups, one type is Mail-enabled universal distribution group, which is used only to distribute messages. The other type is Mail-enabled universal security group, which can be used to distribute messages as well as to grant access permissions to resources in Active Directory. To learn how to create a new Distribution Group, proceed with the following steps...
  • Begin by selecting the option 'New Distribution Group' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • First, we need to decide whether to create a new group or to mail-enable an existing group. This frist option, 'New group', creates a new distribution group.
  • Or you can select this option 'Existing group' to mail-enable an existing group in Active Directory.
  • Once the option 'Existing group' is selected, we then need to select an existing group. To do this, begin by frist clicking 'Browse' to open the 'Select Group' dialog box.
  • Once the 'Select Group' dialog box is open, you can choose any of the following groups to mail-enable.
  • In this case, we will not choose any group to mail-enable and cancel out of this dialog box.
  • Instead, we will select the option to create a new group.
  • Click 'Next >'.
  • Next, we need to choose a group type. The first option 'Distribution', creates a mail-enabled universal distribution group.
  • And the next option, 'Security', creates a mail-enabled universal security group.
  • Select this check box to select an organizational unit (OU) other than the default (which is the recipient scope). If the recipient scope is set to the forest, the default value is set to the Users container in the Active Directory domain that contains the computer on which the Exchange Management Console is running. If the recipient scope is set to a specific domain, the Users container in that domain is selected by default. If the recipient scope is set to a specific OU, that OU is selected by default.
  • To select a different OU, click 'Browse' to open the 'Select Organizational Unit' dialog box.
  • Once the 'Select Organizational Unit' dialog box is open, you will see all OUs in the forest that are within the specified scope.
  • In this case, we will select the 'Users' OU.
  • And then click 'OK'.
  • Use this box to type the name of the group, which can't exceed 64 characters. In this case, the name of the group will be 'DistributionGroup'.

  • Use this box to type the name for the group that is compatible with the legacy versions of Windows (prior to the release of Windows 2000 Server). This required field is automatically populated based on the Name field.

  • Use this box to type the name of the alias for the group. The alias cannot exceed 64 characters and must be unique in the forest. In this case we will enter 'DistributionGroup' for the alias.
  • Click 'Next >'.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
