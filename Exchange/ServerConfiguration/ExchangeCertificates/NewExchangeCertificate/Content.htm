After you have installed the Client Access server role, you'll need to create a Secure Sockets Layer (SSL) certificate for the various services in your organization. To learn how to create a new Exchange certificate, proceed with the following steps...
  • Begin by selecting the option 'New Exchange Certificate' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Begin by entering a friendly name for your certificate.
  • In this case, we will name our new Exchange certificate 'ExchangeCertificate'.
  • Click 'Next' to continue.
  • Next, if you want to apply the certificate to all subdomains automatically by creating a wildcard certificate, select this option 'Enable wildcard certificate'.
  • Once the option 'Enable wildcard certificate' is selected, we next need to enter a root domain.
  • In this case, the root domain we will enter will be 'bronstutorials.local'.
  • In this case we will deselect the option 'Enable Wildcard certificate'.
  • Since we are deselecting this option, we will then need to select the services and protocols that our certificate will need to support. To do this, begin by clicking 'Next >'.
  • If you will be using this certificate for Federated Sharing, begin by expanding 'Federated Sharing'.
  • Then select the option 'Public certificate'.
  • Now that federated sharing is enabled, close it.
  • If you'll be using this certificate for Outlook Web App, then expand 'Client Access server (Outlook Web App)'.
  • If you want to use Outlook Web App in your intranet, select the option 'Outlook Web App is on the intranet'.
  • Then enter the FQDN of the Outlook Web App server.
  • In this case, our Outlook Web App server is 'mx1.bronstutorials.local'.
  • And if you want to use Outlook Web App on the internet, select the option 'Outlook Web App is on the internet'.
  • Then enter the domain name you use to access Outlook Web App.
  • In this case, the domain name that is used to access Outlook Web App is 'mail.bronstutorials.com'.
  • Now that Outlook Web App is configured for this certificate, close it.
  • If you'll be using this certificate for Exchange ActiveSync, then expand 'Client Access server (Exchange ActiveSync)'.
  • Then select the 'Exchange ActiveSync is enabled' check box, which is already selected by default.
  • And then enter the domain name you use to access Exchange ActiveSync.
  • Which in this case is 'mail.bronstutorials.com'.
  • Now that Exchange ActiveSync is configured for this certificate, close it.
  • If you'll be using this certificate for Exchange Web Services, Outlook Anywhere, or the Autodiscover service, then expand 'Client Access server (Web Services, Outlook Anywhere, and Autodiscover)'.
  • Then select the option 'Exchange Web Services is enabled' if you want to enable Exchange Web Services, which is already enabled by default.
  • Or select the option 'Outlook Anywhere is enabled' if you want to enable Outlook Anywhere, which is also already enabled by default.
  • Then enter the external host name for your organization. In this case, there is already information entered, but it is not correct. To fix this, select the information in the text box.
  • Then enter the correct information.
  • Which in this case is 'mail.bronstutorials.com'.
  • Next, select whether you want Autodiscover to be used on the Intranet.
  • Or Autodiscover to be used on the internet.
  • Then decide if a long URL will be used.
  • Or a short URL will be used.
  • Or if the URL will be in another format.
  • Then enter the URL. In this case there is already a URL entered, but it is not correct. To fix it, all that needs to be done is to select 'local'.
  • And then change the 'local' part of the URL.
  • Which in case we will change it to 'com'.
  • Now that Web Services, Outlook Anywhere, and Autodiscover has been configured for this certificate, close it.
  • If you'll be using this certificate for POP and IMAP, then expand 'Client Access server (POP/IMAP)'.
  • Then select this check box if you'll be using POP and IMAP on the Intranet.
  • And select this check box if you'll be using POP and IMAP on the Internet.
  • Then we need to enter the domain name that will be used for POP.
  • In this case, there is already information entered, but it is not correct. To fix it, select the information in the text box.
  • Then enter the correct information.
  • Which in this case is 'pop.bronstutorials.com'.
  • Next, we need to enter the domain name that will be used for IMAP. In this case, there is already information entered, but it is not correct. To fix it, select the information in the text box.
  • Then enter the correct information.
  • Which in this case is 'imap.bronstutorials.com'.
  • Now that we have POP and IMAP configured for this certificate, close it.
  • If you'll be using this certificate for Unified Messaging, then expand 'Unified Messaging server'.
  • Choose whether you'll use a self-signed certificate.
  • Or a public certificate. You must use a public certificate if you are using Unified Messaging with Office Communications Server.
  • Then enter the fully qualified domain name (FQDN) of your Unified Messaging server. In this case, the information has already been entered by default.
  • Now that Unified Messaging has been configured for this certificate, close it.
  • If you'll be using mutual TLS to help secure Internet mail or if you'll be using a Hub Transport server for POP and IMAP client submission, then expand 'Hub Transport Server'.
  • Then select this check box if you'll be using mutual TLS to help secure Internet mail.
  • And enter the fully qualified domain name of your connector. In this case, there is already information entered, but it is not correct. To fix it, select the information in the text box.
  • Then enter the correct information.
  • Which in this case is 'mx1.bronstutorials.local'.
  • Then select this check box if you'll be using a Hub Transport server for POP and IMAP client submission.
  • And enter the fully qualified domain name of your connector. In this case, there is already information entered, but it is not correct. To fix it, select the information in the text box.
  • Then enter the correct information.
  • Which in this case is 'mx1.bronstutorials.local'.
  • Now that this has been configured, close it.
  • If you're upgrading from a previous version of Exchange Server and will be operating in a coexistence scenario for a period of time during the upgrade, then expand 'Legacy Exchange Server'.
  • Then enable the option 'Use legacy domains'.
  • And then enter the legacy domain name, which in this case has already ready been entered by default.
  • Click 'Next >' to continue.
  • Next is a list of domains that will be added to the certificate. Here you can click 'Add' to add another domain or click one of the domains listed and then click 'Edit' if you need to make changes to that domain..
  • You also have the option to choose one of the domains to be the common name of the certificate. In this case, we will select the domain, 'mail.bronstutorials.com'.
  • Then click the button 'Set as common name'.
  • Now this domain will be the common name of the certificate.
  • Click 'Next >'.
  • Here we need to enter information about our Exchange organization. We'll need to enter the name of the Organization, the Organization unit, and location information including the Country/region, City/locality, and State/province.
  • Next, we need to select a location for the certificate request file. To do this, begin first by selecting 'Browse'.
  • Then enter the file name you want to use.
  • In this case, we will enter 'ExchangeCertificate'.
  • Then click 'Save'.
  • Click 'Next >'.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Read the information on how to prepare the certificate.
  • Then press 'Finish' to complete the wizard.
