Receive connectors represent a logical gateway through which all inbound messages are received. Receive connectors are configured on a per-server basis, and they control how that server receives messages from the Internet, e-mail clients, and other messaging servers. By default, the Receive connectors required for internal mail flow are automatically created when the Hub Transport server role is installed. Similarly, when you install the Edge Transport server role, the Receive connector capable of receiving mail from the Internet and from Hub Transport servers is automatically created. However, end-to-end mail flow is possible only after the Edge Transport server has been subscribed to the Active Directory site by using the Edge Subscription process. Other scenarios, such as an Internet-facing Hub Transport server or an Edge Transport server that doesn't use EdgeSync, require manual connector configuration to establish end-to-end mail flow. To learn how to create a new Receive Connector, proceed with the following steps...
  • Begin first by selecting a server, which in this case will be 'MX1'.
  • After a server has been selected, select the option 'New Receive Connector' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Begin by typing a meaningful name for this connector. Specify a name for the Receive connector that helps you distinguish this Receive connector from other Receive connectors in your configuration.
  • In this case, we will name this receive connector 'ReceiveConnector'.
  • Next, we need to select the usage type for this connector. The usage type determines the permissions granted to sessions that connect to the Receive connector and the supported authentication mechanisms. To select a usage type for this connector, begin by first selecting the dropdown.
  • Once the dropdown is selected, you'll have the option choosing one of five usage types. The first usage type is 'Custom'. Select this option to create a customized connector that connects with systems that don't include servers running Exchange.
  • The next option is 'Internet'. Internet Receive connectors receive e-mail from servers on the Internet. This connector is configured to accept connections from anonymous users.
  • Next is the option 'Internal'. Internal Receive connectors are used to receive e-mail from servers within your Exchange organization. This connector is configured to only accept connections from Exchange servers.
  • Then we have the option 'Client'. Client Receive connectors receive e-mail from users of Microsoft Exchange. This connector is configured to only accept client submissions from authenticated Microsoft Exchange users. The Client usage type is only available for Receive connectors configured on Hub Transport servers.
  • And then finally, we have the option 'Partner'. Partner Receive connectors are used to receive e-mail from partner domains. This connector is configured to receive mail from domains included in the list of secure receive domains. You can add domains to this list by using the TLSReceiveDomainSecureListparameter in the Set-TransportConfig command. Mutually authenticated TLS connections are required for domains that are on this list.
  • In this case, we will select the option 'Custom'.
  • Once a usage type is selected, click 'Next >'.
  • Next, we need to specify the IP addresses and port numbers on which this Receive connector listens for incoming mail. To do this, begin first by clicking 'Add'.
  • Select this option to use all IP addresses associated with this computer. This is the recommended option.
  • But if you want to specify a particular IP address for the receive connector, select this option.
  • Then enter the IP address you want to use for this receive connector.
  • In this case, we will enter '192.168.1.2'.
  • This field identifies the TCP port number on which this Receive connector listens for incoming mail. TCP port 25 is the default port used for message transmission between SMTP servers.
  • Click 'OK'.
  • Once 'OK' is clicked, the IP address will be displayed here.
  • Then for this example, we will remove the item in the list refering to all available IP addresses. To do this, select '(All Available IPv4)'.
  • Then click the 'Remove' button.
  • Here we can type the name advertised in response to the SMTP HELO or EHLO verb. If you leave this field blank, the fully qualified domain name (FQDN) of the Hub Transport server or Edge Transport server is automatically added when the connector is created.
  • Click 'Next >' to continue.
  • We next need to enter the IP address or IP address range of the remote servers from which the connector accepts incoming connections. To do this, begin by first clicking 'Add'.
  • Next, enter the IP address directly or specify a subnet using the CIDR notation. For example, if you enter 192.168.1.1, the Receive connector accepts messages from that host only, but if you specify 192.168.1.0/24, the Receive connector accepts messages from the entire class C subnet of 192.168.1.0.
  • In this case, we will enter '192.168.1.50'.
  • Click 'OK'.
  • Once the IP address has been added, it will be displayed here.
  • In this case, we want this receive connector to only receive mail from only one IP address and not a range of addresses. Therefore, we will remove the range from this list. To do this, select the range.
  • Then click 'Remove'.
  • Click 'Next >'.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
