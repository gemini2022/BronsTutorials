Use the 'Network' tab to specify the IP addresses and TCP ports on which this Receive connector receives connections. You can also configure the IP address ranges from which this Receive connector accepts connections. To learn about the Network tab, proceed with the following steps...
  • Use this list to specify the IP addresses and port numbers on which this Receive connector listens for incoming mail. For each entry, you must specify a different set of IP addresses or specify all available IP addresses.
  • To add a new IP address or port number, click 'Add'.
  • Select this option to use all IP addresses associated with this computer. This is the recommended option.
  • Select this option to use a specific IP address associated with this computer.
  • Once this option is selected, enter the IP address here.
  • In this case, we will enter '192.168.1.2'.
  • This field identifies the TCP port number on which this Receive connector listens for incoming mail. TCP port 25 is the default port used for message transmission between SMTP servers.
  • Click 'OK'.
  • After clicking 'OK', the IP address you entered will be displayed here.
  • Use this list to specify the IP address or IP address range from which this Receive connector accepts connections.
  • To enter an IP address without a subnet mask, or to specify the subnet mask by using Classless Interdomain Routing (CIDR) notation, click Add or the drop-down arrow next to Add and select IP Address.
  • In the 'Add IP address(es) of Remote Servers' dialog box, enter the IP address directly or specify a subnet using the CIDR notation. For example, if you enter 192.168.1.1, the Receive connector accepts messages from that host only, but if you specify 192.168.1.0/24, the Receive connector accepts messages from the entire class C subnet of 192.168.1.0.
  • In this case we will enter '192.168.1.100'.
  • Click 'OK'.
  • After clicking 'OK', the IP address you entered will be displayed here.
