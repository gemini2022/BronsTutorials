On the Remote File Servers tab, specify remote file server access. Outlook Web App accesses only internal Windows file shares. A file name can also be specified by using a fully qualified domain name (FQDN) that's internal or that's included in the list of sites that are to be treated as internal. Outlook Web App uses a simple set of criteria to determine whether an address is internal or external. If there are no dots in a URL that a user clicks, it's treated as internal. If there are one or more dots in the URL, it's treated as internal only if the domain suffix has been added to the list of sites to be treated as internal. To learn about the Remote File Servers tab, proceed with the following steps...

  • First, we can specify servers that aren't allowed to be accessed through Outlook Web App. To do this, begin by clicking the 'Block' button to open the 'Block list' dialog box.
  • Once the 'Block List' dialog box is open, we need to specify the servers that cannot be accessed using remote file access.
  • Enter the server you want to block.
  • And then click 'Add'.
  • Once the 'Add' button is clicked, the blocked server will be displayed here.
  • Click 'OK'.
  • Next, we can specify servers that are allowed to be accessed through Outlook Web App. To do this, begin by clicking the 'Allow' button to open the 'Allow list' dialog box.
  • Once the 'Allow List' dialog box is open, we need to specify the servers that can be accessed using remote file access.
  • Enter the server you want to allow.
  • And then click 'Add'.
  • Once the 'Add' button is clicked, the allowed server will be displayed here.
  • Click 'OK'.
  • Select this dropdown to specify how to handle accessing files from servers that aren't in the Block and Allow lists.
  • Once the dropdown is selected, you can choose what will happen when files are accessed from servers that aren't in the Block and Allow lists.
    • We can allow these files to be accessed.
    • Or we can block these files from being accessed.
  • And then finally, we can specify the domain suffixes of sites that are to be treated as internal. You can also add FQDNs to this list of addresses that are to be treated as internal. To do this, begin by clicking the 'Configure' button to open the 'Internal Domain Suffix List'.
  • Once the 'Internal Domain Suffix List' dialog box is open, we need to enter the domain suffix for sites whose FQDN names are treated as internal.
  • Enter the domain suffix.
  • And then click 'Add'.
  • Once the 'Add' button is clicked, the domain suffix will be displayed here.
  • Click 'OK'.
