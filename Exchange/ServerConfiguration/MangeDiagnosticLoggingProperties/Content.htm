Modifying logging levels may help you troubleshoot issues that may occur in an Exchange Server 2010 environment. The logging level for each Exchange process determines which events are written to the Application event log in Event Viewer. Changing the process logging level for a given process may not yield additional events in the event log. Many variables affect whether a change to the process logging level setting will increase the number of events. These variables include, but are not limited to, the actions being performed by the process and the number of events implemented in the source code for the logging level selected. To learn how to manage diagnostic logging properties, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Exchange Management Console' by selecting the 'Exchange Management Console' icon.
  • Once the 'Exchange Management Console' is open, expand the node 'Microsoft Exchange On-Premise'.
  • Then select 'Server Configuration'.
  • After 'Server Configuration' is selected, select the option 'Manage Diagnostic Logging Properties' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Next, we need to select the Exchange service for which we want to change the logging level. In this case, first we'll expand 'MSExchange ActiveSync'.
  • Then we'll select the service 'Configuration'.
  • Once a service is selected, we next need to select a logging level. The first logging level 'Lowest', specifies that only critical events, error events, and events with a logging level of zero (0) are logged.
  • Next, is the logging level 'Low', where events with a logging level of 1 or lower are logged.
  • Then we have the logging level 'Medium', where events with a logging level of 3 or lower are logged.
  • The next logging level is 'High'. With this logging level events with a logging level of 5 or lower are logged.
  • And then finally, this last logging level 'Expert', specifies that events with a logging level of 7 or lower are logged.
  • In this case, we will select the logging level 'Medium'.
  • Once a logging level is selected, any node associated with the service that is selected turns red.
  • With this option, 'Reset all services to default logging levels', you can return all logging levels back to default.
  • Click 'Configure' to configure the logging properties.
  • On this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
