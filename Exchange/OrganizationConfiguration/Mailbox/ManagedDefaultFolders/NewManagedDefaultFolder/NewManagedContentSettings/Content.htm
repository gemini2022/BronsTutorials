Managed content settings are used to define message retention settings and retention action for managed folders. Message lifespan is controlled by either controlling content retention and removing content that is no longer needed, or by automatically journaling (copying) important content to a separate storage location outside the mailbox. This can be any location that has a SMTP e-mail address, including another Exchange mailbox. When the item is journaled, a label is applied to it that indicates how the user classified the item. To learn how to create new Managed Content Settings for a Managed Default Folder, proceed with the following steps...
  • Begin first by selecting a Managed Default Folder. In this case we will select the new Managed Default Folder we just created, 'ManagedDefaultFolder'.
  • Then select the option 'New Managed Content Settings' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • First, use this box to enter a name for the new managed content settings.
  • In this case, we will name the new managed content settings 'ManagedContentSettings'.
  • Next, we need to select the type of message for which we want to create managed content settings for. To do this, first select the dropdown
  • Then select the type of message for which you want to create managed content settings for. If you select All Items, the managed content settings that you create will apply to all the items in the managed folder (except for message types for which you have created separate content settings).
  • In this case, we will select 'All Mailbox Content'.
  • Next, we have the option of setting how long messages live within user's mailbox. To do this, first select the 'Length of retention period (days)' check box.
  • Then, once the 'Length of retention period (days)' check box is selected, in the field to the right, we need to enter the number of days after which the message is to expire.
  • In this case, we will enter '60' days.
  • Next, we need to define when the retention period starts. To do this, first select the dropdown.
  • Then we need to select one of two options. The first option is 'When delivered, end date for calendar and recurring tasks'. With this option, the retention period for mail starts when it is delivered. The retention period for calendar items and recurring tasks starts at their end date.
  • And with the second option, 'When item is moved to the folder', the expiration countdown does not start until the items are transferred to another folder, such as when items are moved from the Inbox to a review before final deletion folder.
  • In this case, we will select the option 'When delivered, end date for calendar and recurring tasks'.
  • Next, we need to define the action that will take place when the retention period ends. To do this, first select the dropdown.
  • Then choose one of five options in the list. With this first option, 'Move to the Deleted Items folder', messages are automatically moved to the Deleted Items folder.
  • This next option 'Move to a managed custom folder', messages are automatically moved to the managed custom folder that you specify in the 'Move to the following managed custom folder' box.
  • Next, the option 'Delete and allow recovery', messages are deleted but may be recovered by using Recover Deleted Items in Microsoft Office Outlook. Note that messages can be recovered only for as long as messages are configured to remain in a recoverable state.
  • With the option 'Permanently delete', messages are permanently deleted and not recoverable by the user.
  • And then finally, 'Mark as past retention Limit', messages are displayed in a shaded, strikethrough font in Outlook, but no other action is taken. This expiration action is used to prompt the user to take some kind of action.
  • In this case, we will select the option 'Move to a Managed Custom Folder'.
  • Since we selected the option 'Move to a Managed Custom Folder', we now have to define what that managed custom folder is. To do this, first click the 'Browse' button.
  • In this case, there is only one managed custom folder that has been created and by default is already selected.
  • Press 'OK'.
  • Press 'Next >' to move on.
  • On this screen, we have the option to automatically forward a copy of the item of the message type that was specified on previous screen to another location or user. To do this, first select the option 'Forward copies to'.
  • Then click 'Browse'.
  • In this case, the administrator is the only user we can forward messages to and by default is already selected.
  • Press 'OK'.
  • Here we have the option of typing a label to be applied to a journaled message.
  • Next, we need to choose the format of the copied messages that will be attached to the journal report.
  • There are two options to choose from, the first option is 'Outlook Message Format (*.msg)'. With this option, Messages are attached in the .msg file format that is commonly used by Outlook.
  • And the second option is 'Exchange MAPI Message Format (TNEF)'. With this option Messages are attached in the format (Transport Neutral Encapsulation Format) that is most commonly used by Exchange 2007.
  • In this case, we will choose the second option, 'Exchange MAPI Message Format (TNEF)'.
  • Click 'Next >'.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
