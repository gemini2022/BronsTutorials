To learn how to edit an Address List, proceed with the following steps...
  • Begin first by selecting an Address List. In this case we will select the new Address List we just created, 'AddressList'.
  • Then select the option 'Edit' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Use this box to view the existing display name, or to type a new display name for the address list. The name can contain as many as 64 characters including wildcard characters, but not the backslash (\) character.
  • Use this box to type the for the new address list. This is the name that's displayed to users when they view the address list from a client such as Outlook. Although this field is automatically populated with the same name you type in the Name box, you can modify it.
  • This read-only box displays the path to the container for the address list. You set this path when you run the New Address List wizard.
  • Press 'Next >' to move on.
  • We next can edit the OU from which to select the recipients. To do this, begin by clicking 'Browse'.
  • Once the 'Browse' button is clicked, the 'Select Organizational Unit' dialog box will appear. It is here where we specify the OU from which to select the recipients.
  • In this case we will leave the selection set as the root OU.
  • Click 'OK'.
  • Next, we can edit the type of recipients we want to include when dynamically creating our address list. However, with this first option, 'None', we have the option of not using any recipient types in the query to create the list.
  • Or we can use all recipient types.
  • Or we have the option of selecting specific types of recipients.
  • Select this check box if you want the address list to apply to users that have Exchange mailboxes. Users that have Exchange mailboxes are those that have a user domain account and a mailbox in the Exchange organization.
  • Select this check box if you want the address list to apply to users that have external e-mail addresses. Users that have external e-mail accounts have user domain accounts in Active Directory, but use e-mail accounts that are external to the organization. This enables them to be included in the GAL and added to distribution lists.
  • Select this check box if you want the address list to apply to Exchange resource mailboxes. Resource mailboxes allow you to administer company resources through a mailbox, such as a conference room or company vehicle.
  • Select this check box if you want the address list to apply to contacts that have external e-mail addresses. Contacts that have external e-mail accounts don't have user domain accounts in Active Directory, but the external e-mail address is available in the GAL.
  • Select this check box if you want the address list to apply to security groups or distribution groups that have been mail-enabled. Mail-enabled groups are similar to distribution groups. E-mail messages that are sent to a mail-enabled group account will be delivered to several recipients.
  • In this case, we will select the option 'All recipient types'.
  • Click 'Next >'.
  • Next, we can edit the conditions for our address list. The first condition that can be edited is 'Recipient is in a State or Province'. Select this check box if you want the address list to include only recipients from specific states or provinces. This information is contained on the Address and Phone tab in the recipient's properties.
  • The next condition is 'Recipient is in a Department'. Select this check box if you want the address list to include only recipients in specific departments. This information is contained on the Organization tab in the recipient's properties.
  • The next condition is 'Recipient is in a Company'. Select this check box if you want the address list to include only recipients in specific companies. This information is contained on the Organization tab in the recipient's properties.
  • And then finally, there are 15 custom attributes for each recipient. There is a separate condition for each custom attribute. If you want the address list to include only recipients that have a specific value set for a specific custom attribute, select the check box that corresponds to that custom attribute.
  • In this case, we will leave the conditions as is, but we will edit our underlined term. To do this, click on the underlined term.
  • Once the underlined term is clicked, a dialog box appears in which you can edit the values of the condition. In this case we will add an additional state to the condition.
  • Which will be 'CA'.
  • Once a value is entered, click 'Add'.
  • After clicking 'Add', the value will appear here above the value we entered previously.
  • Click 'OK'.
  • You can see now that both 'NY' and 'CA' are part of the condition.
  • Click 'Next >' to continue.
  • Next, we can edit when the address list should be applied. This first option, 'Do not apply', means that after the address list is edited, the address list will not be applied until the list is applied manually at a later time.
  • Select this option to apply the address list as soon as it is done being edited.
  • Select this option to specify a time when to apply the address list.
  • Then select the dropdown.
  • Once the dropdown is selected, a calendar will appear where you can set the date when the address list will be applied.
  • Then you can set the time when the address list will be applied.
  • In this case, we will select the option 'Immediately', so that the address list will be applied as soon as the list is done being edited.
  • Not only can we edit when the address list will be applied, but you can also edit how long the address list will be applied for. To do this, select the option 'Cancel tasks that are still running after (hours)'.
  • Then enter in how long the address list will run. The default is 8 hours.
  • Click 'Next >' to move on.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'Edit'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
