An address list is a collection of recipient and other Active Directory objects. Each address list can contain one or more types of objects (for example, users, contacts, groups, public folders, and room and equipment resources). You can use address lists to organize recipients and resources, making it easier to find the recipients and resources you want. Address lists are updated dynamically. Therefore, when new recipients are added to your organization, they're automatically added to the appropriate address lists. An Exchange organization can contain thousands of recipients. If you compile all your recipients in the default address lists, those lists could become quite large. To prevent this, you can create custom address lists to help users in your organization find what they are looking for more easily. For example, consider a company that has two large divisions and one Exchange organization. One division, named Fourth Coffee, imports and sells coffee beans. The other division, Contoso, Ltd, underwrites insurance policies. For most day-to-day activities, the employees at Fourth Coffee don't communicate with the employees at Contoso, Ltd. Therefore, to make it easier for employees to find recipients who exist only in their division, you can create two new custom address lists—one for Fourth Coffee and one for Contoso, Ltd. When searching for recipients in their division, these custom address lists allow employees to select only the address list that's specific to their division. However, if an employee is unsure about the division in which the recipient exists, the employee can search within the GAL, which contains all recipients in both divisions. You can also create subcategories of address lists called hierarchical address lists. For example, you can create an address list that contains all recipients in Manchester and another that contains all recipients in Stuttgart. To learn how to create a new Address List, proceed with the following steps...
  • Begin by selecting the option 'New Address List' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Use this box to type the name for the new address list. The name can contain as many as 64 characters, including wildcard characters, but can't contain the backslash character (\).
  • In this case, we will call the new address list 'AddressList'.
  • Use this box to type the for the new address list. This is the name that's displayed to users when they view the address list from a client such as Outlook. Although this field is automatically populated with the same name you type in the Name box, you can modify it.
  • We next have to select the path to the container for the address list. To do this, begin first by selecting 'Browse'.
  • Once the 'Browse' button is clicked, the 'Select Address List' dialog box will appear. To add the address list as a child to an existing address list, click the existing address list you want. To create a new parent address list, click All Address Lists.
  • In this case, we will select 'All Address Lists'.
  • After a container is selected, click 'OK'.
  • Press 'Next >' to move on.
  • We next need to specify the OU from which to select the recipients. To do this, begin by clicking 'Browse'.
  • Once the 'Browse' button is clicked, the 'Select Organizational Unit' dialog box will appear. It is here where we specify the OU from which to select the recipients.
  • In this case we will select the root OU.
  • Then click 'OK'.
  • Next, we need to select the type of recipients we want to include when dynamically creating our address list. However, with this first option, 'None', we have the option of not using any recipient types in the query to create the list.

 
  • Or we can use all recipient types.
  • Or we have the option of selecting specific types of recipients.
  • Select this check box if you want the address list to apply to users that have Exchange mailboxes. Users that have Exchange mailboxes are those that have a user domain account and a mailbox in the Exchange organization.
  • Select this check box if you want the address list to apply to users that have external e-mail addresses. Users that have external e-mail accounts have user domain accounts in Active Directory, but use e-mail accounts that are external to the organization. This enables them to be included in the GAL and added to distribution lists.
  • Select this check box if you want the address list to apply to Exchange resource mailboxes. Resource mailboxes allow you to administer company resources through a mailbox, such as a conference room or company vehicle.
  • Select this check box if you want the address list to apply to contacts that have external e-mail addresses. Contacts that have external e-mail accounts don't have user domain accounts in Active Directory, but the external e-mail address is available in the GAL.
  • Select this check box if you want the address list to apply to security groups or distribution groups that have been mail-enabled. Mail-enabled groups are similar to distribution groups. E-mail messages that are sent to a mail-enabled group account will be delivered to several recipients.
  • In this case, we will select the option 'All recipient types'.
  • Click 'Next >'.
  • Next, we have the option of selecting conditions for our address list. However, if we don't want to set any conditions for the list, we can simply not select any of the check boxes. The first condition that can be set is 'Recipient is in a State or Province'. Select this check box if you want the address list to include only recipients from specific states or provinces. This information is contained on the Address and Phone tab in the recipient's properties.
  • The next condition is 'Recipient is in a Department'. Select this check box if you want the address list to include only recipients in specific departments. This information is contained on the Organization tab in the recipient's properties.
  • The next condition is 'Recipient is in a Company'. Select this check box if you want the address list to include only recipients in specific companies. This information is contained on the Organization tab in the recipient's properties.
  • And then finally, there are 15 custom attributes for each recipient. There is a separate condition for each custom attribute. If you want the address list to include only recipients that have a specific value set for a specific custom attribute, select the check box that corresponds to that custom attribute.
  • In this case, we will select the condition 'Recipient is in a State or Province'.
  • After a condition is selected, we then need to click the underlined term to create your condition. By default, the underlined term for new conditions will read 'specified'. After you edit the condition, the underlined term will change to the value that you specified.
  • Once the underlined term is clicked, a dialog box appears in which you can specify the values for the condition. To begin, we first need to enter a value in the text box.
  • In this case, we will enter the value, 'NY'.
  • Once a value is entered, click 'Add'.
  • After clicking 'Add', the value will appear here. You can add more than one value, but you cannot enter duplicate values.
  • Click 'OK'.
  • You can see now that instead of the underlined term reading 'specified', the underline term now reads the new value that we just entered.
  • Click 'Next >' to continue.
  • Next, we need to specify when the address list should be applied. This first option, 'Do not apply', means that after the address list is created, the address list will not be applied until the list is applied manually at a later time.
  • Select this option to apply the address list as soon as it is created.
  • Select this option to specify a time to apply the new address list.
  • Then select the dropdown.
  • Once the dropdown is selected, a calendar will appear where you can set the date when the new address list will be applied.
  • Then you can set the time when the new address list will be applied.
  • In this case, we will select the option 'Immediately', so that the new address list will be applied as soon as the list is created.
  • Not only do you have the option to define when the new address list will be applied, but you can also define how long the address list will be applied for. To do this, select the option 'Cancel tasks that are still running after (hours)'.
  • Then enter in how long the address list will run. The default is 8 hours.
  • Click 'Next >' to move on.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
