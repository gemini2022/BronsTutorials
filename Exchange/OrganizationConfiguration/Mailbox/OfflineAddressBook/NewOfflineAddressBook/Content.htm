An offline address book (OAB) in Exchange is a copy of an address book that's been downloaded so that an Outlook user can access the information it contains while disconnected from the server. Exchange administrators can choose which address books are made available to users who work offline, and they can also configure the method by which the address books are distributed (Web-based distribution or public folder distribution). Outlook downloads the latest updates for their offline address book whenever they go online. Then they can use their updated address book when they are offline. To learn how to create a new Offline Address Book, proceed with the following steps...
  • Begin by selecting the option 'New Offline Address Book' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Use this box to provide a name for the new offline address book.
  • In this case we will name our new offline address book 'OfflineAddressBook2'.
  • Next, click the 'Browse' button to specify the location where the OAB generation server will be located. The OAB generation server is the Mailbox server on which the OAs are generated.
  • Once the 'Browse' button is pressed, we need to then select a mailbox server from the list that will be designated as the generation server. In this case, there is only one mailbox server available and by default is already selected.
  • Press 'OK'.
  • The offline address book is a snapshot of the Active Directory information that's available in the global address book. Select this check box to include the default global address book in the OAB.
  • Select this check box to add address lists to or remove address lists from the OAB.
  • Next, click the 'Add' button to open the 'Select Address List' dialog box.
  • Once the 'Select Address List' dialog box is open, you have the option of selecting any of the following address lists. The top-level address list can not be selected.
  • In this case, we will select the 'All Users' address list.
  • Then click 'OK'.
  • You can see here that the 'All Users' address list is now added to our new offline addres book.
  • Click 'Next >' to continue.
  • Select this check box to distribute the OAB from selected virtual directories. Web-based distribution is the distribution method by which Outlook 2007 or later clients that are working offline or through a dial-up connection access OABs. Clients will download the OAB over HTTPS from the virtual directory you specify.
  • Select this check box to enable public folder distribution. Public folder distribution is the distribution method by which Outlook 2003 or earlier clients that are working offline or through a dial-up connection access OABs.
  • In this case we will select the option 'Enable Web-based distribution'.
  • After the 'Enable Web-base distribution' option is selected, we next have define a virtual directory that will be used to distribute the OAB. To do this, begin by clicking the 'Add' button.  This will open the 'Select OAB Virtual Directory' dialog box.
  • Once the 'Add' button is pressed, we need to then select a virtual directory from the list. In this case, there is only one virtual directory available and by default is already selected.
  • Press 'OK'.
  • You can see here that we now have a virtual directory added for our web-based distribution.
  • Click 'Next >'.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
