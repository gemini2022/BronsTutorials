A managed folder mailbox policy is a logical grouping of managed folders that can be applied to mailboxes, allowing you to apply managed content settings for multiple managed folders to a user. A policy can contain a mix of managed default folders and managed custom folders. However, you can't add two managed default folders of the same type (such as the Inbox) to the same policy. Managed folders can be added or removed from a managed folder mailbox policy at any time. A mailbox user can have only one managed folder mailbox policy applied. To learn how to create a new Managed Folder Mailbox Policy, proceed with the following steps...
  • Begin by selecting the option 'New Managed Folder Mailbox Policy' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Use this box to type a name for the new mailbox policy.
  • In this case, we will name the new mailbox policy 'MailboxPolicy'.
  • Next, we need to add the folders that this policy will be applied to. To do this, begin by clicking on the 'Add' button.
  • After clicking 'Add', a dialog box will appear that contains a list of all the managed default folders and managed custom folders.
  • From here, we need to select the managed folders that we want to link to the policy. In this case, we will select the managed default folder 'Entire Mailbox'.
  • Once the folders you want to link to the policy have been selected, click 'OK'.
  • You can see here that the folder we selected has been added to the mailbox policy list.
  • Click 'New' to create the mailbox policy.
  • On this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
