When you add a server to a DAG, the server works with the other DAG members to provide automatic database-level recovery from database, server, or network failures. When you remove a server from a DAG, the server is no longer automatically protected from failures. To learn how to manage a Database Availability Group membership, proceed with the following steps...
  • Begin first by selecting a Database Availability Group. In this case we will select the new Database Availability Group we just created, 'DAG1'.
  • Then select the option 'Manage Database Availability Group Membership' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • We will begin by adding a server to the group. To do this, begin by pressing the 'Add' button.
  • Once the 'Add' button is pressed, we need to then select a mailbox server from the list to add to the group. In this case, there is only one mailbox server available and by default is already selected.
  • Press 'OK' to add the selected mailbox server to the list.
  • You can see that we have one member added to the database availability group.
  • Press 'Manage'.
  • In this case the wizard could not be completed. Having only one server in the network is not enough to complete this setup successfully.
