A DAG network is a collection of one or more subnets used for either replication traffic or MAPI traffic. Each DAG contains a maximum of one MAPI network and zero or more replication networks. In a single network adapter configuration, the network is used for both MAPI and replication traffic. Although a single network adapter and path is supported, we recommend that each DAG have a minimum of two DAG networks. In a two-network configuration, one network is typically dedicated for replication traffic, and the other network is used primarily for MAPI traffic. You can also add network adapters to each DAG member and configure additional DAG networks as replication networks. To learn how to create a new Database Availability Group network, proceed with the following steps...
  • Begin first by selecting a Database Availability Group. In this case we will select the new Database Availability Group we just created, 'DAG1'.
  • Then select the option New Database Availability Group Network' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • First, we need to provide a unique name for the DAG network (a name that doesn't conflict with the name of any other DAG network). The limit is 128 characters.
  • In this case, we will name our database availability group network 'Dag1 Network'.
  • Next, we can provide an optional description for the DAG network. The limit is 256 characters.
  • Click 'Add' to add each network subnet to the DAG network. Subnets should be entered using a format of IP address/bitmask (for example, 192.168.1.0/24 for IPv4 subnets; 2001:DB8:0:C000::/54 for IPv6 subnets). If you add a subnet that's currently associated with another DAG network, the subnet will be removed from the other DAG network and associated with the network being created.
  • After the 'Add' button is pressed, the network subnet is to be entered here.
  • In this case, we will enter '192.168.0.100'.
  • Then press 'OK'.
  • You can see here that one network subnet has been added to the DAG network
  • Select this check box to enable the DAG network for use by replication. When a DAG network is enabled for replication, MAPI (Messaging Application Program Interface) traffic is restricted on that network. Clear this check box to prevent replication from using the DAG network and to enable MAPI traffic on that network. MAPI is a Microsoft Windows program interface that enables you to send  email from within a Windows application and attach a document to the e-mail.
  • Click 'New' to create the new database availability group network
  • In this case the wizard could not be completed. Having only one server in the network is not enough to complete this setup successfully.
