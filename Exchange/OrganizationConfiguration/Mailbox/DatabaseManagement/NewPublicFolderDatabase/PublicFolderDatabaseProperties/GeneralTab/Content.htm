Use the General tab to view or configure the properties of a public folder database, to change its name, and to customize its maintenance schedule. To learn how to do this, proceed with the following steps...
  • This unlabeled box at the top of the tab displays the name of the public folder database. You can modify this name.
  • This read-only field displays the full path to the Exchange database (.edb) file for the selected public folder database. To view the entire path, you may have to click the path and use the RIGHT ARROW key.
  • This read-only field displays the date and time of the last complete backup of the public folder database.
  • This read-only field displays the date and time of the last incremental backup of the public folder database.
  • This read-only field displays whether the public folder database is mounted or dismounted.
  • This read-only field displays which server the database is mounted on.
  • This read-only field displays the last date and time that the public folder database was modified.
  • Use this dropdown to select one of the preset maintenance schedules.
  • Once the dropdown is selected, you can choose from the following options:
    • Run daily from 11:00 P.M. to 3:00 A.M.
    • Run daily from midnight to 4:00 A.M.
    • Run daily from 1:00 A.M. to 5:00 A.M.
    • Run daily from 2:00 A.M. to 6:00 A.M.
    • Use Custom Schedule
  • Or you can configure a custom time, to do this, click 'Customize'.
  • Then use the 'Select Schedule' dialog box to configure a customized time.
  • Once a customized time is configured, click 'OK'.
  • Select this check box to enable background database maintenance. If you select this check box, the Extensible Storage Engine (ESE) performs the database maintenance, and the public folder database reads the object during database mount and initializes the database to perform the background database maintenance. If you don't select this check box, the public folder database reads the object during database mount and initializes the database without the option to perform the background database maintenance.
  • Select this check box to prevent Exchange from mounting this public folder database when it starts.
  • Select this check box to allow the public folder database to be overwritten during a restore process.
  • Click this check box to enable circular logging. Circular logging overwrites and reuses a single log file after the data it contains has been written to the database. Circular logging is disabled by default. By enabling circular logging, you reduce drive storage space requirements. However, you can't recover anything more recent than the last full (normal) backup because the transaction logs no longer contain all the transactions that were completed since the last backup. Therefore, in a normal production environment, circular logging isn't recommended.
