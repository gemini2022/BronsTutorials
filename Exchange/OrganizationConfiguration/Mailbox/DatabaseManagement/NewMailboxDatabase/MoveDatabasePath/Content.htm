You can move the mailbox database path on a server that is running Microsoft Exchange Server. To perform the move operation, you must temporarily dismount the database. This makes the database inaccessible to all users. If the database is currently dismounted, it isn't remounted upon completion. To learn how to move a mailbox database, proceed with the following steps...
  • Begin first by selecting a mailbox database. In this case we will select the new mailbox database we just created, 'MailboxDatabase2'.
  • Then select the option 'Move Database Path' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • This read-only field displays the name of the database for which you are going to perform the move operation on.
  • Next, you can change the location for the database file path by editing the 'Database file path' text field.
  • And you can change the location for the log folder path by editing the 'Log folder path' text field.
  • Once the paths have been updated, press 'Move'.
  • After you press 'Move', a dialog box will appear informing you that the mailbox database will be temporarily dismounted. Press 'Yes' to perform the move operation.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
