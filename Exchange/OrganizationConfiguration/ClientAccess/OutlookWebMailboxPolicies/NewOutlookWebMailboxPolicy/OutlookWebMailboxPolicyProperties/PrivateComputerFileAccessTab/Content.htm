On the Private Computer File Access tab, configure the file access and viewing options available if users select This is a private computer while they're signing in to Outlook Web App, or if users sign in using an authentication method other than forms-based authentication. File access lets users open any file attached to an e-mail message and files available through Windows file shares and Windows SharePoint Services document libraries. To learn about the Private Computer File Access tab, proceed with the following steps...
  • Select this check box if you want to enable direct file access. Direct file access lets users open files attached to e-mail messages.
  • After you select the 'Enable direct file access' option, you can then click the 'Customize' button to customize the direct file access settings.
  • Then click the 'Allow' button to specify which types of files should always be allowed. The Allow list overrides the Block list and the Force Save list.
  • Here you can enter the file extensions you want to allow.
  • This section displays all the extensions that are allowed.
  • Here you can enter the MIME type that you want to allow.
  • This section displays all the MIME types that are allowed.
  • Click 'OK'.
  • Or yo can click the 'Block' button to specify which types of files should be blocked. The Block list overrides the Force Save list and is overridden by the Allow list.
  • Here you can enter the file extensions you want to block.
  • This section displays all the extensions that are blocked.
  • Here you can enter the MIME type that you want to block.
  • This section displays all the MIME types that are blocked.
  • Click 'OK'.
  • And then finally, you can click the 'Force Save' button to specify which types of files the user must save to disk before opening them. The Force Save list is overridden by the Allow and Block lists.
  • Here you can enter the file extensions you want to force save.
  • This section displays all the extensions that are force saved.
  • Here you can enter the MIME type that you want to force save.
  • This section displays all the MIME types that are force saved.
  • Click 'OK'.
  • This dropdown is used to specify how Outlook Web App will handle unknown files that aren't in the Allow, Block, or Force Save lists.
  • Once the dropdown is selected, you can choose from the following options:
    • Allow
    • Force Save
    • Block
  • Click 'OK'.
  • WebReady Document Viewing enables Outlook Web App documents to be converted to HTML and displayed in a Web browser. Select this check box if you want to enable supported documents to be converted to HTML and displayed in a Web browser.
  • Select this check box if you want to force documents to be converted to HTML and displayed in a Web browser before users can open them in the viewing application. Documents can be opened in the viewing application only if Direct File Access is enabled.
  • After you select the option 'Enable WebReady Document Viewing', you can then click the 'Supported' button to select supported document types for WebReady Document Viewing.
  • To allow all supported document types to be viewed from an Internet browser, select this option 'All supported document types'.
  • Or to allow only specific document types to be viewed, select this option 'Specific document types'.
  • This section displays all the extensions that can be viewed from an internet browser.
  • This section displays all the MIME types that can be viewed from an internet browser.
  • Click 'OK'.
