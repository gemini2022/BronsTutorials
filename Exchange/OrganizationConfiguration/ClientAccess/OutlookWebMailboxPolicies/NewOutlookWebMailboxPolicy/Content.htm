Outlook Web App Policies are the set features created to allow or restrict the features of Outlook Web App. You can create an Outlook Web App mailbox policy to apply a common set of policy settings, such as attachment settings. Outlook Web App mailbox policies are useful for applying and standardizing settings for specific groups of users. When the Client Access server role is installed, a default Outlook Web App mailbox policy is created. You can create additional Outlook Web App mailbox policies as needed. You apply Outlook Web App mailbox policies by changing the properties of mailboxes. To learn how to create a new Outlook Web App Mailbox Policy, proceed with the following steps...
  • Begin by selecting the option 'New Outlook Web App Mailbox Policy' found in the Actions pane located at the far right side of the Exchange Management Console window.

  • Use this text box to specify a unique name for the Outlook Web App mailbox policy. This is a display name that appears in the Exchange Management Console. The Outlook Web App mailbox policy name is required, but it is used for display only. Because your organization may use multiple Outlook Web App mailbox policies, it is recommend that you use meaningful names for your Outlook Web App mailbox policies. The maximum length of an Outlook Web App mailbox policy name is 64 characters, and it can include spaces. However, it can't include any of the following characters: " / \ [ ] : ; | = , + * ? < >.
  • In this case, we will name our OutlookWeb App mailbox policy 'OutlookWebAppMailboxPolicy'.
  • Next is a list of features of the OutlookWeb App mailbox policy that can either be enabled or disabled. When a feature is enabled, that feature of Outlook Web App is allowed, and when a feature is disabled, that feature of Outlook Web App is restricted.
  • For example, if we select the feature 'Contacts'.
  • We can then change this feature in Outlook Web App from being allowed to restricted.
  • With 'Contacts' being selected and then clicking on the 'Disable' button.
  • The 'Contacts' feature then becomes disabled, which restricts this feature from being used in Outlook Web App.
  • In this case, we want the 'Contacts' feature to be allowed in Outlook Web App, therefore with this feature still selected, we will click the 'Enable' button.
  • And enable this feature so that it can be used in Outlook Web App.
  • Once the features of this policy have been set, click 'New' to create the policy.
  • On this screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
