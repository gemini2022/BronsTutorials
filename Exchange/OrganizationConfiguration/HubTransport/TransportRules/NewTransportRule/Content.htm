Transport rules allow you to apply messaging policies to messages in the transport pipeline. Actions such as redirecting a message or adding recipients, rights-protecting messages, and rejecting or silently deleting a message can be taken on messages that match the conditions and none of the exceptions defined in the rule. Transport rules can be created on a Hub Transport or an Edge Transport server. Both server roles have many common predicates and actions, but some predicates and actions are exclusive to each transport server role. To learn how to create a new Transport Rule, proceed with the following steps...
  • Begin by selecting the option 'New Transport Rule' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Type a name for the transport rule.
  • In this case, the name we will enter for the tranport rue will be 'TransportRule'.
  • You can use this field to describe the rule's functionality and relevant details such as a change request or trouble ticket number, date, and name of administrator. Text in the comments field has no impact on rule functionality.
  • New rules are enabled by default. If you want the rule to be created in a disabled state, clear the check box.
  • Click 'Next >' to continue.
  • On this screen of the wizard is a list of conditions that can be applied to the rule. If you want this rule to be applied to all e-mail messages, don't select any conditions in this step.
  • In this case, we will select the condition 'from people'.
  • If a condition is selected, the next step in the proccess is to add values to that condition. To do this, click a blue underlined word, which in this case is 'people'.
  • Once a blue underlined word is clicked, a dialog box will appear where you can add values to the condition. To do this, begin by clicking 'Add'.
  • Once the dialog box is open, select a value. In this case there is only one value, which is already selected by default.
  • Click 'OK'.
  • Once 'OK' is clicked, the value will be displayed here.
  • Click 'OK'.
  • You can see now that instead of the underlined word reading 'people', the underline word now reads the new value that we just entered.
  • Click 'Next >'.
  • On this screen, we need to define the actions of the transport rule. Actions in transport rules specify what you want to do to messages that match conditions of the rule.
  • In this case, we will select the action, 'prepend message subject with string'.
  • Now that we have an action selected, the next step in the proccess is to add values to that action. To do this, click a blue underlined word, which in this case is 'string'.
  • Once the underlined word is clicked, a dialog box appears in which you can specify the values for the action. To begin, we first need to enter a value in the text box.
  • In this case, we will enter the value, 'I am the administrator'.
  • Once a value is entered, click 'OK'.
  • You can see now that instead of the underlined word reading 'string', the underline word now reads the new value that we just entered.
  • Click 'Next >' to continue.
  • On this screen, we have the option of defining exceptions for the transport rule. Exceptions identify the messages that the actions shouldn't apply to. The same message identifiers that are available in conditions are also available in exceptions. Exceptions override conditions and prevent the rule actions from being applied to a message, even if the message matches all of the configured conditions.
  • In this case, we will select the exception, 'except when the message is sent to people'.
  • Now that we have an exception selected, the next step in the proccess is to add values to that exception. To do this, click a blue underlined word, which in this case is 'people'.
  • Once a blue underlined word is clicked, a dialog box will appear where you can add values to the exception. To do this, begin by clicking 'Add'.
  • Once the dialog box is open, select a value. In this case there is only one value, which is already selected by default.
  • Click 'OK'.
  • Once 'OK' is clicked, the value will be displayed here.
  • Click 'OK'.
  • You can see now that instead of the underlined word reading 'people', the underline word now reads the new value that we just entered.
  • Click 'Next >'.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
