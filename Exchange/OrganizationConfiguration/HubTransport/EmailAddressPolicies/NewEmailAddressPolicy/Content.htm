Recipients (which include users, resources, contacts, and groups) are any mail-enabled object in Active Directory to which Microsoft Exchange can deliver or route messages. For a recipient to send or receive email messages, the recipient must have an email address. Email address policies generate the primary and secondary email addresses for your recipients so they can receive and send email. By default, Exchange contains an email address policy for every mail-enabled user. This default policy specifies the recipient's alias as the local part of the email address and uses the default accepted domain. The local part of an email address is the name that appears before the at sign (@). However, you can change how your recipients' email addresses will display. For example, you can specify that the addresses display as firstname.lastname@contoso.com. Furthermore, if you want to specify additional email addresses for all recipients or just a subset, you can modify the default policy or create additional policies. For example, the user mailbox for David Hamilton can receive email messages addressed to hdavid@mail.contoso.com and hamilton.david@mail.contoso.com. To learn how to create a new E-mail Address Policy, proceed with the following steps...
  • Begin by selecting the option 'New E-mail Address Policy' found in the Actions pane located at the far right side of the Exchange Management Console window.
  • Use this text box to create the display name for the new e-mail address policy. The name can contain as many as 64 characters but cannot include wildcard characters.
  • In this case we will name the new email address policy 'Email AddressPolicy'.
  • Next, we need to select an organizational unit where the recipients we want the e-mail address policy to affect reside. To do this, begin by first clicking 'Browse' to open the 'Select Organizational Unit' dialog box.
  • Once the 'Select Organizational Unit' dialog box is open, we need to select an organizational unit where the recipients we want the e-mail address policy to affect reside.
  • In this case we will select the root OU.
  • Click 'OK'.
  • Next, we need to select the type of recipients that will be affected by this policy. This first option specifies that all recipient types will be affected.
  • Or we have the option of selecting specific types of recipients.
  • Select this check box if you want your e-mail address policy to apply to users with Exchange mailboxes. Users with Exchange mailboxes are those that have a user domain account and a mailbox in the Exchange organization.
  • Select this check box if you want your e-mail address policy to apply to users with external e-mail addresses. Users with external e-mail accounts have user domain accounts in Active Directory, but use e-mail accounts that are external to the organization. This enables them to be included in the global address list (GAL) and added to distribution lists.
  • Select this check box if you want your e-mail address policy to apply to Exchange resource mailboxes. Resources mailboxes allow you to administer company resources through a mailbox, such as a conference room or company vehicle.
  • Select this check box if you want your e-mail address policy to apply to contacts with external e-mail addresses. Contacts with external e-mail accounts do not have user domain accounts in Active Directory, but the external e-mail address is available in the GAL.
  • And then finally, select this check box if you want your e-mail address policy to apply to security groups or distribution groups that have been mail-enabled. Mail-enabled groups are similar to distribution groups. E-mail messages that are sent to a mail-enabled group account will be delivered to several recipients.
  • In this case, we will select the option 'All recipient types'.
  • Click 'Next >'.
  • Next, we have the option of selecting conditions for our policy. However, if we don't want to set any conditions, we can simply not select any of the check boxes. The first condition that can be set is 'Recipient is in a State or Province'. Select this check box if you want the e-mail address policy to include only recipients from specific states or provinces. This information is contained on the Address and Phone tab in the recipient's properties.
  • The next condition is 'Recipient is in a Department'. Select this check box if you want the e-mail address policy to include only recipients in specific departments. This information is contained on the Organization tab in the recipient's properties.
  • The next condition is 'Recipient is in a Company'. Select this check box if you want the e-mail address policy to include only recipients in specific companies. This information is contained on the Organization tab in the recipient's properties.
  • And then finally, there are 15 custom attributes for each recipient. There is a separate condition for each custom attribute. If you want the e-mail address policy to include only recipients that have a specific value set for a specific custom attribute, select the check box that corresponds to that custom attribute.
  • In this case, we will select the condition 'Recipient is in a State or Province'.
  • After a condition is selected, we then need to click the underlined term to create your condition. By default, the underlined term for new conditions will read 'specified'. After you edit the condition, the underlined term will change to the value that you specified.
  • Once the underlined term is clicked, a dialog box appears in which you can specify the values for the condition. To begin, we first need to enter a value in the text box.
  • In this case, we will enter the value, 'NY'.
  • Once a value is entered, click 'Add'.
  • After clicking 'Add', the value will appear here. You can add more than one value, but you cannot enter duplicate values.
  • Click 'OK'.
  • You can see now that instead of the underlined term reading 'specified', the underline term now reads the new value that we just entered.
  • Click 'Next >' to continue.
  • Next, we need to specify an e-mail address for our e-mail address policy. To do this, begin by first clicking 'Add' to open the 'SMTP E-mail Address' dialog box.
  • Once the 'SMTP E-mail Address' dialog box is open, we then have the option of selecting the option 'E-mail address local part'.
  • If selected, we then can choose the corresponding options to configure how the local part of the SMTP e-mail address will appear. The local part of an e-mail address is the name that appears before the at sign (@). If you clear the check box, the local part uses the recipient's alias.
  • In this case, we will leave the the default option selected, 'Use alias'.
  • Next, with this option 'Select the accepted domain for the e-mail address', we can select the e-mail address domain to which this e-mail address policy applies.
  • Once this option is selected, click 'Browse'.
  • Then select the domain to which this e-mail address policy applies.
  • In this case, we will canel out of this option.
  • And instead, select the option 'Specify the custom fully qualified domain name (FQDN) for the e-mail address'.
  • Once this option is selected, then type the FQDN for the domain part of the e-mail address that appears after the at sign (@). This FQDN must match an accepted domain.
  • Click 'OK'.
  • After pressing 'OK' on the 'SMTP E-mail Address' dialog box, the email address you defined will be displayed here.
  • Click 'Next >' to move on.
  • On this screen of the wizard, we need to specify when the e-mail address policy should be applied. This first option, 'Do not apply', means that after the policy is created, the policy will not be applied to the mailboxes until the policy is applied manually at a later time.
  • Select this option to apply the e-mail address policy as soon as the e-mail address policy is created.
  • Select this option to specify a time when to apply the new e-mail address policy.
  • Then select the dropdown.
  • Once the dropdown is selected, a calendar will appear where you can set the date when the new e-mail address policy will be applied.
  • Then you can set the time when the new e-mail address policy will be applied.
  • In this case, we will select the option 'Immediately', so that the new e-mail address policy will be applied as soon as the policy is created.
  • Not only do you have the option to define when the new e-mail address policy will be applied, but you can also define how long the e-mail address policy will be applied for. To do this, select the option 'Cancel tasks that are still running after (hours)'.
  • Then enter in how long the e-mail policy will run for. The default is 8 hours.
  • Click 'Next >' to move on.
  • On this screen, review the 'Configuration Summary' and verify that the information you entered is correct. If you are not satisfied with any of the information, you may go back and make any changes necessary.
  • If the information in the 'Configuration Summary' is correct, press 'New'.
  • And then finally, on this last screen, you have the option of reviewing the associated shell command that was performed by this wizard.
  • Press 'Finish' to complete the wizard.
