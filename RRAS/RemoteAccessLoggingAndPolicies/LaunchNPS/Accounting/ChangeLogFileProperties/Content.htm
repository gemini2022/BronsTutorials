To learn how to change the log file properties, proceed with the following steps...
  • Begin first by selecting the 'Change Log File Properties' link located in the window to the right.
  • Once in the 'Log File Properties', on the first tab, you can change the type of information that can be logged. First is the option to log 'Accounting Requests'.
  • We can also choose to log 'Authentication requests'.
  • Next, we have the option to log 'Periodic accounting status'.
  • Then we have the option to log 'Periodic authentication status'.
  • And then finally, you have the option to discard connection requests if for any reason logging fails due to network or other problems.
  • If we now select the 'Log File' tab.
  • This section displays the directory where the log file will be located. You can choose a different location if you wish or keep the default.
  • Here you can choose the type of format for the log file.
  • If we select the dropdown.
  • The first format option is 'DTS Compliant', which is the defalut selection.
  • Next is 'ODBC (Legacy)'.
  • And then finally, you have the option to have the format set as 'IAS (Legacy)'.
  • In this section of the 'Log File' tab, you have the option to choose when a new log file will be created. The first option is 'Daily'.
  • Or a new log file can be created 'Weekly'.
  • Or you can choose to have a new log file created 'Monthly'.
  • Or you can choose to never create a new log file.
  • Or, when the log file reaches a certain size, you can have a new log file created.
  • And then finally, when this last option is selected and the disk becomes full, older log files will get deleted.
