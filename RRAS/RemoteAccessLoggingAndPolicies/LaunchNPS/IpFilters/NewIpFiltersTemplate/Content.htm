You can use IP filters as part of a network policy or a connection request policy to control which input and output packets are allowed for both IPv4 and IPv6. You can configure filters individually in the policy, or create a template and apply the template. It makes sense in most cases to create an IP filter template because you can reuse the settings. To learn how to create an IP filter template, proceed with the following steps...
  • Begin by right-clicking on 'IP Filters'.
  • Then select 'New'.
  • Once 'New' is selected, the 'New IP Filters Template' dialog box will appear. First, we need to start by giving a name to our new IP Filter Template.
  • For this tutorial, the name we will give our new IP Filters template will be 'New IP Filter'.
  • With this IP filter template, we can conrol where the IP packets are allowed to be sent. To do this, click the 'Input Filters...' button.
  • Once we click the 'Input Filters...' button, we will be presented with a dialog box that will allow us to create the filter. To do this, press the 'New...' button to create a new filter.
  • First, we need to start by setting this new IP filter as enabled. To do this, enable the option 'Destination network'.
  • Then provide the address of the destination network.
  • And then its subnet mask.
  • Next, we can specify what protocol this filter will use. To do this, select the dropdown.
  • Once the dropdown is selected, you can choose one of the following options.
    • TCP
    • TCP [established]
    • UDP
    • ICMP
    • Any
    • Other
  • In this case, we will choose the option to use any protocol.

  • Press 'OK' to create the filter.
  • Now that the filter is created, we next have to decide whether this filter will be used to deny packets to this destination
  • Or allow packets to this destination.
  • In this case, we will leave the default option selected 'Do not permit packets listed below'.

  • Press 'OK'.
  • With this IP filter template we can also control what packets come in to the network. To do this click the 'Output Filters...' button.
  • Similar from before, once we click the 'Output Filters...' button, we will be presented with a dialog box that will allow us to create the filter. To do this, press the 'New...' button to create a new filter.
  • This time we need to enable the option 'Source network'.
  • Then provide the address of the source network.
  • And then its subnet mask.
  • Like the input filter, we can specify what protocol this filter will use as well. To do this, select the dropdown.
  • Once the dropdown is selected, you have the option to choose the same following options.
    • TCP
    • TCP [established]
    • UDP
    • ICMP
    • Any
    • Other
  • In this case, we will again choose the option to use any protocol.

  • Press 'OK' to create the filter.
  • And like before, we will leave the default option selected 'Do not permit packets listed below'.

  • Press 'OK'.
  • And then finally, the same in and out filters can be created for IPv6 as well.

  • Press 'OK'.
