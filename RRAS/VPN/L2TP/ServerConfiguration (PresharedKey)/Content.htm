To learn how to configure the Routing and Remote Access server for VPN connections using the L2TP protocol with a preshared key, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Routing and Remote Access'...
  • Once in Routing and Remote Acces, right-click the server node, which in this case is 'RRASSERVER (local)'.
  • Then select 'Properties'.
  • Once in the RRASSERVER properties, select the 'Security' tab.
  • In the Security tab, select the option 'Allow custom IPsec policy for L2TP connection'.
  • Once this option is selected, the input field labeled 'Preshared Key' will be enabled. It is here where a preshared key for the L2TP IPsec policy is to be entered. Pre-shared means the parties must agree on a shared, secret key that becomes part of the IPSec policy. During security negotiation, information is encrypted before transmission using the shared key, and decrypted on the receiving end using the same key. If the receiver can decrypt the information, identities are considered authenticated.
  • In this case, the preshared key that we will enter will be 'Preshared Key'.
  • After the preshared key has been entered, press 'OK'.
  • Once you press 'OK', a dialog box will appear stating that the Routing and Remote Access service must be restarted in order for the changes that you just made to take effect.

  • Press 'OK'.
  • To restart the Routing and Remote Acces service, go back to the server node, which in this case is 'RRASSERVER (local)', and right-click it.
  • Then go to 'All Tasks'.
  • Then select the option 'Restart'.
  • Once you select the 'Restart' option, the restarting process of the Routing and Remote Access service will begin.
