To learn how to configure the Routing and Remote Access server for VPN connections using the L2TP protocol with certificate authentication, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Routing and Remote Access'...
  • Once in Routing and Remote Acces, right-click the server node, which in this case is 'RRASSERVER (local)'.
  • Then select 'Properties'.
  • Once in the RRASSERVER properties, select the 'Security' tab.
  • In the Security tab, if the option 'Allow custom IPsec policy for L2TP connection' is selected, then in order for a L2TP VPN connection using certificates can be used, then this option must be disabled.
  • To disable this option, if it's already been enabled, simply uncheck the checkbox.
  • Once this option is unchecked, the preshared key will be cleared and the input field will become disabled, indicating that if a VPN connection is made using the L2TP protocol, then certificate authentication must be used.

  • Press 'OK'.
  • Once you press 'OK', a dialog box will appear stating that the Routing and Remote Access service must be restarted in order for the changes that you just made to take effect and that because you are disabling the custom IPsec policy for L2TP connections option, you should verify that a server certificate is installed for L2TP connections to succeed.

  • Press 'OK'.
  • To restart the Routing and Remote Acces service, go back to the server node, which in this case is 'RRASSERVER (local)', and right-click it.
  • Then go to 'All Tasks'.
  • Then select the option 'Restart'.
  • Once you select the 'Restart' option, the restarting process of the Routing and Remote Access service will begin.
