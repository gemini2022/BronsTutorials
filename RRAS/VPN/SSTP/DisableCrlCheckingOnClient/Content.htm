The SSL VPN client needs to be able to download the CRL to confirm that the server certificate on the VPN server has not been revoked. You must either publish the CRL on a server that is accessible on the Internet or configure the client to not require CRL checking. Because it exposes a private server name to the Internet, you can disable CRL Checking on the VPN Client in the registry. To learn how to do this, proceed with the following steps...
  • Begin first by logging into a Windows client computer and then selecting 'Start' or the 'Start Orb' at the bottom left corner of your screen.
  • Then type 'regedit' in the search field.
  • And select the 'regedit' icon.
  • Once the 'Registry Editor' is open, expand 'HKEY_LOCAL_MACHINE'.
  • Then expand 'SYSTEM'.
  • Then 'CurrentControlSet'.
  • Expand 'services'.
  • Frome here, you may need to scroll down a bit.
  • Scroll down until you find 'SstpSvc', then expand it.
  • Once 'SstpSvc' is expanded, right-click on 'Parameters'.

 
  • And select 'New'.
  • And then 'DWORD (32-bit) Value'.
  • Once the new registry value is created, a name needs to be entered.
  • In this case, the name needs to be 'NoCertRevocationCheck'.
  • After the name has been assigned, right-click it.
  • And select 'Modify...'.
  • Once 'Modify...' is selected, we then need to modify the value of the new registry value.
  • In this case, the value we need to enter is a value of '1'.
  • After the value is entered, press 'OK'.
