A certificate infrastructure is a requirement for VPN connections based on Secure Socket Tunneling Protocol (SSTP). Certificates provide stronger authentication security than password-based authentication does. To learn how to install a certificate on a VPN server, proceed with the following steps...
  • Before we do any configuring or installation on the VPN server, first log into the server that's running the Certification Authority service and then go to 'Start' at the bottom left corner of your screen.
  • And then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, expand the root node for the CA, which in this case is 'EnterpirseCA'.
  • Then right-click 'Certificate Templates'.
  • And select 'Manage'.
  • Now, we need to find the appropriate certificate that we will use for the VPN server. The certificate you install on the VPN server must have either the Server Authentication or All-Purpose enhanced key usage (EKU) property. This certificate is used by the VPN client to authenticate the VPN server when the session is established. In this case, we will choose the certificate 'RAS and IAS Server', which has the Server Authentication property. Once this certificate is located, right-click it.
  • And select 'Properties'.
  • Once in the properties of the 'RAS and IAS Server' certificate, select the 'Security' tab.
  • It is here where we need to make sure that our VPN server is eligible to enroll for this certificate. By default, this certificate already contains a group called 'RAS and IAS Servers'. Our VPN server is already a member of this group, therefore as long as this group is eligible to enroll for this certificate, then enrolling for this certificate should be succesful.
  • To make sure that this group is eligible for enrollment, select it.
  • And make sure that the permission for 'Enroll' is set to 'Allow', which in this case it is.
  • Next, select the 'Subject Name' tab.
  • Instead of having the subject name of the certificate built from Active Directory.
  • Select the option 'Supply in the request'. This will allow when enrolling for the certificate to manually provide the subject name of the certificate. Doing this will demonstrate what the subject name needs to be and why.
  • Once the option 'Supply in the request' is selected, we immediately get a dialogue box warning us that this option may create a security risk and is not recommended. This is because with this option, it allows a client to submit a certificate request using any subject name and does not require approval by a certificate manager.

  • Press 'OK'.
  • Press 'OK' to commit to the changes.
  • Then close the 'Certificate Templates Console'.
  • Now that the certificate is configured, we need to issue the certificate so it can be used. To do this, right-click 'Certificate Templates'.
  • Then select 'New'.
  • And then 'Certificate Template to Issue'.
  • After selecting the option 'Certificate Template to Issue', a list of the preconfigured certificate templates will appear. Among this list, is the certificate template we edited earlier 'RAS and IAS Server'. In this case, to get to it, we need to scroll down the list.
  • Once the template is located, select it.
  • Then press 'OK'.
  • And now you can see our certificate template 'RAS and IAS Server' is in the list and ready to use.
  • Now that the Server Authentication Certificate is set, the next step is to have it installed on the VPN server. To do this, jump over to the VPN server and select 'Start' at the bottom left corner of the screen.
  • Then, in the search field, type 'mmc'.
  • After typing 'mmc' in the search field, select the 'mmc' icon located at the top of the menu. This will open up the MMC Console.
  • Once the MMC Console is open, select 'File'.
  • Then select 'Add/Remove Snap-in...'.
  • From here, we need to add the Certificates snap-in to the MMC Console. To do this, select 'Certificates'.
  • Then press the 'Add >' button.
  • Once we press the 'Add' button, we next need to choose what type of account we want to manage the certificates for. In this case, we want to choose the option 'Computer account'.
  • Press 'Next >'.
  • Then we need to specify which computer this snap-in will manage, this computer or another computer. In this case, we want manage this computer so we will leave the default option selected 'Local computer'.

  • Press 'Finish'.
  • Now press 'OK' to create the snap-in.
  • Now that the Certificates snap-in is added to the MMC Console, we can now use the Certificates snap-in to request the Server Authentication Certificate we need and then install it. To do this, expand 'Certificates'.
  • Then right-click 'Personal'.
  • And go to 'All Tasks'.
  • Then 'Request New Certificate...', which will launch the 'Certificate Enrollment' wizard.
  • Once the 'Certificate Enrollment' wizard is open, press 'Next' to begin the certificate enrollment process.
  • Make sure 'Active Directory Enrollment Policy' is selected, and then press 'Next'.
  • On this next screen is a list of all the certificates that we can enroll for. In this case, the one that we want is 'RAS and IAS Server'.
  • To enroll for the RAS and IAS Server certificate, select it.
  • However, before we can enroll for this certificate, there are additional settings that need to be configured.
  • To configure these addtional settings, select the link 'More information is required to enroll for this certificate. Click here to configure settings'.
  • If you recall, earlier when we configured the RAS and IAS Server certificate template, we chose the option to supply the subject name in the request. This is the part where we are doing just that. To do this, begin by selecting the 'Type' dropdown under the section 'Subject name'.
  • Then select the option 'Common name'.
  • Once the 'Common name' option is selected, we then need to provide the common name in the 'Value' field below.
  • The common name identifies the host name associated with the certificate. So in this case, we would enter in the value of 'rrasserver.bronstutorials.com'. However, if we had configured the RAS and IAS Server certificate template to build from Active Directory, then the common name would have been automatically configured as 'rrasserver.bronstutorials.local'. This is because '.local' is the internal DNS namespace used within this orgainization. But because this VPN server will have a publicly resolvable host name of 'rrasserver.bronstutorials.com', then 'rrasserver.bronstutorials.com' is what we will use for the common name.
  • Once the common name has been entered, press 'add >'.
  • Pressing 'add >' will add the common name value to the text area located at the far right of the form.

  • Press 'OK'.
  • Now that the proper certificate has been selected and all the required information has been entered, we can now enroll for the certificate. To enroll for the certificate, press the 'Enroll' button.
  • Once the 'Enroll' button is pressed, the Certificate Enrollment wizard will begin the enrollment process.
  • Once the certificate has been successfully enrolled, press the 'Finish' button.
