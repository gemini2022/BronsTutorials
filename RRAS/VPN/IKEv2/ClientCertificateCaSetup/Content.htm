To learn about what certificate to use for a IKEv2 VPN client and how to set it up, proceed with the following steps...
  • Before we do any configuring or installation on the VPN client, we first need to choose the certificate that will be used for client authentication. To do this, first log onto the server that's running the Certification Authority service and then go to 'Start' at the bottom left corner of your screen.
  • And then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, expand the root node for the CA, which in this case is 'EnterpirseCA'.
  • Then right-click 'Certificate Templates'.
  • And select 'Manage'.
  • Now, we need to find the appropriate certificate that we will use for the VPN client. In this case, we will choose the certificate 'Workstation Authentication'. Once this certificate is located, right-click it.
  • And select 'Properties'.
  • Once in the properties of the 'Workstation Authentication' certificate, select the 'Security' tab.
  • It is here where we need to make sure that the appropriate user or group is eligible to enroll for this certificate. In this case, even though this certificate is intended for a client computer, we are instead going to allow eligibility to our 'Authenticated Users'. This is the case because the enrollment process will take place via web enrollment, and it is only users who can obtain certificates by this process. Once the user obtains this certificate, the user will then export the certificate to the local machine certificate store.
  • In order for 'Authenticated Users' to be able to enroll for this certificate, the 'Enroll' permission has to be set to 'Allow'.
  • To do this, select the 'Allow' checkbox for the 'Enroll' permission.
  • Next, select the 'Subject Name' tab.
  • Instead of having the subject name of the certificate built from Active Directory.
  • Select the option 'Supply in the request'. This will allow when enrolling for the certificate to manually provide the subject name of the certificate. Doing this will demonstrate what the subject name needs to be and why.
  • Once the option 'Supply in the request' is selected, we immediately get a dialogue box warning us that this option may create a security risk and is not recommended. This is because with this option, it allows a client to submit a certificate request using any subject name and does not require approval by a certificate manager.

  • Press 'OK'.
  • Now, select the 'Request Handling' tab.
  • The only thing we need to do on this tab is select the option 'Allow private key to be exported'. When the user exports the certificate to the local machine certificate store, the private key needs to be exported with it.
  • Press 'OK' to commit to the changes.
  • Then close the 'Certificate Templates Console'.
  • Now that the certificate is configured, we need to issue the certificate so it can be used. To do this, right-click 'Certificate Templates'.
  • Then select 'New'.
  • And then 'Certificate Template to Issue'.
  • After selecting the option 'Certificate Template to Issue', a list of the preconfigured certificate templates will appear. Among this list, is the certificate template we edited earlier 'Workstation Authentication'. In this case, to get to it, we need to scroll down the list.
  • Once the template is located, select it.
  • Then press 'OK'.
  • And now you can see our certificate template 'Workstation Authentication' is in the list and ready to use.
