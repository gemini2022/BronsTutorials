To learn how to install a certificate on a VPN client, proceed with the following steps...
  • Begin first by logging into a computer that will be used as a VPN client and launch 'Internet Explorer'.
  • In this tutorial we will be using web enrollment to request the certificate that this computer will use for VPN client authentication. To start, we need to enter in the address in the address bar at the top of the browser window of the server that hosts the CA Web Enrollment pages.
  • Begin by typing 'https://'.
  • Followed by the address of the server that hosts the CA Web Enrollment pages, which in this case is 'enterpriseca.bronstutorials.com'.
  • And then followed by '/certsrv'.

  • Press 'Enter'.
  • Since this is a remote computer and is not part of the company domain, in order to gain access to the CA to request a certificate, cerdentials need to be provided.
  • In this tutorial our user John Doe will be requesting the client authentication certificate, so in this case, we will begin entering in the user name of 'jdoe'.
  • And then the password.

  • Press 'OK'.
  • Once in the web enrollment web site, begin by selecting the option 'Request a certificate'.
  • And then go to 'advanced certificate request'.
  • Then, 'Create and submit a request to this CA'.
  • Once we prepare to make a certificate request, a dialogue box will appear informing you that this web site is attempting to perform a digital certificate operation on your behalf and that you should only allow known web sites to perform digital certificate operations on your behalf. Since we know and trust this web site, we can move on.

  • Press 'Yes'.
  • We next have to choose the certificate we want to request. To do this, select the dropdown under 'Certificate Template'.
  • And then choose the certificate you want to request, which in this case is 'Workstation Authentication'.
  • Once the certificate we want to request is selected, we next have to enter in a name of the certificate in the input field labeled 'Name'.
  • The name of certificate needs to be the host name of the workstation we are installing this certificate on. In this case, the name of this workstation is 'ws1'.
  • Next, scroll down to the bottom of the page.
  • And make sure that the option 'Mark keys as exportable' is selected. This is because when the certificate gets installed, the certificate will automatically be placed in the 'User' side of the certificate store. However, this certificate will need to be in the 'Computer' side of the certificate store instead, therefore we will have to manually export the certificate to that location. When the export process takes place, it is important to make sure that the private key gets exported with it.
  • Press 'Submit >' to submit the certificate request.
  • Once you press the 'Submit >' button, a dialogue box will appear informing you that this web site is attempting to perform a digital certificate operation on your behalf and that you should only allow known web sites to perform digital certificate operations on your behalf. Since we know and trust this web site, we can move on.

  • Press 'Yes'.
  • As long as the certificate you are requesting is not configured for 'CA certificate manager approval', then once you request the certificate, it should automatically be issued to you. All you need to do at this point is install the certificate. To do this, select the option 'Install this certificate'.
  • After selecting the option 'Install this certificate', the certificate will be installed. However, as stated earlier, when the certificate gets installed, the certificate will automatically be placed in the 'User' side of the certificate store. Therefore, we next need to move the certificate from the 'User' side of the certificate store to the 'Computer' side of the certificate store.
  • To do this, go to the desktop and select 'Start' or the 'Start Orb' at the bottom left corner of your screen.
  • Then type 'mmc' in the search field.
  • And select the 'mmc' icon.
  • Once the 'mmc' icon is selected, the MMC Console window will appear. From here, select 'File'.
  • And then select the option 'Add/Remove Snap-in...'. This will open the 'Add or Remove Snap-ins' screen.
  • Once in the 'Add or Remove Snap-ins' screen, we need to select the snap-in we want to add to the console. In this case, we want to select the snap-in 'Certificates'.
  • Once the 'Certificates' snap-in is selected, press the 'Add >' button.
  • Once we press the 'Add' button, we next need to choose what type of account we want to manage the certificates with. In this case, we want to manage both the user account and the computer account, but we can only choose one at a time, so we will accept the default selection 'My user account'.

  • Press 'Finish'.

 
  • You can see here that the user account is set to be added to the console, but we also need the computer account added as well.
  • To also add the computer account, press the 'Add >' button again.
  • And this time, select the option 'Computer account'.
  • Then press 'Next >'.
  • Then we need to specify which computer this snap-in will manage. Will this snap-in manage this computer, or will it manage another computer on the network? In this case, we want to manage this computer, so we will leave the default option selected 'Local computer'.

  • Press 'Finish'.
  • Now that both the user account and the computer account is set to be added to the console, press 'OK'.
  • You can see here that both the user and the computer certificate snap-ins are now added to the console.
  • We next need to find the 'Workstation Authentication' certificate that was installed in the user certificates store. To do this, expand 'Certificates - Current User'.
  • Then expand 'Personal'.
  • And select the 'Certificates' folder.
  • It is here where our 'Workstation Authentication' certificate that we installed is located.
  • However, this certificate needs to be located in the computer certificate store. To do this, right-click the certificate.
  • And go to 'All Tasks'.
  • And then select 'Export...'.
  • This will open the 'Certificate Export Wizard'.

  • Press 'Next >'.
  • On this screen, select the option 'Yes, export the private key'.
  • Press 'Next >'.
  • Next, on this screen, select the option 'Include all certificates in the certitication path if possible'. This will make sure that the root certificate gets installed in the 'Trusted Root Certification Authorities' folder if it's not already.
  • Press 'Next >'.
  • On this screen, because we're also exporting the private key with this certificate, we need to protect the private key by securing it with a password.
  • To do this, type a password in the input field labeled 'Password'.
  • Then type it again to confirm it.

  • Press 'Next >'.
  • Now on this screen, we need to specify the location where this certificate will get exported to. Ultimately, the certificate needs to be placed in the computer certificate store, but we first need to export it somewhere else and then import it into the computer certificate store from that location. To choose a location for the certificate, press the 'Browse...' button.
  • In this case, the location that we will choose to export the certificate will be on the desktop.
  • Once the location has been chosen, we next have the give the exported certificate a name. In this case, we will name the certificate 'ClientCert'.
  • Then press 'Save'.
  • Now that the file has a location and a name, press 'Next >'.
  • And then finally, to export the certificate, press 'Finish'.
  • Once you press 'Finish', a dialog box will appear informing you that the export was successful.

  • Press 'OK'.
  • Now that the certificate has been exported, we next have to import the certificate to computer certificate store. To do this, expand 'Certificates (Local Computer).
  • Then from here, right-click the 'Personal' store.
  • And select 'All Tasks'.
  • Then 'Import'.
  • This will open the 'Certificate Import Wizard'.

  • Press 'Next >'.
  • On this screen, we need to find the certificate file we saved earlier. To do this, press the 'Browse...' button.
  • And make your way to the location where the file was saved. In this case, we saved the file on the desktop.
  • Once you get to the location where the file was saved, the file may be hidden due to the correct file extention not being selected. To fix this, select the dropdown.
  • And choose the correct file extention. In this case we want to choose 'Personal Information Exchange (*.pfx, *.p12)'.
  • Once the file is visible, select it.
  • And then press 'Open'.
  • Now that the certificate we want to import has been located, we can move on. Press 'Next >'.
  • Because we exported the private key with the certificate, we needed to protect the private key by securing it with a password. In order to successfully import the certificate, we need to provide that password here.
  • Enter the password, then press 'Next >'.
  • On this screen you have the option of specifing the certificate store where the certificate is to be placed. In this case, the default selection is the appropriate location.

  • Press 'Next >'.
  • And then finally, to import the certificate, press 'Finish'.
  • Once you press 'Finish', a dialog box will appear informing you that the import was successful.

  • Press 'OK'.
