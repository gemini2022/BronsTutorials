To learn about the PPP tab, proceed with the following steps...
  • This first option specifies whether the server uses the PPP Multilink Protocol (MP), which allows remote access clients and demand-dial routers to combine multiple physical connections into a single logical connection.
  • This next option specifies whether the server uses the Bandwidth Allocation Protocol (BAP) and Bandwidth Allocation Control Protocol (BACP) to negotiate the dynamic addition or deletion of multiple physical connections for remote access clients and demand-dial routers.
  • Next, this option specifies whether the server uses Link Control Protocol (LCP) extensions. Clearing this check box prevents LCP from sending Time-Remaining and Identification packets and requesting callback during LCP negotiation.
  • And then finally, this last option specifies whether the server uses the Microsoft Point-to-Point Compression Protocol (MPPC) to compress data sent on the remote access or demand-dial connection.
