To learn how to configure the Network Policy Server for RADIUS wireless connections, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Network Policy Server'...
  • Once in Network Policy Server, select the dropdown in the panel to the right of the window.
  • And select the option 'RADIUS server for 802.1X Wireless or Wired Connections'.
  • Then select the link 'Configure 802.1X'. This will launch the 'Configure 802.1X' wizard.
  • Once the 'Configure 802.1X' wizard has started, the first thing we need to do is select the option 'Secure Wireless Connections'.
  • Selecting this option will generate a name that will be used for each policy created with this wizard. You can keep this name or you can change it if you wish. In this case, we will leave the name as is and move on.
  • Press 'Next '.
  • On this next page of the wizard, we need to specify the access point that will be used as the RADIUS client. To do this, press the 'Add...' button.
  • Once the 'Add' button is pressed, the first thing we need to do is enter a name for our RADIUS client.
  • In this case, we will call our RADIUS client 'Access Point'.
  • Once the name has been established, we next have to enter in the IP address of the RADIUS client. In this case, the IP address is '192.168.1.1'.
  • We can then verify if the address is correct by pressing the 'Verify...' button.
  • And then pressing the 'Resolve' button.
  • If the address is able to be resolved, then it will be displayed here.
  • Press 'OK'.
  • Next, we have the option of selecting an existing Shared Secrets template, but in this case we don't have one configured, so we will move on.
  • Since we don't have a Shared Secrets template configured, we will enter a shared secret here.
  • The shared secret that we enter here must match the shared secret that we entered for the access point, which was in this case '1234'.
  • Then confirm it.
  • Press 'OK'.
  • Now that our RADIUS client has been configured, we can move on, press 'Next'.
  • On this screen of the wizared, we need to select the EAP type that will be used for the policy that this wizard will create. To do this, select the dropdown.
  • And select the option 'Microsoft: Protected EAP (PEAP)'.
  • Then press 'Next'.
  • On this screen, we have the option of specifying which group of users will be allowed or denied access to the network. If however no groups are specified, then this policy will apply to all users. In this case, we will not specify any group.
  • Press 'Next'.
  • On this screen we have the option of configuring traffic controls to control network traffic. In this case, we will not be configuring any traffic controls, so we will move on. Press 'Next'.
  • And then finally, on this screen, press 'Finish' to create the policy.
