To learn how to change the SQL Server logging properties, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Network Policy Server'...
  • Once in Network Policy Server, select 'Accounting'.
  • Then select the 'Change SQL Server Logging Properties' link located in the window to the right.
  • Once in the 'SQL Server Logging Properties', you can change the type of information that can be logged. First is the option to log 'Accounting Requests'.
  • We can also choose to log 'Authentication requests'.
  • Next, we have the option to log 'Periodic accounting status'.
  • Then we have the option to log 'Periodic authentication status'.
  • The maximum number of concurrent sessions can be set here.
  • The SQL Server Logging can be configured here.
  • Here you have the option to log to a text file if the SQL logging fails.
  • And then finally, you have the option to discard connection requests if for any reason logging fails due to network or other problems.
