To learn how to create a new Connection Request Policy, proceed with the following steps...
  • Begin first by right-clicking 'Connection Request Policies'.
  • And select 'New'.
  • The first thing we need to do is give a name to our policy.
  • In this case, we will name our policy 'Forward NAP Request'.
  • Next, we need to specify what server will be forwarding the NAP requests. To do this, select the dropdown.
  • And select the type of server that will be forwarding the NAP request, which in this case is 'DHCP Server'.
  • Press 'Next'.
  • Next, we need to specify the condition this policy will use to determine when the forwarding of the NAP request will take place. To add this condition, press the 'Add' button.
  • In this case, the type of condition we want to add is 'Identity Type'. To add this condition, we first need to scroll down.
  • And then once located, select it.
  • And press 'Add'.
  • We then need to select the identifying type, which in this case there is only one 'Machine health check'.
  • Once selected, press 'OK'.
  • You can see here that the condition for the policy has been added and that condition is Identity Type.
  • Press 'Next'.
  • Next, we need to specify where the NAP request will be forwarded to. In this case, we want to tell the wizard to forward the NAP request to our NAP server. To do this, select the option 'Forward reqeusts to the following remote RADIUS server group for authentication'.
  • Then select a RADIUS server group, which is in this case the only option that is available is the RADIUS server group we created earlier.
  • Press 'Next'.
  • On this screen, additional options can be configured. This is more for when you are using third party RADIUS servers. In this case, we are not using any third party RADIUS servers, so we will move on.

  • Press 'Next'.
  • The wizard is now complete, to create the policy, press 'Finish'.
