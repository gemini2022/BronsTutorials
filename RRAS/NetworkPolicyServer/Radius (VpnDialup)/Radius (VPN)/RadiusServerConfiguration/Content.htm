To learn how to configure the Network Policy Server for RADIUS VPN connections, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Network Policy Server'...
  • Once in Network Policy Server, select the dropdown in the panel to the right of the window.
  • And select the option 'RADIUS server for Dial-Up or VPN Connections'.
  • Then select the link 'Configure VPN or Dial-Up'. This will launch the 'Configure VPN or Dial-Up' wizard.
  • Once the 'Configure VPN or Dial-Up' wizard has started, the first thing we need to do is select the option 'Virtual Private Network (VPN) Connections'.
  • Selecting this option will generate a name that will be used for each policy created with this wizard. You can keep this name or you can change it if you wish. In this case, we will leave the name as is and move on.
  • Press 'Next '.
  • On this next page of the wizard, we need to specify the VPN server that will be used as the RADIUS client. To do this, press the 'Add...' button.
  • Once the 'Add' button is pressed, the first thing we need to do is enter a name for our RADIUS client.
  • In this case, we will call our RADIUS client 'Remote Access Server'.
  • Once the name has been established, we next have to enter in either the IP address of the RADIUS client or its actual name. In this case, we will enter in the name which is 'rrasserver'.
  • We can then verify it by pressing the 'Verify...' button.
  • And then pressing the 'Resolve' button.
  • If the address is able to be resolved, then it will be displayed here.
  • Press 'OK'.
  • Next, we have the option of selecting an existing Shared Secrets template, but in this case we don't have one configured, so we will move on.
  • Since we don't have a Shared Secrets template configured, we have the option of manually typing a shared secret or automatically having a shared secret generated.
  • If we select the 'Generate' option.
  • And then press the 'Generate' button.
  • Once the 'Generate' button is pressed, Windows will generate a rather long shared secret for you to use. This will give you greater security, but you will then need a way to copy this shared secret from this server to the client. Having this longer key means you now need some way of managing the key, which may mean storing the key on devices such as a USB stick or even emailing the key to transfer it between the two servers. Having a large key like this does impose more difficulty on the management of keys, but gives you much greater security.
  • A shorter key is often a lot easier to manage and can even be memorized. If we clear the generated key.
  • And select the option 'Manual'.
  • We can then manually type a shared secret here.
  • The shared secret that we type here, must be the same shared secret we enter later on the client.
  • You'll need to type the shared secret twice.
  • After the shared secret is entered, press 'OK'.
  • Now that we have our RAS server assigned as the RADIUS client, we can move on.
  • Press 'Next'.
  • On this screen we need to specify the type of authentication method that will be used with the RADIUS server. This first option 'Extensible Authentication Protocol', allows authentication to be handled by Smart Cards or certificates.
  • This option, 'Microsoft Encrypted Authentication version 2 (MS-CHAPv2)', is the default option and allows users to specify a password for authentication.
  • And then finally this option, 'Microsoft Encrypted Authentication (MS-CHAP)', is an option only if your network runs operating systems that do not support MS-CHAPv2.
  • Press 'Next'.
  • Here we can specify which user groups will be effected by this policy. If no groups are specified, then this policy will apply to all users. In this case, we will not specify any groups and move on.

  • Press 'Next'.
  • On this screen you can specify IP filters. In this case, we will not specify any filters and move on.

  • Press 'Next'.
  • On this screen you can specify the allowed encryption strengths used. In this case, we have all options selected, which means the strongest encryption will be used first. If the strongest encryption is not supported, then it will try to use the next strongest encryption.
  • Press 'Next'.
  • This screen deals with information concerning your ISP. In this case, there is nothing we need to configure here, so we will move on.

  • Press 'Next'.
  • And then finally, on this screen, press 'Finish' to create the policy.
