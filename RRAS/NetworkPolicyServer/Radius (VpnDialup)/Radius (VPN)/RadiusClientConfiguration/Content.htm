To learn how to configure the Routing and Remote Access server as a client for a RADIUS server, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Routing and Remote Access'...
  • Once in Routing and Remote Acces, right-click the server node, which in this case is 'RRASSERVER (local)'.
  • Then select 'Properties'.
  • Once in the RRASSERVER properties, select the 'Security' tab.
  • From here, select the 'Authentication provider' dropdown.
  • And then select the option 'RADIUS Authentication'.
  • Once 'RADIUS Authentication' is selected, press the 'Configure' button.
  • We need to now add our RADIUS server to the list. To do this, press the 'Add' button.
  • And then here we need to enter the name of the server that has been configured as the RADIUS server.
  • In this case, the name of the server that has been configured as the RADIUS server is 'nps'.
  • Next, we need to enter in the same shared secret that we configured on the RADIUS server earlier in a previous tutorial. To do this, press the 'Change' button.
  • And enter that shared secret here.
  • Remember, it needs to be the same as the shared secret that was entered on the RADIUS server.
  • Then enter it again to confirm it.
  • Once the shared secret has been entered, press 'OK'.
  • Now that the RADIUS server and the shared secret is defined, press 'OK'.
  • The NPS server has now been added as a RADIUS server.
  • Press 'OK'.
  • Next, we need to now set up our RADIUS client for RADIUS authentication. To do this, select the 'Accounting provider' dropdown.
  • And then select the option 'RADIUS Accounting'.
  • Once 'RADIUS Accounting' is selected, press the 'Configure' button.
  • Now we need to add our RADIUS server to this list. To do this, press the 'Add' button.
  • And then here, like before, we need to enter the name of the server that has been configured as the RADIUS server.
  • In this case, we will be using the same server we used for authentication, 'nps'.
  • Next, we need to again enter in the same shared secret that we configured on the RADIUS server. To do this, press the 'Change'button.
  • And enter that shared secret here.
  • Again, it needs to be the same as the shared secret that was entered on the RADIUS server.
  • Then enter it again to confirm it.
  • After the shared secret has been entered, press 'OK'.
  • Now that the RADIUS server and the shared secret is defined, press 'OK'.
  • The NPS server has now been added as a RADIUS server.
  • Press 'OK'.
