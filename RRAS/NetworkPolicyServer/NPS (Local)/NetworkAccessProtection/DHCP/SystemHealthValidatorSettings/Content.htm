To learn how to configure the System Health Validator settings, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Network Policy Server'...
  • Once in Network Policy Server, expand 'Network Access Protection'.
  • Then expand 'System Health Validators'.
  • And then expand 'Windows Security Health Validator'.
  • Then select 'Settings'.
  • Once 'Settings' is selected, double-click 'Default Configuration'. This will open the 'Windows Security Health Validator'.
  • Once the 'Windows Security Health Validator' is open, the first thing we will bring your attention to is that there are two types of settings. Settings for Windows 7 and Windows Vista and settings for Windows XP. For this tutorial, we will be covering settings for Windows 7 and Windows Vista.
  • This first setting 'A firewall is enabled for all network connections', checks the firewall settings on the client computer. To use this setting, the firewall software that is running on the client computer must be Windows Firewall or other firewall software that is compatible with Windows Security Center. Firewall software that is not compatible with Windows Security Center cannot be managed or detected by the Windows Security Health Agent (WSHA) on the client computer. If you enable this setting and a Windows Security Center compatible firewall is not enabled on the client computer, it is evaluated as noncompliant. If you do not select this setting , the WSHA on the client computer will ignore firewall settings on the client computer.
  • The next setting 'An antivirus application is on', checks to see if antivirus software is running on the client computer. The antivirus application must be compatible with Windows Security Center. Antivirus software that is not compatible with Windows Security Center cannot be managed or detected by the WSHA on the client computer. If you enable this setting and Windows Security Center does not report the presence of a compatible antivirus application on the client computer, it is evaluated as noncompliant.
  • This next setting 'Antivirus is up to date', checks with Windows Security Center to verify that the antivirus definitions for your antivirus applications are up-to-date. To verify that antivirus software is running and that antivirus definitions are the most recent updates available, you must select both 'An antivirus application is on' and 'Antivirus is up to date' .
  • This setting 'An antispyware application is on', checks to see if antispyware software is running on the client computer. The antispyware application must be one that is compatible with Windows Security Center, such as Windows Defender. Antispyware software that is not compatible with Windows Security Center cannot be managed or detected by the WSHA on the client computer.
  • And this setting 'Antispyware is up to date', checks with Windows Security Center to verify that the definitions for your antispyware application are up-to-date. To verify that antispyware software is running and that antispyware definitions are the most recent updates available, you must select both 'An antispyware application is on' and 'Antispyware is up to date' .
  • To view the rest of the settings, scroll down to the bottom of the screen.
  • If this next setting 'Automatic Updating is on' is selected, and Windows Update is not enabled on the client computer, the client computer is evaluated as noncompliant with requirements of the WSHV. Windows Update is considered enabled when one of these settings 'Install updates automatically (recommended)', 'Download updates, but let me choose whether to install them' or 'Check for updates, but let me choose whether to download and install them' is enabled.
  • This next setting 'Restrict access for clients that do not have all available security updates installed', checks to see if security updates have been installed. It will also check the source used for updates and the last time the source was contacted.
  • If we select the dropdown.
  • We are then presented with four severity level options assigned by the Microsoft Security Response Center (MSRC) for the update. If a client computer is missing one or more security updates of the specified severity level, or a higher level, it will be evaluated as noncompliant. The following options are:

    • Low and above - Requires that all updates are installed.
    • Moderate and above - Requires that moderate, important, and critical updates are installed.
    • Important and above  - Requires that important and critical updates are installed.
    • Critical Only - Requires that critical updates are installed.
  • This setting represents the number of hours since the last time the client synchronized with its update source. This is only assessed when joining the network. If the time since the last online scan exceeds the value specified, then the client computer is considered noncompliant. The default for this value is 22 hours, and it can be configured from 1 to 72 hours. If automatic remediation is selected in the NAP policy, the WSHV will instruct the WSHA to do an online scan to check for new security updates.
  • There are two sources for getting updates. The first option is by 'Windows Update'.
  • And the other option is by 'Windows Server Update Services (WSUS)'. You can configure the WSHV to allow updates from one or both of these sources. When the client reports its status with respect to installed updates and provides the update source, the WSHV will accept this status only if the source is allowed. Windows Update is accepted by default. You can also configure the WSHV to accept updates only from WSUS.
  • Once the desired settings have been set, press 'OK' to save these settings.
