To learn how to configure the DHCP server for NAP, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DHCP'...
  • Once in DHCP, expand the DHCP server node, which in this case is 'dhcp.bronstutorials.local'.
  • Then right-click on 'IPv4'.
  • And select 'Properties'.
  • Once in the IPv4 properties, select the 'Network Access Protection' tab.
  • Once in the 'Network Access Protection' tab, the first setting you can configure is whether or not Network Access Protection is enabled or disabled for all the scopes on the DHCP server.
  • This next section pertains to how DHCP will handle NAP if for any reason the NPS server cannot be reached.
  • This first option, 'Full Access', gives all clients network access, regardless if they are NAP compliant or not.
  • With this option, 'Restricted Access', all clients will only be allowed access to the servers defined in the remediation group.
  • And then finally this option, 'Drop Client Packet', all clients will not be allowed on the network.
  • However, instead of just having NAP on or off for all scopes, you have the option of controlling which scopes you want NAP enabled for or disabled for. To do this, first cancel out of here.
  • Then expand 'IPv4'.
  • And then right-click the scope you want to configure NAP for. In this case we will configure NAP on 'Scope 1'.
  • After right-clicking the scope you want to configure NAP for, select its properties.
  • Then go to the 'Network Access Protection' tab for that scope.
  • Once on the 'Network Access Protection' tab, you can enable NAP for this scope.
  • Once NAP is enabled for this scope, you can then decide if you want to use the default NAP profile.
  • Or use a custom profile.
  • If you want to configure NAP further, you can also configure scope options. To do this, press 'OK'.
  • And then expand your scope, which in this case is 'Scope 1'.
  • Then select 'Scope Options'.
  • Once 'Scope Options' is selected, right-click it.
  • And select 'Configure Options'.
  • Once in the scope options, select the 'Advanced' tab.
  • On the 'Advanced' tab, select the 'User class' dropdown.
  • And then select the option 'Default Network Access Protection Class'.
  • This will give you the settings the DHCP server will allocate to the client when it fails its health check. You can set up as many or as few options as you wish. This means you can set up options that are completely different from the main options. For example, you could set up a completly different DNS server. This different DNS server would be allocated only to clients that fail there health check.
