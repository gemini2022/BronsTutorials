To learn how to configure NAP on the DHCP server, proceed with the following steps...
  • Begin first by logging on to the server that has the DHCP Server role installed. If the DHCP Server role is not on the same computer as the Network Policy Server, then the Network Policy Server role needs to be installed. This is because the Network Policy Server settings for NAP wouldn't be on the DHCP server, so in order to configure NAP settings on the DHCP server, Network Policy Server needs to be installed.
  • After NPS is installed, Begin by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Network Policy Server'...
  • Once in Network Policy Server, the first thing we need to do is configure a reference to our NAP server. To do this, expand 'RADIUS clients and Servers'.
  • Then right-click on 'Remote RADIUS Server Groups'.
  • Then select 'New'.
  • Before we can make a reference to our NAP server, the first thing we need to do is define a remote RADIUS group and then put our NAP server in that group. To begin, we first need to give a name to our remote RADIUS group.
  • In this case, we will name our remote RADIUS group 'Remote NPS'.
  • Now that we have our name defined, we next have to add our NAP server to this group. To do this, press the 'Add' button.
  • Then add the name or the IP address of the NAP server.
  • In this case, we will add the name of our NAP server which is 'NPS'.
  • Next, select the 'Authentication/Accounting' tab.
  • On this tab there isn't much to configure, most of these settings can remain as is. The setting that does need to be configured is the 'Shared secret' setting.
  • The shared secret that is entered here must be the same shared secret that was entered on the NAP server.
  • Then enter the shared secret again to confirm.
  • Once the shared secret is entered, press 'OK'.
  • You can see here that our NAP server named NPS is now added to the remote RADIUS Server Group.
  • Press 'OK'.
  • Now that the DHCP server knows who our NAP server is, we next have to configure a policy that will allow the DHCP server to forward NAP requests to that NAP server. To do this, expand 'Policies'.
  • Then right-click on 'Connection Request Policies'.
  • And select 'New'.
  • The first thing we need to do is give a name to our policy.
  • In this case, we will name our policy 'Forward NAP Request'.
  • Next, we need to specify what server will be forwarding the NAP requests. To do this, select the dropdown.
  • And select the type of server that will be forwarding the NAP request, which in this case is 'DHCP Server'.
  • Press 'Next'.
  • Next, we need to specify the condition this policy will use to determine when the forwarding of the NAP request will take place. To add this condition, press the 'Add' button.
  • In this case, the type of condition we want to add is 'Identity Type'. To add this condition, we first need to scroll down.
  • And then once located, select it.
  • And press 'Add'.
  • We then need to select the identifying type, which in this case there is only one 'Machine health check'.
  • Once selected, press 'OK'.
  • You can see here that the condition for the policy has been added and that condition is Identity Type.
  • Press 'Next'.
  • Next, we need to specify where the NAP request will be forwarded to. In this case, we want to tell the wizard to forward the NAP request to our NAP server. To do this, select the option 'Forward reqeusts to the following remote RADIUS server group for authentication'.
  • Then select a RADIUS server group, which is in this case the only option that is available is the RADIUS server group we created earlier.
  • Press 'Next'.
  • On this screen, additional options can be configured. This is more for when you are using third party RADIUS servers. In this case, we are not using any third party RADIUS servers, so we will move on.

  • Press 'Next'.
  • The wizard is now complete, to create the policy, press 'Finish'.
  • Now that the policy is created, select 'Connection Request Policies'.
  • Once 'Connection Request Policies' have been selected, you can see in the window to the right the new policy we just created. The policies shown here are processed in the order they are listed.
  • In this case, we want our new policy to be first in the list so it can be processed first. To do this, right-click the new policy we just created.
  • And select 'Move Up'.
  • The new policy has now been moved to the top of the list and will now be processed first.
