In this tutorial we are going to configure this server as both a Routing and Remote Access and NAT server. But, before we do this, let's first take a look on how we have our network adapters configured on this server.
  • Begin first by selecting the network icon at the bottom right corner of your screen.
  • Then select 'Open Network and Sharing Center'. This will open the 'Network and Sharing Center' screen.
  • Once in 'Network and Sharing Center', select the option 'Change adapter settings', which will bring you to the 'Network Connections' screen.
  • Once in 'Network Connections', notice that there are two network adapters present on this server.
  • The first network adapter defines the outside or public side of the network
  • And the second network adapter defines the local or private side of the network.
  • If we right-click on the first network adapter labeled 'Public Dynamic IP'.
  • And then select 'Properties'.
  • Then double-click on 'Internet Protocol Version 4 (TCP/IPv4)'.
  • You can see that the IP settings for this network interface is being assigned dynamically. No manual configuration has been set. These dynamically assigned settings are being assigned by our ISP. This interface is what will allow our Routing and Remote Access clients to have access to the internet. Let's have a better look at the settings that are being assigned to us. To do this, first press 'OK' or 'Cancel' to close this dialogue box.
  • Then press 'OK' or 'Cancel' to close our 'Public Dynamic IP' properties dialogue box.
  • Then right-click again on the network adapter labeled 'Public Dynamic IP'.
  • But this time, select the option 'Status'.
  • Then select the 'Details...' button.
  • Here you can see there is a good amount of connection detail information for this network interface. In particular, take note of the information about the IP address, subnet mask, default gateway, DHCP server and its DNS servers. These network connection properties are the public side of the network and have been assigned dynamically by the ISP.

  • Press 'Close'.
  • Press 'Close'.
  • Now, lets take a look at the configuration details of the other network adapter that's labeled 'Private Static IP'. To do this, right-click it.
  • And select 'Properties'.
  • And then from here, double-click on 'Internet Protocol Version 4 (TCP/IPv4)'.
  • Now you can see here that the IP settings for this network interface has been statically assigned. The IP address and the subnet mask assigned is for this network adapter is for the private side of the network.
  • However, also take note of not only what has been assigned, but also take note of what has not been assigned. The default gateway and the DNS server information has been left blank. This information is being handled by our other network adapter that is installed on this server, the public interface.

  • Press 'OK'.
