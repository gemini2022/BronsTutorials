To learn about the Security tab, proceed with the following steps...
  • This first setting lists the available security options for RIP. You can set security for how you accept routes and how you announce routes. All of the options on this tab are specific to the type of security option you select. For example, if we select the dropdown.
  • And if you want to set security for accepting routes, then click 'For incoming routes', and then configure the rest of the incoming route options on the tab.
  • Or if you want to set security for announcing routes, then click 'For outgoing routes', and then configure the rest of the outgoing route options on the tab.
  • In this tutorial we will cover the settings for both incoming and outgoing routes, but first we will start with the option 'For incoming routes'.
  • This first option, 'Accept all routes', specifies that the router looks at every route entry in an incoming RIP announcement to determine whether to update its routing table.
  • This option, 'Accept all routes in the ranges listed', specifies that the router looks at each route entry in an incoming RIP announcement and processes the route only if the route falls between one of the ranges listed.
  • This section here is where these ranges are configured.
  • To configure the range, begin by typing the IP address for the lower end of the range here in the 'From' field. For example, if you want to accept all routes to the private network ID 10.0.0.0, you would type 10.0.0.0.
  • And then type the IP address for the higher end of the range here in the 'To' field. For example, if you want to accept all routes to the private network ID 10.0.0.0, you would type 10.255.255.255.
  • After the begin and end addresses of the range have been entered, press the 'Add' button.
  • Once the 'Add' button is pressed, the range that you configured will be listed here.
  • However, with this option 'Ignore all routes in the ranges listed', specifies that the router looks at each route entry in an incoming RIP announcement and discards the route if it falls into one of the ranges listed.
  • If we now go back and selct the dropdown again.
  • But this time select the option 'For outgoing routes'.
  • Now this option changes to 'Announce all routes', which specifies that the router includes all appropriate route entries in outgoing RIP announcements.
  • Next, this option is now 'Announce all routes in the ranges listed', which specifies that the router includes a route in the outgoing RIP announcement only if the route falls into one of the ranges listed.
  • Which in this case, this section is already configured from the previous configuration.
  • And like last time, we will press the 'Add' button.
  • To add the range to this list.
  • And then finally this option, 'Do not announce all routes in the ranges listed', specifies that the router excludes a route from the outgoing RIP announcement if the route falls into one of the ranges listed.
  • Once selected, no announcements from the range listed below will be sent.
