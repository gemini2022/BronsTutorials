To learn about the Neighbors tab, proceed with the following steps...
  • This first option, 'Use broadcast or multicast only', specifies that all RIP announcements are sent by using the outgoing packet protocol specified on the General tab.
  • This next option, 'Use neighbors in addition to broadcast or multicast', specifies that RIP announcements are unicasted to specific neighboring routers in addition to the outgoing packet protocol specified on the General tab.
  • Once the option 'Use neighbors in addition to broadcast or multicast' is selected, you then have to provide the IP address of the neighboring RIP router. To do this, enter in the address of the neighboring router in the 'IP address' field. In this case, the address of the neighboring router is '192.168.100.1'.
  • Once the IP address of the neighboring router has been entered, press the 'Add' button.
  • Once the 'Add' button is pressed, the IP address of the neighboring router that was entered in the 'IP address' field will be added to the list of configured RIP neighbor IP addresses.
  • And then finally, this last option, 'Use neighbors instead of broadcast or multicast', specifies that RIP announcements are only unicasted to the specified neighboring routers. RIP announcements are not sent according to the outgoing packet protocol specified on the General tab. If you have nonbroadcast networks such as Frame Relay and want to enable RIP between specific routers, you should select this option.
  • Once the option 'Use neighbors instead of broadcast or multicast' is selected, you then also have to provide the IP address of the neighboring RIP router to the list of configured RIP neighbor IP addresses.
