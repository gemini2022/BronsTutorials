To learn about the General tab, proceed with the following steps...
  • This first setting, 'Operation mode', lists the available operation modes. You can select one of two operation modes. To do this, begin by selecting the dropdown.
  • This first option in the dropdown list is 'Auto-static update mode'. This mode sends out RIP announcements only when other routers request an update. Routes learned over RIP while in auto-static mode are marked as static routes when they are stored in the routing table. If you start and stop the router or if RIP is disabled, these routes remain in the routing table until they are manually deleted.
  • The next mode in the dropdown list is 'Periodic update Mode'. This mode sends out RIP announcements periodically, as specified in Periodic announcement interval on the Advanced tab. Routes learned over RIP while in periodic mode are marked as RIP routes and are removed if the router is stopped and restarted.
  • This next setting, 'Outgoing packet protocol', lists the available protocols for outgoing RIP announcements. You can select one of four protocols. To do this, begin by selecting the dropdown.
  • The first option, 'RIP version 1 broadcast', specifies that RIP version 1 announcements are sent as broadcasts.
  • The second option, 'RIP version 2 broadcast, specifies that RIP version 2 announcements are sent as broadcasts. If you have a mixed environment with RIP versions 1 and 2, select this protocol.
  • The third option, 'RIP version 2 multicast', specifies that RIP version 2 announcements are sent as multicasts. Select this protocol only if all the neighboring RIP routers on the network that is connected to this interface are also RIP version 2 routers that are configured for RIP version 2. A RIP version 1 router cannot process a RIP version 2 multicast announcement.
  • And then finally, this last option, 'Silent RIP', disables outgoing RIP announcements on this interface. In silent mode, the computer listens to RIP announcements and updates its routing table but does not announce its own routes.
  • This setting, 'Incoming packet protocol', lists the available protocols for incoming RIP announcements. You can select one of four options. To do this, begin by selecting the dropdown.
  • The first option, 'Ignore incoming packets', ignores announcements. If you want a router that only announces routes, select this option.
  • Next is the option 'RIP version 1 and 2', which specifies that this interface will accept announcements of both types.
  • This option, 'RIP version 1 only', specifies that this interface will accept only RIP version 1 announcements.
  • And then finally, this last option, 'RIP version 2 only', specifies that this interface will accept only RIP version 2 announcements.
  • Now this setting, 'Added cost for routes', specifies the cost associated with sending packets over this interface. You can also click the arrows to select a new setting. When routes are added to the routing table, this number is added to the hop count as advertised in the RIP message. By default, this value is 1 to indicate the additional hop as created by the receiving router. If you want to associate a higher cost with the route, set a higher cost (for example, if you have a heavily trafficked route that you do not want used unless other routes are unavailable). The accumulated costs/hops for the farthest nodes on the IP network must not exceed 15.
  • This next setting, 'Tag for announced routes', specifies a tag number for the routes advertised on this interface. You should use this option if you want all packets sent over this interface to include a tag in RIP version 2 announcements.
  • This option, 'Activate authentication', specifies whether you want to enable authentication by using a plaintext password for RIP version 2 announcements over this interface. All incoming and outgoing RIP version 2 packets must contain the same plaintext password. Therefore, you must enable this option and configure the same password for all routers that are connected to this interface. This option is a form of identification. It is not a security option.
  • And then finally, if the option 'Activate authentication' is selected, then a password between 1 and 16 characters long must be entered. This password is sent as plaintext in the RIP version 2 message for authentication between neighboring RIP version 2 routers.
