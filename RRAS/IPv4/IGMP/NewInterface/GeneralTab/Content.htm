To learn about the general tab, proceed with the following steps...
  • This first option, 'Enable IGMP', specifies whether the Internet Group Management Protocol (IGMP) is enabled for multicast forwarding on this interface. IGMP is used by hosts to register membership in IP multicast groups. Enabling IGMP on this interface allows the router to either listen for IGMP messages sent by hosts or forward IGMP messages received on other interfaces.
  • This next option 'IGMP router', specifies that IGMP router mode is enabled on this interface. An interface in IGMP router mode listens for IGMP messages from hosts and updates the TCP/IP multicast forwarding table.
  • Or this option 'IGMP proxy', specifies that IGMP proxy mode is enabled on this interface. An interface in IGMP proxy mode acts as a host, forwarding IGMP host membership messages for all IGMP host membership messages received on all other interfaces on which IGMP router mode is enabled. Additionally, all nonlocal multicast traffic received on all other interfaces (on which IGMP router mode is enabled) is forwarded over the IGMP proxy mode interface.
  • You will notice that when the option 'IGMP proxy' mode is selected, this option does not apply.
  • And also, if we select the 'Router' tab.
  • You will notice that any information on this tab also does not apply when 'IGMP proxy' mode is selected.
  • So if we go back to the 'General' tab.
  • And select the option 'IGMP router'.
  • We can now take a look at the setting 'IGMP protocol version'.
  • This dropdown lists all the versions of IGMP supported on this interface.
  • In this case, we will keep the default setting selected, 'Version 3'.
