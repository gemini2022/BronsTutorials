To learn about the General tab, proceed with the following steps...

  • The first setting, 'Relay DHCP packets', specifies whether DHCP messages are relayed between DHCP clients on the subnet to which this interface is attached and DHCP servers. If this option is selected, you must then configure the DHCP server addresses in the 'DHCP Relay Agent Properties' dialog box.
  • This next setting 'Hop-count threshold', specifies the maximum number of DHCP relay agents that will handle DHCP relayed traffic. You can also click the arrows to select a new setting. The default value is 4 hops. The maximum value is 16 hops.
  • And then finally, the last setting 'Boot threshold (seconds)', specifies the number of seconds the relay agent waits before forwarding DHCP messages. You can also click the arrows to select a new setting. The default value is 4 seconds. This option is useful when you want a local DHCP server to respond first, but if the local DHCP server does not respond, you want to forward messages to a remote DHCP server.

  • Press 'OK'.
  • Because we selected the option 'Relay DHCP packets' in the previous dialog box, we must now configure the DHCP server addresses in the 'DHCP Relay Agent Properties' dialog box. To do this, we need to go back to 'DHCP Relay Agent' and right-click it.
  • And select 'Properties'.
  • Once in the properties of the DHCP Relay Agent, we have to configure the address of the DHCP server that this DHCP relay agent will send DHCP requests to. In this field labeled 'Server address', is where the address of the DHCP server is entered.
  • In this case, the address of the DHCP server is '192.168.200.50'.
  • Once the address of the DHCP server has been entered, press the 'Add' button.
  • You may add more DHCP server addresses if needed, otherwise press 'OK'.
