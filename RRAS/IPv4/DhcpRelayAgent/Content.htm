The DHCP relay agent is a machine with the DHCP Relay Agent service installed and configured to forward DHCP requests to a DHCP server on a different subnet. Because DHCP is a broadcast-based protocol, by default its packets do not pass through routers. A DHCP relay agent receives any DHCP broadcasts on the subnet and forwards them to the specified IP address of a DHCP server on a different subnet. The DHCP Relay Agent is compliant with RFC 1542, "Clarifications and Extensions for the Bootstrap Protocol." For each IP network segment that contains DHCP clients, either a DHCP server or a computer acting as a DHCP Relay Agent is required. You cannot use the DHCP Relay Agent component on a computer that is running the DHCP service, the network address translation (NAT) routing protocol component with automatic addressing enabled, or Internet Connection Sharing (ICS). To learn about DHCP Relay Agents, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Routing and Remote Access'...
  • Once in Routing and Remote Acces, expand the server node, which in this case is 'RRASSERVER (local)'.
  • Then expand 'IPv4'.
  • Once you expand 'IPv4', select 'DHCP Relay Agent'.
  • Once you select 'DHCP Relay Agent', in the window to the right side of the Routing and Remote Access screen, you will see a list of details that pertain to DHCP Relay Agents.
