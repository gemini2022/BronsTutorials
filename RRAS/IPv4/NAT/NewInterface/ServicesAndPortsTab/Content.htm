To learn about the Services and Ports tab, proceed with the following steps...
  • This tab is used to create, modify, and activate mappings between public IP addresses and ports, and services running on hosts on the private network at specified IP addresses and ports. To configure one of the predefined services and activate the mapping, we need to select the box next to the service in the list. In this case, the service we want to activate is at the bottom of the list, therefore, first scroll to to bottom.
  • Once at the bottom of the list, the service we want to activate is 'Web Server (HTTP)'.
  • Once the service is selected an 'Edit Service' dialog box will appear. At the top, under 'Description of Service', will be displayed the name of the service that you are editing.

  • Next, is the option 'On this interface', which specifies that the incoming IP address is the public IP address that is assigned to this interface.
  • Or this option 'On this address pool entry', which specifies that the incoming IP address is from the public address pool.
  • Once this option is selected, we need to type the appropriate public IP address (in dotted decimal notation) from the public address pool. In this case, the public IP address that we will use from the address pool will be '172.100.235.250'.
  • This section specifies the protocol used for data reception and transmission.
  • This next setting, 'Incoming port', specifies the destination port number of incoming public traffic. In this case, for the Web server, the incoming port number is 80.
  • Next, is 'Private address', which is the IP address (in dotted decimal notation) of the private network computer. In this case, the IP address for our Web server is '192.168.1.50'.
  • And then finally 'Outgoing port', specifies the destination port number of the private traffic. In this case, for the Web server, the outgoing port number is 80.

  • Press 'OK' to activate the service.
  • You also have the option to Add, Edit, or Delete any service if necessary.
