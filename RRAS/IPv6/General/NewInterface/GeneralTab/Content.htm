To learn about the General tab, proceed with the following steps...
  • This first option, 'Enable IPv6 router manager', specifies whether TCP/IP is enabled on this interface. Select this check box to set the Administrative Status of the interface to Up. Clear this check box to set the Administrative Status of the interface to Down.
  • Here you can set filters based on where the packet is coming from (the source), where the packet is going (the destination), or what protocol the packet is using. You can set all three of these values for a filter or set any one or combination of the three, depending on the level of control that you want. First, press the 'Inbound Filters...' button.
  • Then click 'New...' to add a filter.
  • This option specifies whether the filter checks the network from which the packet is coming. After selecting this check box, you must specify a network IP address and prefix length.
  • Then enter the IPv6 address of the network from which the packet was sent.
  • Then enter the prefix length.
  • This option specifies whether the filter checks the network to which the packet is going. After selecting this check box, you must specify a network address and prefix length.
  • Then enter the IPv6 address of the network from which the packet is going.
  • Then enter the prefix length.
  • Here, this setting lists the available protocol types that you can base a filter on. The default is Any protocol. If you select Other, you must type a protocol identifier in Protocol number.
  • Once the dropdown is selected, you can choose one of the following options.
    • TCP
    • TCP [established]
    • UDP
    • ICMP
    • Any
    • Other
  • In this case, we will choose the option to use any protocol.

  • Press 'OK' to create the filter.
  • Now that the filter is created, we next have to decide whether this filter will be used to deny packets from the source network, or allow packets from the source network. This option specifies that all incoming packets that match the filters listed are discarded. All incoming packets are compared to the filters listed. If a packet matches a filter, the packet is dropped. If the packet does not match a filter, the packet is accepted.
  • This option specifies that all incoming packets that match the filters listed are accepted. All incoming packets are compared to the filters listed. If a packet matches a filter, the packet is accepted. If the packet does not match a filter, the packet is dropped.

  • Press 'OK'.
  • This time we will create a filter for all outbound packets.
  • Again, click 'New...' to add a filter.
  • Like before, this option specifies whether the filter checks the network from which the packet is coming. After selecting this check box, you must specify a network IP address and prefix length.
  • Then enter the IPv6 address of the network from which the packet will be sent.
  • And then enter the prefix length.
  • And also like before, this option specifies whether the filter checks the network to which the packet is going. After selecting this check box, you must specify a network address prefix length.
  • Then enter the IPv6 address of the network from which the packet is going.
  • Then enter the prefix length.
  • And like before, this setting lists the available protocol types that you can base a filter on. The default is Any protocol. If you select Other, you must type a protocol identifier in Protocol number.
  • And also like before, once the dropdown is selected, you can choose one of the following options.
    • TCP
    • TCP [established]
    • UDP
    • ICMP
    • Any
    • Other
  • In this case, we will again choose the option to use any protocol.

  • Press 'OK' to create the filter.
  • Now that the filter is created, we next have to decide whether this filter will be used to deny packets to the destination network, or allow packets to the destination network. This option specifies that all outgoing packets that match the filters listed are discarded. All outgoing packets are compared to the filters listed. If a packet matches a filter, the packet is dropped. If the packet does not match a filter, the packet is transmitted.
  • And this option specifies that all outgoing packets that match the filters listed are transmitted. All outgoing packets are compared to the filters listed. If a packet matches a filter, the packet is transmitted. If the packet does not match a filter, the packet is dropped.

  • Press 'OK'.
