To learn how to add the 'DNS Server Role' to a server, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Server Manager'...
  • Once in Server Manager, select 'Roles' at the upper left corner of your screen.
  • Then, at the far right side of your screen, select 'Add Roles' to launch the Add Roles Wizard.
  • The first screen of the wizard asks you to verify if this server is properly configured.

  • Click 'Next'.
  • Select the 'DHCP Server' option.

  • Then press 'Next >'.
  • This screen of the wizard gives a brief introduction to DHCP.

  • Click 'Next >'.
  • On this screen, you first need to set the IPv4 address you wish to bind to the DHCP server. The DHCP server will listen and send responses on this interface.

  • Click 'Next >'.
  • On this next screen, you'll need to specify the name of the parent domain that clients will use for name resolution. This domain will be used for all scopes that is created on this DHCP server. If this server is part of a domain, this field will automatically be filled in, which is the case for this example. If however this server is not part of a domain, this field will need to be entered manually.
  • And then here, a preferred DNS server needs to be entered. The wizard in most cases will enter a DNS server for you, based on the primary DNS server this computer has configured.
  • You can validate this DNS server by pressing the 'Validate' button.
  • If the DNS server that is entered is valid, then a 'valid' icon will appear.
  • You can then enter in an alternate DNS server. This field is optional.

  • Click 'Next >'.
  • On this screen, you can set a WINS server if you are using one.WINS is Microsoft's name resolution software before Microsoft switched to DNS when Windows Server 2000 was introduced. Your network may or may not have a WINS server. If your network does NOT then you can leave it on the default option of, 'WINS is not required for applications on this network'.
  • If your network does have a WINS server, then select this option, 'WINS is required for applications on this network', then proceed with filling in the additonal required information.

  • Click 'Next >'.
  • This screen allows you to add DHCP scopes. A scope is simply a group of IP addresses which can be given out to clients on the network. You have the option of creating a scrop now or later after DHCP has been installed. In this case, we will add a scope later, but we'll go ahead and press the 'Add...' button now to demonstrate the options that are available if we were to create one.
  • After pressing the 'Add...' button on the previous screen, the first field is asking to enter in the name of the scope. It is possible to have multiple scopes, therefore is the reasoning for giving a name to the scope.
  • This field is asking for the first IP address of the scope.
  • And this field is asking for the last IP address of the scope.
  • For this field, if we select the dropdown...
  • We can then choose the subnet type, either wired or wireless.
  • Here you have the option of activating the scope. Before the DHCP server can issue any IP address, the scope must first be activated. You have the option of activating the scope now.
  • The subnet mask of the IP addresses for this scope can be entered here.
  • And finally, the default gateway can be entered here.

  • Press 'Cancel' because we're not going to configure any scopes at this time.
  • Click 'Next >'.
  • This next screen enables IPv6 stateless mode or disables it. If you have a router on your network that can advertise IPv6 network IDs, then you can enable stateless mode for this server.
  • If however, you do not have a router on your network that can advertise IPv6 network IDs, then you'll need to disable stateless mode for this server. With stateless mode disabled, the DHCP server will advertise the network ID for your clients.

  • Click 'Next >'.
  • On this next screen, you'll need to specify the name of the parent domain that clients will use for name resolution. This domain will be used for all scopes that is created on this stateless IPv6 DHCP server. If this server is part of a domain, this field will automatically be filled in, which is the case for this example. If however this server is not part of a domain, this field will need to be entered manually.
  • And then here, a preferred DNS server IPv6 address needs to be entered. IP addresses in version 6 are difficult to remember and easy to mistype. To make sure the IPv6 address of DNS server that is being entered is accurate we can do the following:
  • First, select 'Start' at the bottom left corner of your screen.
  • Then select the 'Command Prompt'.
  • Once the command prompt has been opend, ping your DNS server. In this case, our DNS server is 'dc1', therefore the command we will enter is 'ping dc1'.
  • The output from the command we will pipe to clip. Clip coppies the output to the clipboard. To do this, type '| clip', then press 'Enter'.
  • After the command has been run, close the command prompt.
  • Then go back down to 'Start'....
  • And this time select 'Notepad'.
  • Once 'Notepad' has been opened, paste the contents of the clipboard into it.
  • From here, select and copy the IPv6 address of the DNS server you pinged earlier. In this case the IPv6 address is 'fe80::c0a8:2'.
  • Now, back at the 'Add Roles Wizard' screen, select the current preferred DNS address.
  • And paste in the correct IPv6 address.
  • From here, we can press the 'Validate' button.
  • To confirm that it is valid.
  • Here we have the option to enter an alternate DNS server, but we will leave as is and move on.

  • Click 'Next >'.
  • On this screen of the wizard, you can provide credentials to authorize the DHCP server in 'Active Directory'. A DHCP server will need to be authorized in 'Active Directory' before you can use it on your network. This option will allow the 'Administrator' to authorize the DHCP server.
  • With this option, you can specify an alternate user for authorizing the DHCP server.
  • Or you can choose not to authorize the DHCP server at this time. But, if you choose this option, keep in mind that the DHCP server, at some point, must be authorized in 'Active Directory' before it can service clients.

  • Click 'Next >'.
  • Confirm the installation selections, then click 'Install'.
  • Wait while the Add Roles Wizard installs the DHCP server.
  • View the installation results and then click the 'Close' button.
