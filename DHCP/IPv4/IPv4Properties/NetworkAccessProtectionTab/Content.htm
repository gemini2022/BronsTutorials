Network Access Protection (NAP) is a system designed to protect networks from clients which are not deemed to be secure or healthy (to use Microsoft's terminology). When NAP is implemented, clients without the required level of "health" are directed to a remediation server where the necessary updates may be obtained to bring the system into compliance with the Network Access policy of the network. In addition, the user may also be directed to a web page providing details of why access to the network has been declined and outlining the steps necessary to remedy the problem. To learn about the Network Access Protection tab, proceed with the following steps...
  • If the 'Enable on all scopes' button is pressed...
  • Then we will get a message stating that the Network Access Protection settings of all the scopes will be overwritten. If we then press the 'Yes' button, then Network Access Protection will be enabled on all scopes.
  • Or, if we press the 'Disable on all scopes' button...
  • Then we will get a message stating that the Network Access Protection settings of all the scopes will be overwritten. If we then press the 'Yes' button, then Network Access Protection will be disabled on all scopes.
  • This option, 'Full Access', gives all clients network access, regardless if they are NAP compliant or not.
  • With this option, 'Restricted Access', all clients will only be allowed access to the servers defined in the remediation group.
  • And then finally this option, 'Drop Client Packet', all clients will not be allowed on the network.
