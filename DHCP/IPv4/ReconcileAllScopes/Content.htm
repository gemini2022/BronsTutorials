You can use the 'Reconcile All Scopes' procedure to fix inconsistencies, such as incorrect or missing information for client IP addresses, that are stored in scope lease information. The DHCP Server service stores detailed and summary IP address information in the DHCP database. When reconciling scopes, the detailed and summary entries are compared to find inconsistencies. After you select and reconcile scope inconsistencies, the DHCP Server service either restores those IP addresses to the original owner or creates a temporary reservation for those addresses. These reservations are valid for the lease time assigned to the scope. When the lease time expires, the addresses are recovered for future use. To learn how to reconcile all scopes, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DHCP'...
  • Once in DHCP, expand your DHCP server, in this case the DHCP server is 'dc1.bronstutorials.local'.
  • Then select 'IPv4'.
  • Once 'IPv4' has been selected, right-click it...
  • And then select 'Reconcile All Scopes...'.
  • Now press the 'Verify' button.
  • In this case, the database is consistant with the registry. If this was NOT the case and the database and the registry were NOT consistant, then the DHCP Server service either restores those IP addresses to the original owner or creates a temporary reservation for those addresses. These reservations are valid for the lease time assigned to the scope. When the lease time expires, the addresses are recovered for future use.

  • Press 'OK'.
  • On this screen, select the scopes you wish to add to the superscope. In this case we only have one scope to select.

  • Press 'Cancel'.
