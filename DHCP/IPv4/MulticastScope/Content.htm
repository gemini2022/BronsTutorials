Multicasting is the process of sending a message to a group of recipients as opposed to unicasting which is a message sent to a specific recipient. Normally, DHCP is used to assign each DHCP client a single unique IP address from a range of IP addresses configured in a scope. You can also create multicast scopes so that messages destined to a multicast IP address can be sent to all clients in a multicast group. Multicast scopes are supported through a protocol known as the Multicast Address Dynamic Client Allocation Protocol (MADCAP). MADCAP controls how the DHCP servers dynamically assign IP addresses on a TCP/IP network. The multicast server (in this case, the DHCP server) is configured with a group of Class D IP addresses (in the range of 224.0.0.0–239.255.255.255) that can be assigned to multicast clients. The server is also responsible for maintaining the group membership list and updating the list as members join and leave a group. To learn how to create a multicast scope, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DHCP'...
  • Once in DHCP, expand your DHCP server, in this case the DHCP server is 'dc1.bronstutorials.local'.
  • Then select 'IPv4'.
  • Once 'IPv4' has been selected, right-click it...
  • And then select 'New Multicast Scope...' to launch the 'New Multicast Scope Wizard'.
  • The first screen of the wizard is the welcome screen. Press 'Next' to continue.
  • On this screen, we need to enter in the name of the multicast scope. In this case we're going to name the multicast scope, 'MyMulticastScope'.
  • You can also add a description if you wish. This field is optional, but for this example, we'll enter 'My Multicast Scope Group'.

  • Press 'Next >'.
  • On this screen, you first need to enter in the starting IP address of the range. The range must be in the 'Class D' IP address range (224.0.0.0–239.255.255.255). In this case, we'll start the range at '224.10.10.1'.
  • Next, enter in the end IP address of the range. In this case the end IP address will be '224.10.10.254'.
  • And then finally the TTL setting can be configured here. The TTL or Time to Live represents the number of routers that multicast traffic passes through on your network. In this case, we'll leave it on the default and move on.

  • Press 'Next >'.
  • On this next screen, you have the option of entering in exclusions. Any IP addresses you set in this range will NOT be distributed by the server. In this case, we will exclude a range of IP address starting with '224.10.10.10'.
  • And ending with '224.10.10.20'.
  • Once the start IP address and the end IP address of the excluded range has been entered, press 'Add'.
  • After pressing 'Add', the exclued range will be displayed here. You may add more excluded ranges if you wish.

  • Press 'Next >'.
  • On this screen, you can set how long the client will have the IP address before it will lose it. In this case the default is 30 days.

  • Press 'Next >'.
  • On this screen, you have the option of activating this scope now. All scopes must be activated before they can start to issue out IP addresses. In this case, we will go ahead and keep the default option 'Yes' selected.

  • Press 'Next >'.
  • And then finally, press 'Finish' to create the multicast scope.
