Vendor Classes work in the same manner as user classes do. However, vendor classes are created when a network setup has multi-platform environment and the administrators want the DHCP client computers running non-Microsoft operating systems to obtain a different set of DNS and default gateway addresses. For example, in an organization some computers have Microsoft Windows 7 installed on them and receive 192.168.11.1 as their DNS and default gateways. On the other hand, other computers have Linux OS installed on them. A Vendor class for Linux can be created so that all Linux clients can be assigned with different DNS and default gateway addresses, e.g. 192.168.24.2. To learn how to create a vendor class, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DHCP'...
  • Once in DHCP, expand your DHCP server, in this case the DHCP server is 'dc1.bronstutorials.local'.
  • Then select 'IPv4'.
  • Once 'IPv4' has been selected, right-click it...
  • And then select 'Define Vendor Classes...'.
  • Once the 'DHCP Vendor Classes' dialogue box opens, press the 'Add...' button.
  • Then begin by entering in a name for the new class. In this case, since we're creating a vendor class for the company's IP phone system, we'll name the new class 'Aastra IP Phone'.
  • You can also add a description if you wish. This field is optional, but for this example, we'll enter 'Aastra IP Phone Vendor Class Definition'.
  • And then here under the 'ASCII' column, enter in the class ID. In this case we'll enter the ID of 'Aastra IP-Phone'.
  • Press 'OK'.
  • Then press 'Close'.
  • Now that the vendor class has been defined, we next have to set the predefined options. To do this, right-click 'IPv4'.
  • And select 'Set Predefined Options...'.
  • Once the 'Predefined Options and values' form is open, select the 'Option Class' dropdown.
  • And select the newly created vendor class, which in this case is 'Aastra IP Phone'.
  • This vendor class now needs options applied to it. To do this, press the 'Add...' button.
  • First, enter a name for this option. In this case, we'll call this option 'Handset Configuration'.
  • Then, we have to define what data type this option will be. To do this, select the 'Data type' dropdown.
  • And select the data type. In this case, the data type we will choose for this option is 'Binary'.
  • Now, enter in the code that this option will be referred by. In this case we will use code '43' for 'Vendor Specific Info'.
  • And then finally, add a brief description.
  • Once finished configuring this option, press 'OK'.
  • You may add additional options for this vendor class if you wish, otherwise press 'OK'.
  • Now that the vendor class has been defined and the options for the vendor class has been created, we next have to apply these options to a scope. We can apply these options to a single scope or all IPv4 scopes using the 'Server Options'. In this case, we will apply these options to a single scope. To do this, we'll first expand the scope.
  • Then select 'Scope Options'.
  • Once 'Scope Options' is selected, right-click it.
  • And select 'Configure Options...'.
  • Once the 'Scope Options' has been opened, select the 'Advanced' tab.
  • We now have to select the vender class we created. To do this, select the 'Vendor class' dropdown.
  • And then select the newly created vendor class, in this case it's 'Aastra IP Phone'.
  • Now, select the options you want this vendor class to apply to the clients. In this case, we only created one option, but if more were created, we would enable them here.
  • And then finally, enter in the option details.
