Every certificate has a validity period. After the end of the validity period, the certificate is no longer considered an acceptable or usable credential. The Certificates snap-in enables you to renew a certificate issued from a certification authority before or after the end of its validity period by using the Certificate Renewal Wizard. You can either renew the certificate with the same key set that you used before, or you can renew a certificate with a new key set. This decision can be based on a number of factors, including the lifetime of the certificate, the length of the existing or future key, the value of the data protected by the key pair, and the possibility that a private key has been obtained by a malicious user. To learn how to renew a certificate, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, right-click the root node for the CA, which in this case is 'RootCA'.
  • First, select 'All Tasks'.
  • And then select 'Renew CA Certificate...'.
  • Before the certificate can be renewed, Active Directory Certificate Services needs to stop running. A dialogue box will appear asking if you want to stop Active Directory Certificate Services now.

  • Press 'Yes'.
  • And then finally, before the certificate is renewed, you need to decide if you want to generate a new public and private key or renew the certificate with the same key set that you used before. In this case we leave it on the default of 'Yes', generate a new public and private key.

  • To renew the certificate, press 'OK'.
