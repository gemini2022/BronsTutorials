A recovery agent is a person who is authorized to recover a certificate on behalf of an end user. Because the role of recovery agents can involve sensitive data, only highly trusted individuals should be assigned to this role. With this tab, you have the ability to enable key archival for the certificates that are configured to do so. To learn how to do this, proceed with the following steps...
  • The first thing we need to do is select the option, 'Archive the key'.
  • In the 'Number of recovery agents to use' input field, type the number of key recovery agents that will be used to encrypt the archived key. The number of recovery agents to use must be between one and the number of key recovery agent certificates that have been configured. In this case, we will leave it on the default value of '1'.
  • Next, press the 'Add...' button.
  • Then click the key recovery certificates that are displayed and press 'OK'.
  • After pressing 'OK' on the previous form, the certificate(s) will appear in the 'Key recovery agent certificates' list, but their status will be listed as 'Not loaded'.

  • Press 'Apply'.
  • Once you press 'Apply' on the CA properties form, a message will appear asking if you want to restart Active Directory Certificate Services.

  • Press 'Yes'.
  • After restarting Active Directory Certificate Services, the status of the certificates will change to 'Valid'.

  • Press 'OK'.
