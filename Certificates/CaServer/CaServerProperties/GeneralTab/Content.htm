To learn about the General tab, proceed with the following steps...
  • This first section at the top displays the name of the CA.
  • Here in this section will be a list of all the certificates that this CA has generated. For example, when a certificate is renewed, that new certificate will be added here in this list.
  • You can view any certificate in this list by selecting the certificate and then pressing the 'View Certificate' button.
  • By pressing the 'View Certificate' button on the previous form, a new form will appear displaying additional information about that certificate. For example, you can see here what this certificate is intended for. In this case this certificate is indended for all puposes. If the 'EnhancedKeyUsageExtension' was defined in the CAPolicy.inf, that information would be displayed here.
  • You can see here what CA this cerficate was issued to and what CA this certificate was issued by. In this case, this certificate was issued to and by the same CA. This referred to as a self signed certificate.
  • And then here you can see information about the validity period of this certificate.
  • If the 'PolicyStatementExtension' was defined in the CAPolicy.inf, then you can view those details by pressing the 'Issuer Statement' button here.
  • The legal policy statement you defined in the CAPolicy.inf will be displayed here. Pressing the 'More Info' button will take you to the URL location defined in the CAPolicy.inf.

  • Press 'Close'.
  • If we now select the 'Details' tab.
  • In this section, we can view a more extensive list of information about the certificate.
  • Also on this tab, you have the option of exporting this certificate. This can be done by first clicking on the 'Copy to File...' button.
  • Once the 'Copy to File..' button has been clicked, the 'Certificate Export Wizard' will appear.

  • Press 'Next >'.
  • If you are exporting certificates for import onto a computer running Windows, PKCS #7 format is the preferred export format. This format preserves the chain of certification authorities (CAs), or the certification path, of any certificate that includes countersignatures associated with signatures. If you are exporting certificates for import onto a computer running another operating system, it is possible that the PKCS #7 format is supported. If it is not supported, the DER-encoded binary format or the Base64-encoded format is provided for interoperability. In this case, we will be importing the certificate onto a computer running Windows, therefore we will select the 'Cryptographic Message Syntax - PKCS #7 Certificate (.P7B)' option.
  • Also, be sure to select the 'Include all certificates in the certification path if possible' option as well.
  • Press 'Next >'.
  • On this screen, we will choose the location where the certificate will be exported. To do this, begin by pressing 'Browse...'.
  • The certificate can be exported to any location, in this case, we'll save it to the desktop. To do this, select 'Desktop'.
  • Then give the certificate a name. In this case, we will name it 'SubCert'.
  • Then press 'Save'.
  • Once the location and the name of the certificate has been defined, press 'Next >'.
  • And finally, press 'Finish' to export the certificate..
  • Once the certificate has been successfully exported, press 'OK'.
  • Now, back to the certificate. If we select the 'Certification Path' tab.
  • You can view the certificate hierarchy here. In this case, there is only one certificate, the root CA.
  • If there were more than just one cerficate in the hierarchy, you could select that certificate and view it's details here.

  • Press 'OK'.
