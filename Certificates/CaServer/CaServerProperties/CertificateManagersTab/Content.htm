A certificate manager can approve certificate enrollment and revocation requests, issue certificates, and manage certificates. This role can be configured by assigning a user or group the Issue and Manage Certificatespermission. When you assign this permission to a user or group, you can further refine their ability to manage certificates by group and by certificate template. For example, you might want to implement a restriction that they can only approve requests or revoke smart card logon certificates for users in a certain office or organizational unit that is the basis for a security group. This restriction is based on a subset of the certificate templates enabled for the certification authority (CA) and the user groups that have Enroll permissions for that certificate template from that CA. To learn how to do this, proceed with the following steps...
  • First, begin by selecting the option 'Restrict certificate managers'.
  • Once the option 'Restrict certificate managers' has been selected, a list of all the certificate managers that have been configured on the Security tab will be displayed.
  • The next step is to define what certificate templates certificate managers can manage. The default setting is all certificate templates, but we want to limit what certificate templates certificate managers can manage. To do this, we first need to add these certificate templates to a list. We can do this by first pressing the 'Add...' button.
  • Then select the certificate template. In this case, we are going to select 'Smartcard Logon'.
  • Once the certificate template is selected, press 'OK'.
  • Now that the certificate template has been added, we can now remove the <All> certificate templates option. To do this, select it.
  • And press the 'Remove' button.
  • This last step is to define what users certificate managers can manage. Now, the default setting is managing all users. In this case, we only want certificate managers to be able to manage only a certain group of users. To do this, we first need to add these group of users . We can do this by first pressing the 'Add...' button.
  • Then enter in the group of users, which in this case is the 'sales' group.

  • Press 'OK'.
  • Now that the 'Sales' group has been added, we can now remove the group we don't want, which in this case is 'Everyone'. To do this, select the 'Everyone' group.
  • Then press the 'Remove' button.
  • For this example we are defining what group of users certificate managers are allowed to manage. However, you also have the option of defining what users certificate managers Cannot manage. We can do this by simply pressing the 'Deny' button.
  • So now you can see that the certificate managers that are configured in the Security tab are denied the ability to manage Smartcard Logon certificate templates to users in the 'Sales' group.
  • But for this situation, we do want the certificate managers that are configured in the Security tab the ability to manage Smartcard Logon certificate templates to users in the 'Sales' group. Therefore, we will press the 'Allow' button.
  • Now the certificate managers that are configured in the Security tab have the ability to manage Smartcard Logon certificate templates to users in the 'Sales' group.

  • Press 'OK'.
