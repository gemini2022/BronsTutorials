The policy module determines the default action of a CA upon receiving a certificate request: approve, deny, or mark as pending. In the majority of instances, the administrator of a stand-alone CA should set all incoming certificate requests to pending. Otherwise, because the stand-alone CA does not verify the identity of requesters via Active Directory Domain Services (AD DS), there is no way to verify the identity and validity of the certificate requester. The CA can have only one policy module loaded at a time. To learn about the Policy Module tab, proceed with the following steps...
  • This first section displays the policy module that is currently active. There can be only one policy module active at a time and this section displays which one is currently active.
  • If you want to select another policy module, the way to do this is to press the 'Select...' button.
  • And choose the policy module in the list you want to set as active. In this case there is only one policy module available to choose from, but if there was more than one, then they would be displayed here.

  • Press 'OK'.
  • You can view the properties of the currently selected policy module by press the 'Properties...' button here.
  • Here in the properties of the currently selected policy module there are two options in which you can have the CA handle certificate requests. The first option is to set the certificate request status to pending. This is the most common option to use if the CA is a standalone CA. Because the standalone CA does not verify the identity of requesters via Active Directory Domain Services (AD DS), there is no way to verify the identity and validity of the certificate requester.
  • The second option is to automatically issue the certificate. In this case, this option is set by default becuase this CA is an enterprise CA.

  • Press 'OK'.
