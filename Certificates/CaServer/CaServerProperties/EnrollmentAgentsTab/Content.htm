A domain user cannot enroll for a Smart Card Logon (authentication) or Smart Card User (authentication plus e-mail) certificate unless a system administrator has granted the user access rights to the Certificate Template stored in the Active DirectoryTM service. This is done this way because enrollment for a smart card certificate must be a controlled procedure in the same manner that employee badges are controlled for identification and physical access purposes. The recommended method for enrolling users for smart card-based certificates and keys is through the enroll-on-behalf-of station that is integrated with Certificate Services. When an Enterprise Certification Authority (CA) is installed, the installation includes the enroll-on-behalf-of station. This station allows an administrator to act on behalf of a specific user to request and install a Smart Card Logon or Smart Card User certificate onto the user's smart card. The Enrollment Agents tab allows administrators to restrict enrollment agents from whom they can enroll on behalf of. To learn how this works, proceed with the following steps...
  • First, begin by selecting the option 'Restrict enrollment agents'.
  • Once the option 'Restrict enrollment agents' has been selected, a message will appear telling you that restrictions on delegated enrollment agents can only be enforced on Windows Server 2008 CAs and later.

  • Press 'OK'.
  • The first step is to define what enrollment agents we want to restrict. Now, the default setting is restricting all enrollment agents. In this case, we only want to restrict a certain group of designated enrollment agents. To do this, we first need to add the group of enrollment agents we want to restrict. We can do this by first pressing the 'Add...' button.
  • Then enter in the group of enrollment agents we want to restrict. In this case, we want to restrict the 'help desk' group.

  • Press 'OK'.
  • Now that the 'Help Desk' group has been added, we can now remove the group we don't want, which in this case is 'Everyone'. To do this, select the 'Everyone' group.
  • Then press the 'Remove' button.
  • The next step is to define what certificate templates the enrollment agents can enroll on behalf of. The default setting is all certificate templates, but we want to limit what certificate templates the enrollment agents can enroll for the other users. To do this, we first need to add the certificate template(s) that the enrollment agents can use. We can do this by first pressing the 'Add...' button.
  • Then select the certificate template. In this case, we are going to select 'Smartcard Logon'.
  • Once the certificate template is selected, press 'OK'.
  • Now that the certificate template has been added, we can now remove the <All> certificate templates option. To do this, select it.
  • And press the 'Remove' button.
  • This last step is to define what users the enrollment agents can enroll on behalf of. Now, the default setting is enrolling for all users. In this case, we only want the enrollment agents to be able to enroll for only a certain group of users. To do this, we first need to add these group of users . We can do this by first pressing the 'Add...' button.
  • Then enter in the group of users, which in this case is the 'sales' group.

  • Press 'OK'.
  • Now that the 'Sales' group has been added, we can now remove the group we don't want, which in this case is 'Everyone'. To do this, select the 'Everyone' group.
  • Then press the 'Remove' button.
  • For this example we are defining what group of users the enrollment agents are allowed to enroll on behalf of. However, you also have the option of defining what users the enrollment agents Cannot enroll on behalf of. We can do this by simply pressing the 'Deny' button.
  • So now you can see that the enrollment agents that belong to the 'Help Desk' group are denied the ability to enroll a Smartcard Logon certificate template on behalf of the 'Sales' group.
  • But for this situation, we do want the enrollment agents that belong to the 'Help Desk' group the ability to enroll a Smartcard Logon certificate template on behalf of the 'Sales' group. Therefore, we will press the 'Allow' button.
  • Now the enrollment agents that belong to the 'Help Desk' group have the ability to enroll a Smartcard Logon certificate template on behalf of the 'Sales' group.

  • Press 'OK'.
