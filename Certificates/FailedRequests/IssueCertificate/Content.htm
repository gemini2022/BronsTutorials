When a certificate gets denied or the request fails, an administrator still has the option to issue that certificate. To learn how to do this, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, expand the root node for the CA, which in this case is 'RootCA'.
  • Then select 'Failed Requests'.
  • Once 'Failed Requests' has been selected, look at the window to the right. There you may see a list of failed certificate requests.
  • When a certificate gets denied or the request fails, an administrator still has the option to issue that certificate. To do this, right-click the certificate.
  • Then select 'All Tasks'.
  • And then finally, select 'Issue'.
