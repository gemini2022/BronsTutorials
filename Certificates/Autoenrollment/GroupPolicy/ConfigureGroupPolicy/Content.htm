The last and final step is to configure Group Policy for autoenrollment. To learn how to do this, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then select 'Group Policy Management'...
  • Once in Group Policy Management, expand the forest node.
  • Then expand 'Domains'.
  • Then expand your domain, which in this case is 'bronstutorials.local'.
  • In order for autoenrollment to work, a GPO has to be configured for autoenrollment and then assigned to an OU. For this example, we will create a new GPO and link it to our 'Sales' OU. To do this, right-click the OU.
  • And select 'Create a GPO in this domain, and link it here...'.
  • First, we need to give our new GPO a name.
  • In this case, we will rename the GPO to 'Certificate Autoenrollment'.

  • Then press 'OK'.
  • We now have to configure our new GPO for autoenrollment. To do this, first expand the OU where the GPO was linked, in this case the 'Sales' OU.
  • Then right-click the new GPO.
  • And select 'Edit...'.
  • For this example, we are configuring autoenrollment for users. However, if we were also configuring autoenrollment for computers, then we would first expand 'Policies'.
  • Then expand 'Windows Settings'.
  • Expand 'Security Settings'.
  • And then select 'Public Key Policies'.
  • The setting for configuring autoenrollment for computers can be found here.
  • But for this example, we are only configuring autoenrollment for users, so we will collapse 'Computer Configuration'.
  • And then expand 'Policies' under 'User Configuration'.
  • Then expand 'Windows Settings'.
  • Then 'Security Settings'.
  • And then select 'Publick Key Policies'.
  • The setting we want to configure is 'Certificate Services Client - Auto-Enrollment'. To do this, right-click it.
  • And select 'Properties'.
  • Once in the properties of the autoenrollment setting, the first thing we need to do is enable it. To do this, first select the dropdown.
  • And select the option, 'Enabled'.
  • We now need to select this first option. This option allows us to enable autoenrollment for certificate renewal, issuance of pending certificate requests, and the automatic removal of revoked certificates from a user's certificate store.
  • Then select this next option. This option enables autoenrollment for issuance of certificates that supersede issued certificates.
  • And then finally this last option allows users to be notified when a certain percentage of the certificate's liftime remains.
  • In this case we will leave the percentage at the default of '10' %.

  • Press 'OK'.
