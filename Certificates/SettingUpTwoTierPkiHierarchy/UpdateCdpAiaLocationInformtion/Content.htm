In this step we'll be updating the CDP and AIA Location information. After the root CA is installed, you must configure the Authority Information Access (AIA) and CRL distribution point (CDP) extensions before the CA issues any certificates. The AIA extension specifies where to find up-to-date certificates for the CA. The CDP extension specifies where to find up-to-date CRLs that are signed by the CA. These extensions apply to all certificates that are issued by that CA. Configuring these extensions ensures that this information is included in each certificate that the CA issues so that it is available to all clients. This ensures that PKI clients experience the least possible number of failures due to unverified certificate chains or certificate revocations, which can result in unsuccessful VPN connections, failed smart card sign-ins, or unverified email signatures. As a CA administrator, you can add, remove, or modify CRL distribution points and the locations for CDP and AIA certificate issuance. Modifying the URL for a CRL distribution point only affects newly issued certificates. Previously issued certificates will continue to reference the original location, which is why you should establish these locations before your CA distributes any certificates. To learn how to configure the AIA and CDP extensions, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, right-click the root node for the CA, which in this case is 'RootCA'.
  • Then select 'Properties'.
  • Once in the properties of the root CA, select the 'Extensions' tab.
  • In the 'Extensions' tab, select the 'Select extension' dropdown.
  • It is here where you have the option to configure the CDP or the AIA.
  • In this case, we'll start first with configuring the CDP.
  • The first distribution point in the list is the local distribution point or local path. The CA uses the local CRL to validate all certificates that are generated before the certificates are issued.
  • With the local path selected, deselect the option 'Publish Delta CRLs to this location'. You should avoid publishing delta CRLs on an offline root CAs. In most cases, you do not revoke many certificates on an offline root CA, for this reason, a delta CRL is not needed.
  • Next, we need to remove these locations from the list. We will replace these locations with a custom path using a HTTP URL location. This will ensure that users and applications outside the organization can perform certificate validation. 
  • Begin removing these locations by first selecting the LDAP location.
  • Then press the 'Remove' button.
  • Once the 'Remove' button has been pressed, you'll need to confirm the removal. Press the 'Yes' button to continue.
  • Next, select the HTTP location.
  • And then press the 'Remove' button.
  • Again, once the 'Remove' button has been pressed, you'll need to confirm the removal. Press the 'Yes' button to continue.
  • And then finally, select the file address location.
  • And press 'Remove'.
  • Once again, press the 'Yes' button to confirm the removal.
  • Now, as we stated earlier, we will replace these locations with a custom path using a HTTP URL location. This will ensure that users and applications outside the organization can perform certificate validation. To do this, press the 'Add...' button.
  • And then start by entering 'http://' in the 'Location' field.
  • Followed by the address or the server name where CRL can be accessed. In this case we are using the name of the server that will be our Enterprise CA with IIS installed. However, in a production enviroment, using an outside web address ensures higher availability.
  • And finish by entering '/CertEnroll/<CaName><CRLNameSuffix><DeltaCRLAllowed>.crl'.

  • Press 'OK'.
  • Once the HTTP location has been created, select the 'Include in the CDP extension of issued certificates' option. This means that this CDP will be used in all certificates.
  • The CDP extension is now complete. We now have to configure the AIA extension. To do this, select the 'Select extension' dropdown.
  • And select the option 'Authority Information Access (AIA).
  • Like with the CDP extension, we will remove these unneeded locations.
  • Like before, begin removing these locations by first selecting the LDAP location.
  • Then press the 'Remove' button.
  • Again, once the 'Remove' button has been pressed, you'll need to confirm the removal. Press the 'Yes' button to continue.
  • Next, select the HTTP location.
  • And then press the 'Remove' button.
  • Press 'Yes'.
  • And then finally, select the file address location.
  • And press 'Remove'.
  • And press 'Yes'.
  • Now, press the 'Add...' button to add the HTTP location for the AIA extension.
  • And then start by entering 'http://' in the 'Location' field.
  • Followed by the address or the server name where CRL can be accessed. Again, in this case we are using the name of the server that will be our Enterprise CA with IIS installed. But, it is best to us an outside web address to ensure higher availability.
  • And finish by entering '/CertEnroll/<ServerDNSName>_<CAName><CertificateName>.crt'.

  • Press 'OK'.
  • Once the HTTP location has been created, select the 'Include in the AIA extension of issued certificates' option. This means that this AIA will be used in all certificates.
  • The AIA extension is now complete.

  • Press 'OK'.
  • And then finally, press 'Yes' to restart Certificate Services.
