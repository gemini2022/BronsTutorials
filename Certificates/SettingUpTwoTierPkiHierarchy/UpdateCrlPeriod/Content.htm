In this step we'll be updating the CRL Period. The CRL Period establishes the validity period for the base CRL. CRL stand for 'certificate revocation list'. The certificate revocation list is a list of certificates that have been revoked. The CRL Period essentially defines how long the certificate revocation list is valid for. By default, the CRL Period is set to 1 Week. Once the CRL has expired, the root CA server will need to republish the CRL to sub-CA server again. In this step, we are going to increase this value to eleminate the need to republish the CRL every week. To learn how to configure CRL Period, proceed with the following steps...
  • If the CRL Period was configured in the CAPolicy.inf before the CA installation, then there is no need to continue with this step. If however the CRL Period was NOT configured in the CAPolicy.inf before the CA installation, then proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, expand the root node for the CA, which in this case is 'RootCA'.
  • Then right-click 'Revoked Certificates'.
  • And select 'Properties'.
  • Once in the properties of the Revoked Certificates, we need to update two settings. The first setting is the CRLPeriodUnits, which is defined here in this input field.
  • In this case, we will change the CRLPeriodUnits to '20'.
  • The next setting we need to update is the CRLPeriod. To do this, select the dropdown.
  • Here you can see there is five options to choose from, "Hours", "Days", "Weeks", "Months" and "Years". In this case we will select the option 'Years'.
  • Next, Leave the 'Publish Delta CRLs' option NOT selected. We don't want to publish any delta CRLs.

  • This completes this step. However, there is a couple other methods in which the CRL Period can be updated. One other way is through the registry.
  • To update the CRL Period through the registry, begin by selecting 'Start' at the bottom left corner of your screen.
  • Then type 'regedit' in the search field.
  • Now click on 'regedit'. This will open the 'Registry Editor'.
  • Once the 'Registry Editor' is open, expand 'HKEY_LOCAL_MACHINE'.
  • Then expand 'SYSTEM'.
  • Then 'CurrentControlSet'.
  • Expand 'services'.
  • Frome here, you may need to scroll down a bit.
  • Scroll down until you find 'CertSvc', then expand it.
  • And then finally, expand 'Configuration'.
  • Once 'Configuration has been expanded, select the key that represents the name of your root CA. In this case, the name of the root CA is 'RootCA'.
  • Once the root CA key has been selected, in the window to the right, make your way down to the middle of the list.
  • The value that we are looking for is CRL Period. This value is broken up into two parts, 'CRLPeriod' and 'CRLPeriodUnits'.
  • First we will update the 'CRLPeriod'. To do this, double-click on 'CRLPeriod'.
  • By default, this value is set to 'Weeks'.
  • In this case, we want to change the value to 'Years'.

  • Press 'OK' when finished.
  • Now we want to update the 'CRLPeriodUnits'. To do this, double-click on 'CRLPeriodUnits'.
  • First, select the option 'Decimal'.
  • Then enter in the value in the 'Value data:' field. In this case, the value we will enter is '20'.

  • Press 'OK'.
  • The CRL Period is now updated. However, there is one more way in which these values can be configured and that is by command line.
  • To update the CRL Period by command line, begin by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Once in the command prompt, begin by entering in the command 'certutil -setreg ca\'.
  • Followed by 'CRLPeriod'.
  • Then enter in the CRLPeriod value, which in this case we will enter in 'Years'

  • When finished, press 'Enter'.
  • Next, enter in the command 'certutil -setreg ca\' again.
  • But this time enter in 'CRLPeriodUnits'.
  • Followed by the CRLPeriodUnits value, which in this case we will enter '20'.

  • When finished, press 'Enter'.
  • The CRLPeriod is now updated.
