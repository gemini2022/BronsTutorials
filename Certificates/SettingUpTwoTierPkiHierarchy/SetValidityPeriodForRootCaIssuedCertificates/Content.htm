In this next step we'll be setting the validity period for the root CA's issued certificates. Because this is a Standalone CA and not an Enterprise CA, we don’t have templates available to us that we can use to define the validity period for issued certificates. By default, the lifetime of a certificate that is issued by a Standalone CA is one year. For certificates that are issued by Enterprise CAs, the validity period is defined in the template that is used to create the certificate. But since this is a Standalone CA and has no templates, having a default lifetime of only one year is not nearly long enough. Therefore, it is a good idea to increase this value. It is recommended to make the validity period for these issued certificates half the lifetime of Root CA’s certificate validity period. For example, if you have configured the root CA's validity period for 20 years, then you would set the validity period of the certificate that this root CA will issue to a value of 10 years. To learn how to do this, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then type 'regedit' in the search field.
  • Now click on 'regedit'. This will open the 'Registry Editor'.
  • Once the 'Registry Editor' is open, expand 'HKEY_LOCAL_MACHINE'.
  • Then expand 'SYSTEM'.
  • Then 'CurrentControlSet'.
  • Expand 'services'.
  • Frome here, you may need to scroll down a bit.
  • Scroll down until you find 'CertSvc', then expand it.
  • And then finally, expand 'Configuration'.
  • Once 'Configuration has been expanded, select the key that represents the name of your root CA. In this case, the name of the root CA is 'RootCA'.
  • Once the root CA key has been selected, in the window to the right, make your way down to the bottom of the list.
  • The value that we are looking for is Validity Period. This value is broken up into two parts, 'ValidityPeriod' and 'ValidityPeriodUnits'. The value 'ValidityPeriod' can be set to either Hours, Days, Weeks, Months, or even Years. In this case, the value is set to 'Years', which is what we want. What needs to be changed is the 'ValidityPeriodUnits'. This value is set to '1', but in this case, we want to set it to '10'.
  • To do this, double-click on 'ValidityPeriodUnits'.
  • Then, select 'Decimal'.
  • Then enter in the value in the 'Value data:' field. In this case, the value we will enter is '10'.

  • Press 'OK'.
  • Once the Validity Period has been set, the certificate service needs to be restarted. However, before we do this, another way to set the Validity Period is by command line.
  • To do this, begin by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Once the command prompt is open, enter in the command 'certutil -setreg ca\'.
  • Followed by 'ValidityPeriodUnits'.
  • And then finally, enter in the 'ValidityPeriodUnits' value, which in this case will be '10'.
  • After the command has been entered, a message will appear stating that the certificate service may need to be restarted for changes to take effect.
  • There is a couple of ways in which the certificate service can be restarted. The first way is by command line. To do this, begin by entering in the command 'net stop certsvc'.

  • Then press 'Enter'.
  • The command that you entered will stop the certificate service.
  • Once the certificate service has stopped, the next thing we'll need to do is start the service back up.
  • To do this, enter in the command 'net start certsvc'.

  • Then press 'Enter'.
  • After running this command, the certificate service will start back up.
  • Another way to restart the certificate service is through the Certification Authority interface.
  • To do this, begin by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, right-click the root node for the CA, which in this case is 'RootCA'.
  • Then select 'All Tasks'.
  • And then 'Stop Service'.
  • Now that the service is stopped, we want to start the service back up. To do this, right-click the root node for the CA, which again is in this case 'RootCA'.
  • Then select 'All Tasks'.
  • And then finally 'Start Service'.
