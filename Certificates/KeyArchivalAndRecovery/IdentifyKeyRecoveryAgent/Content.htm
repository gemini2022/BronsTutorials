A key recovery agent is a person who is authorized to recover a certificate on behalf of an end user. Because the role of key recovery agents can involve sensitive data, only highly trusted individuals should be assigned to this role. To identify a key recovery agent, you must configure the Key Recovery Agent certificate template to allow the person assigned to this role to enroll for a key recovery agent certificate. To learn how to do this, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, expand the root node for the CA, which in this case is 'EnterpirseCA'.
  • Then right-click 'Certificate Templates'.
  • And select 'Manage'. This will open the 'Certificate Templates Console'.
  • Once in the 'Certificate Templates Console', right-click 'Key Recovery Agent'.
  • And select 'Properties'. In this case, there is no need to duplicate this certificate because all the options that require configuring are editable.
  • Once in the properties of the 'Key Recovery Agent' template, select the 'Security' tab.
  • The user we want to designate as a 'Key Recovery Agent' is not in the list. Therfore, we need to add this user to the list so we can give him the appropriate permissions for certificate enrollment. To do this, press the 'Add...' button.
  • Then type in the user, which in this case is 'bob smith'.

  • Press 'OK'.
  • This user already has 'Read', but needs 'Enroll'. To assign this user 'Enroll' permissions, select the 'Enroll' option under the 'Allow' column.
  • Now that the appropriate permissions have been assigned to our designated 'Key Recovery Agent', we next have to make some adjustments on how this certificate will be issued. To do this, select the 'Issuance Requirements' tab.
  • Once on the 'Issuance Requirements' tab, uncheck the option 'CA certificate manager approval'. By default, this option is selected, but by deselecting it, it will allow our designated 'Key Recovery Agent' to receive the Key Recovery Agent certificate without the need for CA certificate manager approval.
  • That's it! Press 'OK'.
