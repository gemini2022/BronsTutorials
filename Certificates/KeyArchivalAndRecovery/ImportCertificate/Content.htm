Recovered certificates and private keys can be imported from the password protected file by using the Certificate Import wizard. To learn how to do this, proceed with the following steps...
  • Begin first by logging into a user who lost thier key, which in this case is 'Jane Doe'. Then select 'Start' or the 'Start Orb' at the bottom left corner of your screen.
  • Then type 'certmgr.msc' in the search field.
  • Now click on 'certmgr'.
  • Once in certmgr, right-click on the 'Personal' folder.
  • And, select 'All Tasks'.
  • Then, select 'Import...'. This will launch the 'Certificate Import' wizard.
  • Once the 'Certificate Import Wizard' is open, press 'Next >' to begin the certificate import process.
  • On this screen, we need to provide the file that contains the recovered key. To do this, first press the 'Browse...' button.
  • Then enter in the path where the file is located. In this case the file is located in a shared folder at '\\dc1\certfiles'.
  • Once there, you may need to select the appropriate file format in order for the file to be visible.
  • In this case, the type of file we a looking for is 'Personal Information Exchange (*.pfx,*.p12)'.
  • Now that the correct file format is selected and the file is now visible, we can now select the file.
  • Once the file is selected, press the 'Open' button.
  • Now that the file has been entered, we can continue with the wizard.

  • Press 'Next >'.
  • On this screen, we need to provide the password we set for this file earlier in the previous step.

  • Press 'Next >'.
  • On this screen, we need to specify which store the certificate will be imported in. In this case, the default option is the setting we want.

  • Press 'Next >'.
  • And then finally, press 'Finish' to complete the wizard and import the certificate.
  • Once you press 'Finish' on the 'Certificate Import wizard', you should then get a notification that informs you that the import was successful.

  • Press 'OK'.
  • Now, to confirm that we imported the correct certificate, we can expand the 'Personal' store.
  • And then select 'Certificates'.
  • Once the 'Certificates' folder has been selected, you'll see in the window to the right, that a certificate has been issued to Jane Doe.
  • If we now scroll to the far right.
  • We can see that the certificate that was issued was the 'EFS with Key Archive' certificate. Jane Doe now has access again to her encryped files.
