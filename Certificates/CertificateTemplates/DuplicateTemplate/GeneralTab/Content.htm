The General tab is the first tab that appears when you duplicate an existing certificate template. On the General tab, you can configure the following attributes of the certificate template...
  • This is the display name that is shown in the Certificate Templates snap-in, Certificates snap-in, and in the Certification Authority snap-in. You can keep the default name that is provided for you, or you can rename this template.
  • For this example, we will rename the template to 'User v2'.
  • This is the name of the certificate template object created in the following container: CN=Certificate Templates,CN=Public Key Services,CN=Services,DC=ForestRootDomain.
  • This defines the validity period for an issued certificate. The validity period cannot be greater than the validity period of the CA's certificate. The minimum renewal period is 80 percent of the certificate lifetime or six weeks, whichever is greater.
  • This is the period of time before the validity period expires when the certificate will be renewed if re-enrollment is supported for the certificate template.
  • You can choose whether to publish the certificate to Active Directory® Domain Services (AD DS) based on the certificate template. The certificates are published as an attribute of the security principal contained in the subject of the issued certificate.
  • If the certificate template is published in AD DS, you can prevent re-enrollment if a valid certificate of the same certificate template exists for the security principal indicated in the subject.
  • For smart card renewal, this option enables the existing key to be used if a new key cannot be created. This helps prevent smart card renewal failures resulting from smart card capacity issues.
