The Certification Authority Web Enrollment role service provides a set of web pages that allow interaction with the Certification Authority role service. These web pages are located at https://<servername>/certsrv, where <servername> is the name of the server that hosts the hosts the CA Web Enrollment pages. To learn how to request a certificate using the Certification Authority Web Enrollment, proceed with the following steps...
  • Before a certificate can be requested through web enrollment, the web server hosting the cert enrollment site must first be configured for SSL. To do this, begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Internet Information Services (IIS) Manager'...
  • Once in Internet Information Services (IIS) Manager, select the server node, which in this case is 'ENTERPRISECA'.
  • Then, in the center window, double-click 'Server Certificates'.
  • Once 'Server Certificates' has been double clicked, in the center window, you will see some certificates that's already been added to the Web server. Along with these certificates, we also need to add a certificate that the Web server can use for SSL.
  • In the window to the far right of the screen, there are a few options in which we can obtain a certificate for SSL. In a real production enviroment, you would most likely obtain an SSL certificates from a trusted commercial certificate authority organization or SSL provider like Verisign or Digicert. However, for this demonstration, we will create our own certificate for our SSL configuration. In this case, we will create a domain certificate. A domain certificate is an internal certificate that does not have to be issued by an external certification authority. If your Windows domain has a server that acts as a CA, you can create a domain certificate. This approach helps you reduce the cost of issuing certificates and eases certificate deployment. To create a domain certificate, click on 'Create Domain Certificate...'.
  • After clicking on 'Create Domain Certificate...', a form will appear. On this form, you need to fill out some information regarding your company.
  • Once this information has been properly filled out, press 'Next'.
  • Next, we need to specify the CA in our domain that will sign the certificate. To do this, press the 'Select...' button.
  • Then select the CA that will be used to sign our domain certificate, which in this case is 'EnterpriseCA'.
  • Once the CA is selected, press 'OK'.
  • Finally, specify a friendly name for the certificate. In this case, we will name this certificate 'DomainCertificate'.

  • Press 'Finish'.
  • Once you press 'Finish' on the previous form, the domain certificate will appear in the list of server certificates.
  • The next step is to take our domain certificate we just created and apply it to the HTTPS binding. To do this, go back to the window located at the far left of the screen and expand 'Sites'.
  • Then select 'Default Web Site'.
  • Once you select 'Default Web Site', look to the window at the far right of the screen. There you will find an option called 'Bindings...'. Select this option to open the 'Site Bindings' form.
  • Once the 'Site Bindings' form is open, you will see a binding that has already been configured for HTTP.
  • We need to now configure a binding for HTTPS. To do this, first press the 'Add...' button.
  • Then select the dropdown labeled 'Type:'.
  • And select the option 'https'.
  • The next step is to select the certificate that will be used with the HTTPS binding. To do this, select the dropdown labeled 'SSL certificate:'.
  • And then select the domain certificate we created earlier.
  • Once the certificate has been selected, press 'OK'.
  • You should now see the binding that is configured for HTTPS.

  • Press 'Close'.
  • Now that HTTPS has been configured, we can now request a certificate using web enrollment. To do this, log on to a client computer and launch 'Internet Explorer'.
  • Once 'Internet Explorer' is open, begin by typing 'https://' in the address bar.
  • Followed by the name of the server that hosts the CA Web Enrollment pages, which in this case is 'enterpriseca/'. 
  • And then finally type 'certsrv'.

  • Press 'Enter'.
  • Because we didn't use a certificate from a trusted certificate authority like Verisign or Digicert for our HTTPS bindings, we are now receiving a message stating that the security certificate presented by this website is not issued by a trusted certificate authority. But because we know the reason behind this message, we can safely continue. To move on, click on the link 'Continue to this website (not recommended)'.
  • Now we are finally at the website where we can request a certificate via web enrollment. First, click on the link 'Request a certificate'.
  • Next, we need to select the type of certificate we want to enroll for. We can choose either 'User Certificate.
  • Or we can choose from a broader list of certificates by selecting the 'Advanced Certificate Request' option.
  • In this case, we will choose the option 'User Certificate'.
  • Once you choose the certificate you want to enroll for, a dialogue box will appear informing you that this web site is attempting to perform a digital certificate operation on your behalf and that you should only allow known web sites to perform digital certificate operations on your behalf. Since we know and trust this web site, we can move on.

  • Press 'Yes'.

 
  • Unlike a Standalone CA, because we are using an Enterprise CA, and the clicents requesting the certificates are part of the domain, there is no need to provide any user information. All we need to do is press the 'Submit >' button.
  • Once you press the 'Submit >' button, a dialogue box will appear informing you that this web site is attempting to perform a digital certificate operation on your behalf and that you should only allow known web sites to perform digital certificate operations on your behalf. Since we know and trust this web site, we can move on.

  • Press 'Yes'.
  • As long as the certificate you are requesting is not configured for 'CA certificate manager approval', then once you request the certificate, it should automatically be issued to you. All you need to do at this point is install the certificate.
  • Your new certificate has now been successfully installed!
