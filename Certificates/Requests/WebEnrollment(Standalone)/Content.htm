To learn how to unrevoke a certificate, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, expand the root node for the CA, which in this case is 'RootCA'.
  • Then select 'Revoked Certificates'.
  • Once 'Server Certificates' has been double clicked, in the center window, you will see the root CA certificate that's been added to the Web server. Along with this root CA certificate, we also need a certificate that the Web server can use for SSL.
  • In the window to the far right of the screen, there are a few options in which we can obtain a certificate for SSL. In a real production enviroment, you would most likely obtain an SSL certificates from a trusted commercial certificate authority organization or SSL provider like Verisign or Digicert. However, for this demonstration, we will create our own certificate for our SSL configuration. In this case, we will create a self-signed certificate. Self-signed certificates are certificates created on your computer. They're useful in environments where it's not important for an end user to trust your server, such as a test environment. Using a self-signed certificate will work for our purpose, but as you will see later, the browser will send a warning message to the user that the website may not be trustworthy. To create a self-signed certificate, click on 'Create Self-Signed Certificate...'.
  • Then 'Unrevoke Certificate'.
  • Once you select 'Unrevoke Certificate', you may receive a message telling you that the unrevoke command failed. This will be because the certificate you are trying to revoke wasn't revoked with the reason 'Certificate Hold'.
  • What this means is, before you revoke a certificate, you must provide a reason as to why you are revoking the certificate. If you choose the reason 'Certificate Hold', then and only then can a certificate become unrevoked.
