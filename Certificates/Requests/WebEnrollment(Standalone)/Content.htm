The Certification Authority Web Enrollment role service provides a set of web pages that allow interaction with the Certification Authority role service. These web pages are located at https://<servername>/certsrv, where <servername> is the name of the server that hosts the hosts the CA Web Enrollment pages. CA Web Enrollment is useful when you interact with a stand-alone CA because the Certificates Microsoft Management Console (MMC) snap-in cannot be used to interact with a stand-alone CA. To learn how to request a certificate using the Certification Authority Web Enrollment, proceed with the following steps...
  • Before a certificate can be requested through web enrollment, the web server hosting the cert enrollment site must first be configured for SSL. To do this, begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Internet Information Services (IIS) Manager'...
  • Once in Internet Information Services (IIS) Manager, select the server node, which in this case is 'STANDALONECA'.
  • Then, in the center window, double-click 'Server Certificates'.
  • Once 'Server Certificates' has been double clicked, in the center window, you will see the root CA certificate that's been added to the Web server. Along with this root CA certificate, we also need a certificate that the Web server can use for SSL.
  • In the window to the far right of the screen, there are a few options in which we can obtain a certificate for SSL. In a real production enviroment, you would most likely obtain an SSL certificates from a trusted commercial certificate authority organization or SSL provider like Verisign or Digicert. However, for this demonstration, we will create our own certificate for our SSL configuration. In this case, we will create a self-signed certificate. Self-signed certificates are certificates created on your computer. They're useful in environments where it's not important for an end user to trust your server, such as a test environment. Using a self-signed certificate will work for our purpose, but as you will see later, the browser will send a warning message to the user that the website may not be trustworthy. To create a self-signed certificate, click on 'Create Self-Signed Certificate...'.
  • After clicking on 'Create Self-Signed Certificate...', a form will appear. On this form, you need to specify a friendly name for the certificate. In this case, we will name the certificate 'SelfSignedCertificate'. Once you have given the certificate a friendly name, press 'OK'.
  • Once you press 'OK' on the previous form, the self signed certificate will appear in the list of server certificates.
  • The next step is to take our self signed certificate we just created and apply it to the HTTPS binding. To do this, go back to the window located at the far left of the screen and expand 'Sites'.
  • Then select 'Default Web Site'.
  • Once you select 'Default Web Site', look to the window at the far right of the screen. There you will find an option called 'Bindings...'. Select this option to open the 'Site Bindings' form.
  • Once the 'Site Bindings' form is open, you will see a binding that has already been configured for HTTP.
  • We need to now configure a binding for HTTPS. To do this, first press the 'Add...' button.
  • Then select the dropdown labeled 'Type:'.
  • And select the option 'https'.
  • The next step is to select the certificate that will be used with the HTTPS binding. To do this, select the dropdown labeled 'SSL certificate:'.
  • And then select the self signed certificate we created earlier.
  • Once the certificate has been selected, press 'OK'.
  • You should now see the binding that is configured for HTTPS.

  • Press 'Close'.
  • Now that HTTPS has been configured, we can now request a certificate using web enrollment. To do this, log on to a client computer and launch 'Internet Explorer'.
  • Once 'Internet Explorer' is open, begin by typing 'https://' in the address bar.
  • Followed by the name of the server that hosts the CA Web Enrollment pages, which in this case is 'standaloneca/'. 
  • And then finally type 'certsrv'.

  • Press 'Enter'.
  • Because we used a self signed certificate for the HTTPS bindings, we are now receiving a message stating that the security certificate presented by this website is not issued by a trusted certificate authority. If we had used a certificate from a trusted certificate authority like Verisign or Digicert for our HTTPS bindings, then we would of not recevied this message. But because we know the reason behind this message, we can safely continue. To move on, click on the link 'Continue to this website (not recommended)'.
  • Now we are finally at the website where we can request a certificate via web enrollment. First, click on the link 'Request a certificate'.
  • Next, select the type of certificate you want to enroll for. You can choose between two common certificates, 'Web Browser Certificate' or 'E-Mail Protection Certificate'.
  • Or you can choose from a broader list of certificates by selecting the 'Advanced Certificate Request' option.
  • In this case, we will choose the option 'Web Browser Certificate'.
  • Once you choose the certificate you want to enroll for, a dialogue box will appear informing you that this web site is attempting to perform a digital certificate operation on your behalf and that you should only allow known web sites to perform digital certificate operations on your behalf. Since we know and trust this web site, we can move on.

  • Press 'Yes'.

 
  • The next step is to fill out a form regarding information about you and your company.
  • Once the form has been completed, press 'Submit >'.
  • After you press the 'Submit >' button, you will be brought to a page informing you that your request has been received and now you must wait for an administrator to issue the certificate you requested. In the mean time, we can view the status of our pending request. To do this, select the address bar at the top of the browser window.
  • And then type in 'https://standaloneca/certsrv' so we can get back to the first page of the web enrollment.

  • Press 'Enter'.
  • And now this time, select the link 'View the status of a pending certificate request'.
  • And here is the certificate request we submitted. If there were more than one request submitted, they would show up in the list here. In this case, there is just the one we submitted earlier.
  • Now, click on the link to view the status of the request.
  • Once you click on the link, a dialogue box will appear informing you that this web site is attempting to perform a digital certificate operation on your behalf and that you should only allow known web sites to perform digital certificate operations on your behalf. Since we know and trust this web site, we can move on.

  • Press 'Yes'.
  • Good! An administrator has approved and issued our certificate request.
  • If however an administrator had not yet approved and issued our certificate request, this is the message we would of received.
  • But since an administrator has issued our certificate request, we can now install the certificate we requested. To do this, click on the link 'Install this certificate'.
  • Our new certificate has now been successfully installed!
