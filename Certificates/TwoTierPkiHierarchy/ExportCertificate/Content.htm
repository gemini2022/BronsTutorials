In this step, we're going to export the certificate from the certificate store that was just issued and place it on the desktop so that later it can be transerred to the Enterprise Subordinate CA server. To learn how to export the certificate, proceed with the following steps...
  • If not already, log into the offline Standalone Root CA server.
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, expand the root node for the CA, which in this case is 'RootCA'.
  • Then select 'Issued Certificates'.
  • Once 'Issued Certificates' has been selected, look at the window to the right. There you will see the certificate we issued in the previous step. This certificate needs to be installed on the Enterprise Subordinate CA server. In order to do this, the first step in the process is to export this certificate.
  • To export the certificate, begin by double-clicking on it.
  • Once the certificate is double-clicked, a dialogue box will appear displaying information about the certificate. On the first tab, the 'General' tab, one thing that is worth pointing out is the section displaying how long this certificate is valid for. This information was defined when we set the validity period in a previous step. If you recall, we set the validity period for ten years.
  • If we now select the 'Certification Path' tab.
  • This tab displays the certificate server hierarchy. In our case, the path should be displayed as a two level hierarchy, Root CA at the Top and Sub CA (Issuing CA) under the RootCA.
  • If we move on to the 'Details' tab.
  • It is on this tab where we will begin the certificate export process. To do this, click on the 'Copy to File...' button.
  • Once the 'Copy to File..' button has been clicked, the 'Certificate Export Wizard' will appear.

  • Press 'Next >'.

  • If you are exporting certificates for import onto a computer running Windows, PKCS #7 format is the preferred export format. This format preserves the chain of certification authorities (CAs), or the certification path, of any certificate that includes countersignatures associated with signatures. If you are exporting certificates for import onto a computer running another operating system, it is possible that the PKCS #7 format is supported. If it is not supported, the DER-encoded binary format or the Base64-encoded format is provided for interoperability. In this case, we will be importing the certificate onto a computer running Windows, therefore we will select the 'Cryptographic Message Syntax - PKCS #7 Certificate (.P7B)' option.
  • Also, be sure to select the 'Include all certificates in the certification path if possible' option as well.
  • Press 'Next >'.
  • On this screen, we will choose the location where the certificate will be exported. To do this, begin by pressing 'Browse...'.
  • The certificate doesn't need to be exported to any particular location at this point, so in this case, we save it to the desktop. To do this, select 'Desktop'.
  • Then give the certificate a name. In this case, we will name it 'SubCert'.
  • Then press 'Save'.
  • Once the location and the name of the certificate has been defined, press 'Next >'.
  • And finally, press 'Finish' to export the certificate..
  • Once the certificate has been successfully exported, press 'OK'.
