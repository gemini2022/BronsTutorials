To learn how to view a certificate that's been issued, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'Certification Authority'...
  • Once in Certification Authority, expand the root node for the CA, which in this case is 'RootCA'.
  • Then select 'Issued Certificates'.
  • Once 'Issued Certificates' has been selected, look at the window to the right. There you will see a list of all the certificates that have been issued. In this case, there is only one certificate that has been issued.
  • To view this certificate, right-click it.
  • And select 'Open'.
  • Once 'Open' has been selected, a dialogue box will appear displaying information about the certificate. On the first tab, the 'General' tab, we can first see what the purpose of this certificate is for. In this case, this certificate is intended for all application policies.
  • Here we can see who this certificate was issued to.
  • And who this certificate was issued by.
  • And then finally, the validity of the certificate.
  • If we now select the 'Certification Path' tab.
  • This tab displays the certificate server hierarchy. In our case, the path is displayed as a two level hierarchy, Root CA at the Top and Sub CA (Issuing CA) under the RootCA.
  • If we move on to the 'Details' tab.
  • If we first select the dropdown.
  • We can see all the options that can be chosen on how we want to view the details of the certificate. However, for this example, we will leave it on the default of '<All>'.
  • If we select one of the fields on the details list.
  • We can see information about that feild also displayed here.
  • And then finally, we can copy this certificate to a file if needed. To do this, click on the 'Copy to File...' button.
  • Once the 'Copy to File..' button has been clicked, the 'Certificate Export Wizard' will appear.

  • Press 'Next >'.

  • If you are exporting certificates for import onto a computer running Windows, PKCS #7 format is the preferred export format. This format preserves the chain of certification authorities (CAs), or the certification path, of any certificate that includes countersignatures associated with signatures. If you are exporting certificates for import onto a computer running another operating system, it is possible that the PKCS #7 format is supported. If it is not supported, the DER-encoded binary format or the Base64-encoded format is provided for interoperability. In this case, we will be importing the certificate onto a computer running Windows, therefore we will select the 'Cryptographic Message Syntax - PKCS #7 Certificate (.P7B)' option.
  • Also, be sure to select the 'Include all certificates in the certification path if possible' option as well.
  • Press 'Next >'.
  • On this screen, we will choose the location where the certificate will be exported. To do this, begin by pressing 'Browse...'.
  • In this case, we will save the certificate to the desktop. To do this, select 'Desktop'.
  • Then give the certificate a name. In this case, we will name it 'SubCert'.
  • Then press 'Save'.
  • Once the location and the name of the certificate has been defined, press 'Next >'.
  • And finally, press 'Finish' to export the certificate..
  • Once the certificate has been successfully exported, press 'OK'.
