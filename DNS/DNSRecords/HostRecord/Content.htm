A records map a FQDN (fully qualified domain name) to an IP address and are the most often used record type in any DNS configuration. A records can be configured in a domain for a specific host such as www.example.com or for the root record (sometimes represented with an @ symbol) of a domain such as example.com. To learn how to create an A record, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DNS'...
  • Once in 'DNS', select 'View' on the top menu of the 'DNS Manager' window.
  • Then select 'Advanced'. This will unhide any additional options that are normally hidden.
  • Now expand your server, in this case it is 'DC1'.
  • Then expand 'Forward Lookup Zones'.
  • Once 'Forward Lookup Zones' has been expanded, select a zone. In this case, we're going to select the 'bronstutorials.local' zone.
  • Now that the zone has been selected, right-click it.
  • And select 'New Host (A or AAAA)...'.
  • For this example, we will create an A record for our router, so we will name the A record 'Router1'.
  • The next step is a matter of entering in the IP address of the router. In this case, the IP address is '192.168.0.1'.
  • This option, when selected, will automatically create an associated pointer record in the reverse lookup zone for this A record. This will allow reverse lookups to be performed on this IP address, which are great when troubleshooting problems.
  • When this option is selected, it permits the resource record to be updated dynamically. When the update is performed, the host that requests the update is granted permission to modify the resource record, but all other nonadministrative permissions are removed from the access control list (ACL) that protects the resource record. This makes it possible for the administrator to create a secure resource record for a host that is not yet online and still enable the resource record to be updated dynamically when the host obtains its IP address through Dynamic Host Configuration Protocol (DHCP).
  • Here at the bottom is the time to live value. This value is what was made present when the 'Advanced' option in the 'View' menu was switched on. For this DNS record, the time to live is set to one hour. This means that a DNS server will only be able to keep this DNS record in its cache for one hour before it is forced to contact an athoritative DNS server to obtain the DNS record again. This value can be edited, but you should consider how often the record will be changed. Because of caching, changes to a DNS record will not reach the entire network until the original TTL has expired - a good reason for setting a short TTL. However, caching helps reduce network traffic. The longer the TTL, the longer the record will live in other DNS server caches around the world, and so fewer requests to the original DNS server are needed - a good reason for setting a long TTL. Generally, for a record pointing to a server/device with a static IP address and no need for quick updates, a TTL of one day is a good starting point. However, if the record is for a host with a dynamic IP address or for server which is part of a failover set, you should be using a TTL value of a few minutes or less.
  • Press 'Add Host'.
  • Once the 'Add Host' button has been pressed, the A record will be created, including the pointer record (if that option was selected). If for any reason either record could not be created, an error message would be generated. However, if the records were successfully created, a message will be displayed informing you of the success.

  • Press 'OK'.
