A primary server has the "master copy" of a zone, and secondary servers keep copies of the zone for redundancy. When changes are made to zone data on the primary DNS server, these changes must be distributed to the secondary DNS servers for the zone. This is done through zone transfers. Most DNS servers automatically notifies secondary servers whenever changes are made through a NOTIFY request, and most DNS servers will request a Zone Transfer whenever such a notification is received. Secondary servers also periodically check for changes by querying the primary server for the SOA-record of the zone, and checking the serial number. In addition to whatever other changes are made to a zone and its records, the serial number of the SOA-record must always be incremented. The periodic polling by the secondary servers is controlled by the refresh, retry, and expire parameters of the SOA-record. The secondary server waits for the "refresh" interval before checking with the primary for a new serial number. If this check cannot be completed, new checks are started every "retry" interval. If the secondary finds it impossible to perform a serial check within the "expire" interval, it discards the zone. When the poll shows that the zone has changed (higher serial number), the secondary server will fetch a fresh copy of the zone through a zone transfer request. The actual zone transfer operation transfers all the records in the zone from the primary to the secondary server.
  • In order for a zone to be allowed to transfer its zone data, the option 'Allow zone transfers:' must be first selected.
  • Once the 'Allow zone transfers:' option has been selected, you then have the option of choosing what server is allowd to receive a copy of the zone. The first option, 'To any server' allows any server anywhere to request a copy for the zone data. This option is not very secure.

 
  • This next option, 'Only to servers listed on the Name Servers tab' limits transfers to servers you specify. This is a more secure setting than the previous option 'To any server', because it limits zone transfers to other servers from the same zone.
  • And finally this last option, 'Only to the following servers' allows you to specify exactly which servers are allowed to request zone transfers. This list can be larger or smaller than the list specifed on the Name Servers tab.
  • Once the option 'Only to the following servers' has been selected, you can then press the 'Edit' button.
  • Once the 'Edit' button on the previous form has been pressed, you can then begin to add the servers you want to receive zone transfers. To do this, first select the clickable area.
  • Then enter in the IP address of the server you want to receive zone transfers and then press the 'Enter' key.
  • You can add or remove as many servers as needed. Once complete, press 'OK'.
