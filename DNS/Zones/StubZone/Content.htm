Stub zones contain only NS Name Servers (DNS server) records of their authoritative DNS domains. Years ago, prior to Stub or Conditional Forwarders, there weren't many options to handle this other than to use Secondary Zones and keep copies of each others zones via zone transfers. While the solution worked well in regards to name resolution, it was not the best security-wise, due to trust level between partners, because zone data is fully exposed at the partner. This became a security concern because the partner is able to see all of their business partner's records. When the zone was transferred to partners, who knows what they were doing with the information. If the information was made public, attackers would have a field day with all of the IPs for the networked devices. When stub zones were made available, it became a solution to overcome this security issue. What is also beneficial about Stubs, is you can AD integrate them instead of manually creating a Stub on each indivdual DC. This way the zone will be available domain or forest-wide, depending on replication scope. However, some may say due to the fact that the SOA records are included in the zone file, it may be a concern that the SOA and NS data is exposed. In such high security concerns, the better solution would be to use a Conditional forwarder. To learn how to create a stub zone, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DNS'...
  • Once in DNS, expand your server, in this case it is 'DC1'.
  • Then select 'Forward Lookup Zones'.
  • Once 'Forward Lookup Zones' has been selected, right-click it...
  • And select 'New Zone...' to launch the 'New Zone Wizard'.
  • The first screen of the wizard is the welcome screen. Press 'Next >' to continue.
  • On this next screen, we are asked to select the type of zone we want to create. In this case, we will select the option 'Stub zone', since the type of zone we want to create is a stub zone.

  • Press 'Next >'.
  • Here on this next screen, we are to enter in the zone we want to create the stub zone for. In this case, 'otherguystutorials.local'.
  • For this screen, because we didn't set this zone as an Active Directory-Integrated zone, we are now being asked which file to store this zone in. We have the choice of creating and storing the zone information in a new file, or storing the zone information in an existing file. In this case, we will accept the default option of creating a brand new file for this zone.

  • Press 'Next >'.
  • On this next screen, we need to enter in the name or the IP address of the 'otherguystutorials' DNS server.
  • In this case, we will enter in the IP address of the DNS server, which will be '192.168.0.2'. Once the name or the IP address has been entered, press the 'Enter' key.
  • After the 'Enter' key has been pressed, the DNS server that you entered should get resolved. If this isn't the case, confirm the remote DNS server is authoritative for the zone you are attempting to configure the stub for.

  • Press 'Next >'.
  • This will complete the wizard. Press 'Finish' to create the stub zone.
