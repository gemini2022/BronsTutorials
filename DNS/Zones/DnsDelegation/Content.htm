When the authoritative name server for a domain receives a request for a subdomain's records and responds with NS records for other name servers, that is DNS delegation. Essentially it is saying "I am passing on authority for this subdomain to another collection of name servers, go ask them for the details." For example, "com" is a domain that delegates all (or perhaps almost all) of its subdomains to other name servers. When a request is received at a "com" name server for "google.com", the "com" name server responds with NS records pointing to Google's name servers. You can divide your Domain Name System (DNS) namespace into one or more zones. You can then delegate management of part of your namespace to another location or department in your organization by delegating the management of the corresponding zone. To learn how to configure delegation for your zones, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DNS'...
  • Once in DNS, expand your server, in this case it is 'DC1'.
  • Then expand 'Forward Lookup Zones'.
  • Once 'Forward Lookup Zones' has been expanded, select a zone. In this case, we're going to select the 'bronstutorials.local' zone.
  • Now that the zone has been selected, right-click it.
  • And select 'New Delegation...' to launch the 'New Delegation Wizard'.
  • The first screen of the wizard is the welcome screen. Press 'Next' to continue.
  • On this screen, we need to enter in the name of the delegated domain. In this case, 'West'.

  • Press 'Next >'.
  • On this screen, we need to enter in the address of a DNS server that is authoritative for the west.bronstutorials.local zone. To do this, first press the 'Add...' button.
  • After pressing 'Add...' on the previous screen, we then need to add the fully qualified domain name of the authoritative server for the west.bronstutorials.local zone. In this case it's 'dc1.west.bronstutorials.local'.
  • If we then press the 'Resolve' button...
  • We get an error message that states, "The IP address for this server cannot be found". This is because currently there is no way to resolve request for the 'west' DNS domain.

  • Press 'OK'.
  • To get around this, we can just manually enter in the IP address of the DNS server, rather than relying on the 'Resolve' button to try to resolve the name. If we wanted to use the name, what we would need to do is create another host record in the 'bronstutorials.local' zone for the 'west' DNS server. This is referred to as a 'Glue' record. 'Glue' records allow the DNS server to resolve host names that it cannot resolve using other methods.

  • After entering in the IP address, which in this case is '192.168.0.2', press 'Enter'.
  • Then press 'OK'.
  • Now press 'Next >' to continue.
  • This will complete the wizard. Press 'Finish' to create the new delegation.
  • Notice that when the delegation is created, in the zone file, it appears as grey. This indicates that it is a delegation and thus not stored on this DNS server.
