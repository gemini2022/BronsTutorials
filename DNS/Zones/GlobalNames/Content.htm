A common requirement in computer networks is the ability to resolve simple, single-label names. The use of single-label names makes it possible for a computer to access hosts such as file and Web servers by using short, easy-to-remember names instead of the fully qualified domain names. To make the use of single-label names possible, many networks deploy Windows Internet Name Service (WINS) technology in their environment. As a name resolution protocol, WINS is an alternative to DNS. It is an older service that uses NetBIOS over TCP/IP (NetBT). WINS and NetBT do not support Internet Protocol version 6 (IPv6) protocols; therefore, they are being phased out in many networks. To help network administrators migrate to DNS for all name resolution, the DNS Server role in Windows Server 2008 supports a specially named zone, called GlobalNames. By deploying a zone with this name, you can have the static, global records with single-label names, without relying on WINS. These single-label names typically refer to records for important, well-known and widely-used servers—servers that are already assigned static IP addresses and that are currently managed by IT-administrators using WINS. The GlobalNames zone is not designed to be a complete replacement for WINS. You should not use the GlobalNames zone to support the name resolution of records that are dynamically registered in WINS, records which typically are not managed by IT administrators. Support for these dynamically registered records is not scalable, especially for larger customers with multiple domains or multiple forests. To learn how to configure a GlobalNames zone, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then open the 'Command Prompt'.
  • Once the command prompt is open, the first step for configuring global names support is to run the command 'DNSCmd'. This command needs to be run on every DNS server that will be answering DNS single-label names. This will allow the DNS server to start answering queries for single-label names.
  • Next, you need to enter in the DNS server that you want to configure. In this case, 'dc1' is our DNS server.
  • Then you need to enter in '/Config'.
  • Followed by '/EnableGlobalNamesSupport'.
  • And then finally, followed by a '1'. Then press 'Enter'.
  • This command changes the configuration of the DNS server so that it will respond when asked for single-label names.
  • Next, we need to configure the DNS server. To do this, select 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DNS'...
  • Once in DNS, expand your server, in this case it is 'DC1'.
  • Then select 'Forward Lookup Zones'.
  • Once 'Forward Lookup Zones' has been selected, right-click it...
  • And select 'New Zone...' to launch the 'New Zone Wizard'.
  • The first screen of the wizard is the welcome screen. Press 'Next >' to continue.
  • Next, we need to make sure that 'Primary zone' is selected, and also the option 'Store the zone in Active Directory (available only if DNS server is a writeable domain controller)' is selected.

  • Press 'Next >'.
  • This next screen will determine the replication of the DNS zone. Since this will be the 'GlobalNames' zone, we need to make sure that the top option, 'To all DNS servers running on domain controllers in this forest' is selected. GlobalNames can be used throughout the entire forest. In fact, that is the whole idea, to have a single name that can be used throughout the forest without having to worry about which domain the client is in. For this reason, make sure that you always replicate this data to all DNS servers in the forest.

  • Press 'Next >'.
  • Here on this next screen, we are to enter in the name of the zone. In this case, the name of the zone needs be 'GlobalNames'.

  • Press 'Next >'.
  • On this screen, configure the zone with the option 'Do not allow dynamic updates'. If a user was able to make changes to the GlobalNames zone, they would be able to create a DNS record that would be used in all domains in the forest. This would be a security concern, so dynamic updates should be switched off.

  • Press 'Next >'.
  • This will complete the wizard. Press 'Finish' to create the GlobalNames zone.
  • Now that the GlobalNames zone has been created, we now have to go into the zone. To do this, first expand 'Forward Lookup Zones'.
  • And select the 'GlobalNames' zone you just created.
  • Then, to the far right of the window, right-click anywhere in the white space...
  • And select 'New Alias (CNAME)...' to create a CNAME.
  • In the Alias text box, enter the required name for the server/workstation. In this case we are creating a global name for the intranet server.
  • After the name has been entered, press 'Browse...'.
  • From here, we need to select the server/workstation that will be associated with this global name. To start, we will first double-click on our DNS server 'DC1'.
  • Then double-click 'Forward Lookup Zones'.
  • Then double-click the zone of where the DNS records of the server/workstation you're trying to reach is located. In this case it's 'bronstutorials.local'.
  • And then finally, select the A record for that server/workstation and press 'OK'.
  • Now when we press 'OK' on this form, the CNAME record will be created.
  • If we select the 'GlobalNames' zone
  • And then take a look at the far right of the screen, we will notice that the CNAME record for 'intranet' is pointing to the DNS record 'dc1', just as we configured it. So now, if changes need to be made, these changes can be made to 'dc1's' DNS record, either by an administrator or dynamically, and never have the need to update our global name.
  • One last thing that needs to be done. The DNS server has had a configuration change to enable GlobalName support, however this change does not take effect until the DNS server has been restarted. To do this, right-click on the DNS server, in this case it's 'DC1'.
  • And select 'All Tasks'...
  • Then 'Restart' to restart the DNS server.
  • Once the DNS server has been restarted, Global Names support will be enabled.
