An option available with Windows DNS server is the ability to integrate your DNS data into Active Directory. Effectively, this means that you can store the contents of the zone file in Active Directory as a hierarchical structure, rather than in a text file on the DNS server. Integrating DNS into Active Directory means that the DNS structure is replicated among all DCs of a domain. Each DC holds a writeable copy of the DNS data. The DNS objects stored in Active Directory could be updated on any DC via LDAP operations or through DDNS against DCs that are acting as DNS servers. This effectively makes the entire set of DCs act like primary name servers, where each DC can write to the zone and issue authoritative answers for the zone. To learn how to configure DNS to integrate with Active Directory, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DNS'...
  • Once in DNS, expand your server, in this case it is 'DC1'.
  • Then select 'Forward Lookup Zones'.
  • Once 'Forward Lookup Zones' has been selected, right-click it...
  • And select 'New Zone...' to launch the 'New Zone Wizard'.
  • The first screen of the wizard is the welcome screen. Press 'Next >' to continue.
  • On this next screen, we are asked to select the type of zone we want to create. In this case, we will leave it on the default option of 'Primary zone'.

  • At the bottom of the screen, we will leave the option 'Store the zone in Active Directory (available only if DNS server is a writeable domain controller)' selected. This option, when selected, will allow this zone to be configured as an Active Directory-Integrated zone.

  • Press 'Next >'.
  • Because this zone is being configured to be integrated with Active Directory, we now have to decide on the scope of the replication throughout Active Directory. For example, the first option will replicate the zone data to all DNS servers running on domain controllers in the forest.
  • The next option, which is the default option, will replicate the zone data to all DNS servers running on domain controllers in the domain.
  • And this option will replicate the zone data to all domain controllers in this domain. Meaning, even if you do not install DNS on all DCs, ALL DCs in that domain will participate in replication of the zone.
  • And finally, this last option, which will replicate the zone data to all domain controllers specified in the scope of this directory partition. This option is grayed out if there are no custom DNS application partitions.

  • When using custom application partition for DNS Zones, you have 100% control of exactly which domain controllers participate in replication of the DNS information, and it makes it easier to smoothly retire a DNS server.

  • Press 'Next >'.
  • Here on this next screen, we are to enter in the name of the zone. In this case, the zone will be 'bronstutorials.local'.

  • Press 'Next >'.
  • This next screen will ask if we will be using dynamic updates. Dynamic updates enable DNS client computers to register and dynamically update their resource records with a DNS server whenever changes occur. Since this is an Active Directory-Integrated zone, we have the option of allowing only secure dynamic updates, which ensures that the host initiating the record creation or update has been authenticated by Active Directory.
  • With this option, both authenticated Active Directory clients and non-Active Directory Clients can create and update DNS records. This option isn't recommended because it allows rogue clients to create DNS records with false information. A rogue DNS client can impersonate a server by updating the server's A record with its own IP address, thereby redirecting client computers to a fraudulent server.
  • With this option, all DNS records must be entered manually. This option helps secure the enviroment, but on a network with many hosts that must be accessed by name, and on networks using DHCP, it's an administrative nightmare. However, this option does work well for a DNS server that manages names for public resources, such as web and mail servers with addresses that are usually assigned statically and don't change often.

  • Press 'Next >'.
  • This will complete the wizard. Press 'Finish' to create the Active Directory-Integrated zone.
