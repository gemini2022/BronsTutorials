Normal DNS queries are forward lookup queries, they request the IP address that corresponds to a fully qualified domain name. A reverse lookup is the opposite of a forward lookup, it returns the fully qualified domain name of a host based on its IP address. Reverse lookups are possible because of a special domain called the in-addr.arpa domain, which provides a separate fully qualified domain name for every possible IP address on the Internet. To enable a reverse lookup for a particular IP address, all you have to do is create a PTR record in a reverse lookup zone (a zone that is authoritative for a portion of the in-addr.arpa domain). The PTR record maps the in-addr.arpa domain name for the address to the host’s actual domain name. The technique used to create the reverse domain name for a given IP address is pretty clever. It creates subdomains beneath the in-addr.arpa domain by using the octets of the IP address, listing them in reverse order. For example, the reverse domain name for the IP address 207.126.67.129 is 129.67.126.207.in-addr.arpa. Why list the octets in reverse order? Because that correlates the network portions of the IP address (which work from left to right) with the subdomain structure of DNS names (which works from right to left). To learn how to create a reverse lookup zone, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DNS'...
  • Once in DNS, expand your server, in this case it is 'DC1'.
  • Then select 'Reverse Lookup Zones'.
  • Once 'Reverse Lookup Zones' has been selected, right-click it...
  • And select 'New Zone...' to launch the 'New Zone Wizard'.
  • The first screen of the wizard is the welcome screen. Press 'Next >' to continue.
  • The procedure is much the same as creating a forward lookup zone. Like a forward lookup zone, you can choose if the zone is a 'Primary zone', 'Secondary zone', 'Stub zone', and if it is to be stored in Active Directory. In this case, we will leave it on the default option of 'Primary zone'.

  • Press 'Next >'.
  • On this next screen, we need to decide if the reverse lookup zone is for IPv4 or IPv6. For this zone, we will leave it on the default of 'IPv4 Reverse Lookup Zone' and move on.

  • Press 'Next >'.

 
  • On this screen, we need to decide which network ID this reverse lookup zone applies to. For this network ID, we will use '192.168.0'. The 'New Zone Wizard' will create the required network mask based on the network ID that you have entered.

  • Press 'Next >'.
  • For this screen, because we didn't set this zone as an Active Directory-Integrated zone, we are now being asked which file to store this zone in. We have the choice of creating and storing the zone information in a new file, or storing the zone information in an existing file. In this case, we will accept the default option of creating a brand new file for this zone.

  • Press 'Next >'.
  • This next screen will ask if we will be using dynamic updates. Dynamic updates enable DNS client computers to register and dynamically update their resource records with a DNS server whenever changes occur. Since we didn't set this zone as an Active Directory-Integrated zone, we don't have the option of allowing only secure dynamic updates. In this case we will just leave it on the default option of 'Do not allow dynamic updates' and move on.

  • Press 'Next >'.
  • This will complete the wizard. Press 'Finish' to create the reverse lookup zone.
