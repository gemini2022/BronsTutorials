Secondary zones are the read-only copies of the primary zone of the DNS domain. Therefore, unlike a primary zone, one DNS domain can have multiple secondary zones. When secondary zones are configured for a domain, administrators can schedule zone transfers in which DNS records are transferred from the primary (read/write) zone to the secondary (read-only) zone. DNS zone transfers are one-way in this case, i.e. from primary to the secondary zone. Since secondary DNS zones are the read only copies of the primary DNS zone and do not allow DNS records to be created on them, no unauthorized person or hacker can create false DNS records or poison the DNS database. This is the reason why secondary zones are mostly installed and configured to face the external networks, i.e. the Internet. In production environments, servers that are configured to run secondary DNS zones are located in the Demilitarized Zones or DMZ. To learn how to create a secondary zone, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DNS'...
  • Once in DNS, expand your server, in this case it is 'DC1'.
  • Then select 'Forward Lookup Zones'.
  • Once 'Forward Lookup Zones' has been selected, right-click it...
  • And select 'New Zone...' to launch the 'New Zone Wizard'.
  • The first screen of the wizard is the welcome screen. Press 'Next >' to continue.
  • On this next screen, we are asked to select the type of zone we want to create. In this case, we will select the option 'Secondary zone'. Another thing to note is that since secondary zones are the read-only copy of the primary zone, secondary zones can NOT be configured to be Active Directory-Integrated.

  • Press 'Next >'.
  • Here on this next screen, we are to enter in the name of the zone. The zone name is basically the DNS name we want to replicate. So in this case, the zone name we will enter will be 'east.bronstutorials.local', since this is the zone we want to replicate.

  • Press 'Next >'.
  • On this screen, we need to enter in the name or the IP address of the DNS server that this secondary zone will get information from. It should be pointed out here, that even though the DNS server we are getting the information from holds a primary zone, there is nothing stopping you from getting this information from a DNS server that holds a secondary zone.
  • In this case, we will enter in the name of the DNS server, which will be 'DC1'. Once the name or the IP address has been entered, press the 'Enter' key.
  • After the 'Enter' key has been pressed, the DNS server that you entered should get resolved. If this isn't the case, check to see if the DNS server you are trying to reach is configured to allow zone transfers.

  • Press 'Next >'.
  • This will complete the wizard. Press 'Finish' to create the secondary zone.
