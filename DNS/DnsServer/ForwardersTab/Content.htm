The Forwarders tab allows you to configure forwarders. A forwarder is a Domain Name System (DNS) server on a network that is used to forward DNS queries for external DNS names to DNS servers outside that network. By using a forwarder, you can manage name resolution for names outside your network, such as names on the Internet, which can improve the efficiency of name resolution for the computers in your network
  • First, you'll notice the option 'Use root hints if no forwarders are available' is currently greyed out. This is because no forwarders have been configured.
  • To be able to edit the option, at least one forwarder has to be configured. To configure a forwarder, first press the 'Edit...' button
  • After pressing the 'Edit...' button on the previous screen, you will be able to enter in the IP addresss of the DNS server you want to forward the DNS queries to.
  • In this case, we will enter in the IP address of the DNS server from our ISP.

  • Once the IP address has been entered, press the 'Enter' key.
  • Your local ISP should have at least two DNS servers for redundancy. Some ISP will even have more.
  • So in this case, we will enter in the second IP address of the DNS server from our ISP.

  • Once the IP address has been entered, press the 'Enter' key.
  • After you have completed entering all the IP address of the DNS servers you want to use as forwarders, press 'OK'.
  • You can see now that the option 'Use root hints if no forwarders are available' is now enabled. In most cases you'll want this option selected because if for any reason the forwarders that have been configured can not be contacted, then your DNS server will resort to using the root hints instead. However, in some cases it is a good idea to keep this option uncheked for security reasons. Keeping this option off will prevent the DNS server from contacting any root hint servers, avoiding directly communicating to the internet.
