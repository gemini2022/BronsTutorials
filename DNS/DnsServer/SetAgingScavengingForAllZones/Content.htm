The DNS Server service supports aging and scavenging features. These features are provided as a mechanism for performing cleanup and removal of stale resource records, which can accumulate in zone data over time. You can use this procedure to set the default aging and scavenging properties for the zones on a server. To learn how to set aging/scavenging for all zones, proceed with the following steps...
  • Begin first by selecting 'Start' at the bottom left corner of your screen.
  • Then go to 'Administrative Tools'...
  • Then 'DNS'...
  • Once in DNS, right-click your server, in this case it is 'DC1'.
  • Then select 'Set Aging/Scavenging for All Zones...'. You would think by the wording, that this would configure scavenging for all zones, but, this is not the case. This only sets the default for Active Directory-Integrated zones.
  • First, select the option 'Scavenge stale resource records'.
  • DNS clients by default update their DNS resource records every 24 hours. There are two ways to do this. The first way is if the client’s IP has changed since the last update. If it has, this is considered an “update”. The second way is if the IP has NOT changed then it is considered a “refresh”. In both cases a new timestamp is set.

  • The No-Refresh interval is the period of time when the DNS server will NOT accept a “refresh” from the client. The DNS server will accept an "update' (where the IP changes), but NOT a "refresh'. The reason for this is to cut down on DNS replication. If you have multiple DC’s, then this makes sense. There is no point of updating a record with a new timestamp and causing unnecessary replication traffic if nothing else has changed. By setting the No-refresh interval you prevent these refreshes from taking place causing less replication.
  • The Refresh interval starts AFTER the no-refresh interval. Within this time, both an "update' and a "refresh" are allowed to happen. If a resource record hasn’t been updated by the time the no-refresh and refresh intervals have elapsed, then that resource record becomes eligible for scavenging.

  • Press 'OK'.
  • Next, you will be presented with a dialogue message. You will notice that the dialogue states that 'Scavenge stale resource records: Enabled'. This would tend to suggest that all zones are changed.
  • However, you'll notice here that this setting only changes the 'Default settings for new Active Directory-integrated zones:'. This does not effect existing zones, and if you decide to create any other non-Active Directory-integrated zones later, this setting will also NOT apply to those zones either.
  • If you want these settings to apply to existing Active Directory-integrated zones, you can select this option 'Apply these settings to the existing Active Directory-integrated zones'.

  • Press 'OK'.
